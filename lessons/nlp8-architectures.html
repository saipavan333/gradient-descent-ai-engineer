Track 5 · NLP & Transformers

Encoder/decoder vs decoder-only

⏱ 22 min Lesson 5.8 ⬇ Runnable notebook
Why this existsThe same transformer block, wired differently, gives BERT, GPT, and T5. The difference is just what each token may attend to — bidirectional vs causal — controlled by a single mask.
By the end, you can
  • Contrast encoder-only, decoder-only, encoder-decoder
  • Explain causal masking for generation
  • Match architectures to tasks
  • Understand why decoder-only dominates LLMs

Concept Same block, three shapes for three jobs

The transformer block (Lesson 5.6) is a universal building unit, but how you arrange the blocks — and crucially, what each token is allowed to attend to — gives three architectures suited to different tasks.

Encoder-only models (like BERT) use bidirectional attention: every token attends to every other token, both left and right. Seeing the full context in both directions makes them excellent at understanding — classification, sentiment, named-entity recognition, and producing embeddings for search (Track 7). But they can't naturally generate text, because generation needs to proceed left-to-right without peeking ahead.

Decoder-only models (like GPT) use causal (or "masked") attention: each token may attend only to itself and earlier tokens, never future ones. This left-to-right constraint is exactly what generation requires — when predicting token 5, the model mustn't see tokens 6+ (they don't exist yet, and during training it would be cheating). Decoder-only is the architecture of essentially every modern large language model.

Encoder–decoder models (like T5, and the original transformer) use both: an encoder reads the entire input bidirectionally, then a decoder generates the output causally while cross-attending to the encoder's representation. This fits transformation tasks where input and output differ — translation, summarization — because the model fully digests the source before producing the target.

Worked example The causal mask that makes a decoder

The only mechanical difference between bidirectional and causal attention is a mask: before the softmax, set the scores for "future" positions to $-\infty$, so they get zero weight. Let's apply a causal mask to a 4-token score matrix and watch the attention become lower-triangular.

import numpy as np
softmax = lambda x: np.exp(x - x.max(-1, keepdims=True)) / np.exp(x - x.max(-1, keepdims=True)).sum(-1, keepdims=True)

scores = np.array([[2., 1., .5, 1.], [1., 3., .5, 1.],
                   [.5, 1., 2., 1.], [1., 1., 1., 3.]])

# causal mask: token i may not attend to any j > i (the future)
mask = np.triu(np.ones((4, 4)), k=1).astype(bool)   # True above the diagonal
scores_masked = scores.copy()
scores_masked[mask] = -np.inf                         # future -> -inf -> 0 after softmax
A = softmax(scores_masked)
print("causal attention (each token sees only itself + past):\n", A.round(2))
Outputcausal attention (each token sees only itself + past): [[1. 0. 0. 0. ] [0.12 0.88 0. 0. ] [0.14 0.23 0.63 0. ] [0.1 0.1 0.1 0.71]]

The attention matrix is now lower-triangular — all the upper-right (future) entries are exactly 0. Token 0 can only attend to itself (weight 1.0 — it has no past). Token 1 attends to tokens 0 and 1, token 2 to 0–2, and token 3 to all four. No token ever sees the future. That single mask — setting future scores to $-\infty$ before the softmax (recall from Lesson 5.4 that $e^{-\infty} = 0$) — is the entire difference between a GPT-style generator and a BERT-style understander. Remove the mask and every token sees everything (bidirectional); add it and you have a left-to-right generator. The same blocks, one mask apart.

★ Key points to remember
  • Encoder-only (BERT): bidirectional attention — best for understanding, classification, embeddings.
  • Decoder-only (GPT): causal attention (no peeking ahead) — built for text generation; the basis of modern LLMs.
  • Encoder–decoder (T5): read input bidirectionally, generate output causally with cross-attention — for translation/summarization.
  • The causal mask sets future scores to $-\infty$ before softmax, making attention lower-triangular.
  • The architectures share the same block; what differs is what each token may attend to.

Practice Build the reasoning yourself

1Why would training a generative model without a causal mask be a form of cheating, and what would happen at generation time?
Show solution & reasoning

During training, the model predicts each next token while the full target sentence is present. Without a causal mask, when predicting token 5 the model could simply look at token 5 (and 6, 7, …) in the input — so it would learn the trivial "copy the answer" shortcut instead of actually modeling language. Training loss would plummet, but the model would have learned nothing useful.

At generation time this collapses: the future tokens don't exist yet (you're producing them one at a time), so the shortcut the model relied on is gone, and it produces garbage. The causal mask forces the model, during training, to predict each token using only the genuine past — exactly the situation it faces at generation. It aligns training with inference, which is essential for the autoregressive loop (Lesson 5.3) to work.

2For each task, pick the most natural architecture and justify it: (a) classify a movie review as positive/negative; (b) write a continuation of a story; (c) translate English to French.
Show solution & reasoning

(a) Sentiment classification → encoder-only (BERT). You have the full review available and want to understand it; bidirectional attention reads the whole text (both directions) to produce a representation you classify. No generation needed.

(b) Story continuation → decoder-only (GPT). This is pure left-to-right generation; causal attention produces each next token conditioned on what's written so far.

(c) Translation → encoder–decoder (T5-style). Input and output are different sequences; the encoder fully reads the English (bidirectionally), and the decoder generates French (causally) while cross-attending to the encoded English. That said, modern large decoder-only models do translation well too by framing it as "English: … French: …" generation — a sign of how versatile decoder-only has become (see the deep dive).

3In the worked example's causal attention matrix, token 3's row was [0.1, 0.1, 0.1, 0.71] while token 0's was [1, 0, 0, 0]. Explain both.
Show solution & reasoning

Token 0 is the first token — it has no past, so the only position it's allowed to attend to is itself. After masking, its row has a single available entry, and softmax of one value is 1.0 → [1, 0, 0, 0]. It can only ever represent itself.

Token 3 is the last token — it may attend to all four positions (itself and tokens 0–2), so none of its entries are masked. Its weights [0.1, 0.1, 0.1, 0.71] reflect the actual scores: it attends mostly to itself (0.71, the highest original score of 3) but draws a little from each earlier token. This is the general pattern of causal attention: earlier tokens have fewer positions to look at (narrower rows), later tokens see more context — the triangular structure that lets information flow forward but never backward.

★ Deep dive: why decoder-only won, and the rise of one architecture▸

A striking trend of recent years is that decoder-only models came to dominate, even for tasks that once seemed to call for encoders or encoder–decoders. Why? Several reasons converge. First, generality through prompting: a big decoder-only model can do classification, translation, and summarization by framing them all as text generation ("Review: … Sentiment:", or "Translate to French: …"). One model, one objective (next-token prediction), handles everything — no task-specific architecture needed. Second, scaling and simplicity: a uniform decoder-only stack is simpler to scale and train than an encoder–decoder, and the field discovered that scale + the right data beats architectural cleverness. Third, in-context learning: decoder-only models trained at scale can learn new tasks from examples in the prompt alone (Track 6), a capability that made the "one general model" approach irresistible. So GPT-style decoder-only became the default, and encoder-only models like BERT now occupy a narrower (if still important) niche: efficient embeddings and classification where you don't need generation.

That said, the distinctions still matter, and the picture is nuanced. Encoder-only models remain the efficient choice for retrieval embeddings (Track 7) — you don't want to run a giant decoder just to embed a document — and bidirectional context genuinely helps pure understanding tasks. Encoder–decoder models are still strong for well-defined transformation tasks. And the boundaries blur: some models use "prefix" attention (bidirectional over the prompt, causal over the generation), and multimodal models bolt encoders for images onto decoder-only text models. The deeper lesson is the one this whole track has built toward: there is a single, remarkably flexible computational primitive — the attention block — and "architecture" is mostly a matter of how you wire it and what you let each token see. Master that primitive, and the zoo of model names (BERT, GPT, T5, and their descendants) resolves into a few simple configuration choices. You now understand all of them from the inside, which is exactly the foundation Track 6 builds on to explain how these models are trained, aligned, and used at scale.
