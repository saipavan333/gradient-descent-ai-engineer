Attention, intuitively

⏱ 26 min Lesson 5.4 ⬇ Runnable notebook
Why this existsAttention lets each token gather information directly from the words that matter to it — connecting any two positions in one step. It is the mechanism that beat recurrence and powers every transformer.
By the end, you can
  • Explain what attention solves vs RNNs
  • Describe the Query/Key/Value mechanism
  • Compute attention weights with softmax of dot products
  • See attention as a soft, differentiable lookup

Concept The problem: which earlier words matter for this one?

Consider "the cat sat down because it was tired." To represent "it" correctly, the model must realize "it" refers to "cat," not "down." More generally, the meaning of every word depends on specific other words in the sentence, often far away. RNNs (Lesson 4.7) tried to carry this context in a single hidden state passed step by step, but that state is a bottleneck and forgets distant words. Attention solves it directly: it lets each word look at all the others at once and pull in information from whichever ones are relevant.

The key advantages over recurrence: attention connects any two positions in a single step (no signal has to survive a long chain, so no vanishing gradient), and it processes all words in parallel (fast on GPUs). This is why the 2017 paper that introduced the transformer was titled "Attention Is All You Need" — recurrence wasn't necessary; attention alone, done right, was enough.

Concept Query, Key, Value: attention as soft lookup

The mechanism is best understood as a soft dictionary lookup. Every token produces three vectors (by multiplying its embedding by three learned matrices): a Query (what I'm looking for), a Key (what I offer/advertise), and a Value (the information I'll contribute if selected).

The computation, for one query: compare its Query to every Key with a dot product (Lesson 1.3 — high dot product means "this key matches what I'm looking for"), apply a softmax (Lesson 3.3) to turn those scores into weights that sum to 1, then take the weighted sum of all the Values. A token whose Key strongly matches the Query contributes most of its Value to the output. Unlike a hard dictionary lookup (exact key or nothing), this is soft — every entry contributes a little, weighted by match — which makes it differentiable and trainable by backprop.

Worked example Attention for a single query

Let's compute attention for one query over three key/value pairs, by hand. The query is compared to each key (dot product), softmaxed into weights, and used to blend the values.

import numpy as np
q = np.array([1.0, 0.0])                       # the query: "looking in the x-direction"
K = np.array([[1.0, 0.0],                      # key 0: matches q well
              [0.0, 1.0],                      # key 1: orthogonal (no match)
              [0.9, 0.1]])                     # key 2: matches q almost as well
V = np.array([[10.0, 0.0], [0.0, 10.0], [5.0, 5.0]])   # each key's value

scores = K @ q                                 # dot product of q with each key
weights = np.exp(scores) / np.exp(scores).sum()  # softmax -> sums to 1
output = weights @ V                           # weighted blend of the values
print("scores (q·each key):", scores.round(2))
print("attention weights:  ", weights.round(3))
print("output:             ", output.round(2))
Outputscores (q·each key): [1. 0. 0.9] attention weights: [0.44 0.162 0.398] output: [6.39 3.61]

The query matched key 0 (score 1.0) and key 2 (0.9) well, and key 1 not at all (0.0). After softmax, keys 0 and 2 get the most weight (0.44 and 0.40) while key 1 gets little (0.16). The output is the weighted blend of the values — dominated by value 0 ([10, 0]) and value 2 ([5, 5]), giving [6.39, 3.61], pulled toward the values whose keys matched. That's attention in its entirety: match the query to keys, softmax into weights, blend the values. The next lesson applies this to a whole sentence at once (every word is a query) and adds the learned projections and scaling that make it the transformer's engine.

★ Key points to remember
  • Attention lets each token gather information directly from the other tokens that are relevant to it.
  • It connects any two positions in one step (no vanishing gradient) and runs in parallel — beating recurrence.
  • Each token makes a Query (what I seek), a Key (what I offer), and a Value (what I contribute).
  • Weights = softmax of Query·Key dot products; output = weighted sum of Values.
  • It's a soft, differentiable dictionary lookup — every entry contributes, weighted by match.

Practice Build the reasoning yourself

1In the worked example, key 1 was orthogonal to the query (score 0) yet still got weight 0.16, not 0. Why does softmax give it nonzero weight, and is that a feature or a bug?
Show solution & reasoning

Softmax exponentiates every score and normalizes, and $e^0 = 1 > 0$, so every entry gets a strictly positive weight — even a zero-scoring one. Key 1's weight is $e^0 / (e^1 + e^0 + e^{0.9}) = 1/6.18 \approx 0.16$. Nothing is ever fully excluded.

It's a feature: because all weights are positive and the operation is smooth, attention is differentiable everywhere, so gradients flow to every Query/Key and the model can learn to adjust the scores (a hard "take the max" would have zero gradient almost everywhere and couldn't be trained by backprop). If you truly want to exclude a token (e.g., future tokens in a decoder, Lesson 5.8), you set its score to $-\infty$ before the softmax, which makes $e^{-\infty} = 0$ — exactly how masking works.

2Explain, using attention, how the model could correctly resolve "it" in "The trophy didn't fit in the suitcase because it was too big."
Show solution & reasoning

When processing "it," the model forms a Query encoding "what does this pronoun refer to?" It compares that Query against the Keys of every word. The Keys for "trophy" and "suitcase" both advertise "I'm a noun this could refer to," but the surrounding context ("too big," and that something didn't fit) makes the Query match "trophy" more strongly — so attention puts most weight on "trophy"'s Value, and "it"'s representation becomes trophy-like. Change the sentence to "...because it was too small" and the Query shifts to match "suitcase" instead.

This is exactly the kind of long-range, context-dependent reference that defeated earlier models, and attention handles it directly by comparing the query word to all candidates at once. (This famous example, the "Winograd schema," is a classic test of whether a model truly uses context — and attention is what gives it a fighting chance.)

3An RNN processes a 1,000-word document, and a key fact at word 5 is needed at word 990. Contrast how an RNN and attention each handle this.
Show solution & reasoning

RNN: the fact at word 5 must survive being passed through the hidden state across 985 sequential steps. Each step risks overwriting or diluting it, and the gradient connecting word 990 back to word 5 must survive 985 multiplications — almost certainly vanishing (Lesson 4.7). So the RNN likely "forgets" the early fact by the time it's needed.

Attention: word 990 simply forms a Query and compares it to all 1,000 Keys, including word 5's, in a single step. If word 5 is relevant, its Key matches and its Value flows directly into word 990's representation — distance is irrelevant, and there's no long chain for the signal to traverse. This direct, distance-independent access is the core reason transformers handle long-range dependencies so much better, and why they replaced RNNs (Lesson 4.7's deep dive).

★ Deep dive: where Q, K, V come from, and why dot products measure relevance▸

Where do the Query, Key, and Value vectors actually come from? Each is a learned linear projection of the token's embedding: $Q = x W_Q$, $K = x W_K$, $V = x W_V$, for three weight matrices learned by backprop (Lesson 5.5 makes this concrete). This is what gives attention its power — the model learns what to look for. $W_Q$ shapes the embedding into "what this token wants to find," $W_K$ into "what this token can be found by," and $W_V$ into "what this token will share." Different attention heads learn different projections, so one head might learn to track grammatical subjects while another tracks coreference (Lesson 5.5). The separation of Key from Value is subtle but important: a token can advertise itself one way (Key) while contributing different information (Value) — like a library catalog entry (Key) pointing to the actual book (Value).

Why is a dot product the right way to measure relevance? Because, as Lesson 1.3 showed, the dot product is large when two vectors point the same way and small when they're orthogonal — it's a similarity measure. So "Query·Key is large" literally means "this key points the way my query is looking," i.e. "this token is relevant to what I want." The model learns to arrange Queries and Keys so that relevant pairs have aligned vectors (large dot product) and irrelevant pairs are orthogonal (near zero). The softmax then sharpens these into a focused weighting. It's a remarkably elegant design: the entire mechanism is built from two operations you already knew — the dot product (similarity) and the softmax (turning scores into a distribution) — wired so that "pay attention to what's relevant" becomes a differentiable, learnable computation. Lesson 5.5 writes this for a whole sequence as a couple of matrix multiplies, revealing how compact and powerful it really is.

You have the intuition: match queries to keys, blend the values. Next, in 5.5 Self-attention & multi-head attention, we write it as the exact matrix equation that runs inside every transformer — and see why using several attention "heads" at once makes it far more powerful.
