Multi-head attention & the transformer block

One attention head captures one kind of relationship. Real language has many at once — grammar, reference, topic. The fix is several heads in parallel, wrapped with two ideas that make deep stacks trainable: residual connections and normalization. Together they form the transformer block you’ll stack into a GPT.

Track 5 · Lesson 2 Concept + code ~28 min Builds on: Lesson 5.1, Track 4

💡 Multi-head attention: many relationships at once

A single attention head produces one set of weights — one “view” of how tokens relate. But a token relates to others in many ways simultaneously: one head might track grammatical subject–verb agreement, another might resolve what a pronoun refers to, another might follow the topic. Multi-head attention runs several attention mechanisms in parallel, each with its own \(W_q, W_k, W_v\), then combines them. Concretely, split the model dimension \(d\) into \(h\) heads of size \(d/h\), run attention independently in each, concatenate the results, and apply a final linear projection \(W_o\):

📐 The transformer block: attention + MLP, with two safety nets

Attention lets tokens exchange information; a per-token feed-forward MLP (Track 4) then processes each token’s gathered context, typically expanding to a larger hidden size and back (e.g. \(d\to 4d\to d\)) to add capacity and nonlinearity. A transformer block stacks these two sublayers — multi-head attention, then MLP — but wraps each in two ideas that are essential for training deep stacks:

🔑 Residual connections: the gradient highway

Residual (skip) connections — adding a sublayer’s input back to its output, \(x + f(x)\) — are what make 100-layer transformers trainable. Recall Track 4: the gradient through a deep stack is a long product that tends to vanish or explode. A residual adds an identity path: the gradient can flow straight through the + unchanged (add distributes gradients, Lesson 4.1), bypassing the sublayer. So even if a sublayer’s gradient is tiny, the skip keeps a strong signal flowing to earlier layers. This one idea, more than any other, is why “deep” learning could finally go truly deep.

The second safety net is layer normalization: before each sublayer, it re-centers and re-scales each token’s vector to zero mean and unit variance (then learns a scale and shift). This keeps activations in a stable range as they pass through many layers — the same variance-control instinct as good initialization (Track 4), applied continuously during the forward pass. Residuals keep gradients flowing; normalization keeps activations well-behaved. With both, depth becomes safe.

🛠️ From scratch: multi-head attention

code/track-5/attention.py
def multi_head_attention(X, Wq, Wk, Wv, Wo, n_heads, causal=False):
    """Split into heads, attend in each, concatenate, project."""
    T, d = X.shape
    dh = d // n_heads                         # each head's dimension
    Q, K, V = X @ Wq, X @ Wk, X @ Wv          # (T, d)
    heads = []
    for h in range(n_heads):
        sl = slice(h*dh, (h+1)*dh)            # this head's slice of the dimensions
        out, _ = attention(Q[:, sl], K[:, sl], V[:, sl], causal)
        heads.append(out)
    concat = np.concatenate(heads, axis=1)    # (T, d): rejoin the heads
    return concat @ Wo                        # final linear mix across heads

# 2 heads of size 4 over a model dimension of 8
mh = multi_head_attention(X, Wq, Wk, Wv, Wo, n_heads=2, causal=True)
print(mh.shape)        # (4, 8)  -- same shape in, same shape out, so blocks stack

Notice the output shape equals the input shape \((T, d)\) — true of the whole block (attention and MLP preserve it). That’s what lets us stack \(N\) identical blocks: each one refines the same-shaped sequence of vectors.

🔑 Key results to remember

Lock these in

  1. Multi-head attention: split \(d\) into \(h\) heads, attend in each, concatenate, project with \(W_o\). Different heads learn different relationships.
  2. A transformer block = multi-head attention sublayer + per-token MLP sublayer.
  3. Residual connections \(x + f(x)\) give gradients an identity highway — the key to training very deep stacks.
  4. LayerNorm keeps each token’s activations zero-mean/unit-variance, stabilizing deep forward passes.
  5. Block output shape = input shape \((T,d)\), so \(N\) blocks stack to form the body of a GPT.

🧩 Check your understanding
Answer from memory, then read why.

Q1. Why use multiple attention heads instead of one?

  • To reduce the number of parameters.
    Splitting one head into several of smaller size keeps the parameter count about the same; the goal isn’t fewer parameters.
  • To make attention causal.
    Causality comes from the mask, not from having multiple heads. Heads address a different need.
  • So different heads can capture different types of relationships at once.
    Correct. Each head attends in its own subspace and can specialize (syntax, reference, topic…), giving the model several simultaneous “views” of how tokens relate.
  • To avoid the softmax.
    Every head still uses softmax attention. Multiple heads don’t remove it.

Q2. What is the main purpose of residual (skip) connections in a transformer block?

  • They give gradients an identity path, so deep stacks stay trainable.
    Correct. Adding the input back (\(x+f(x)\)) lets gradient flow straight through the + to earlier layers, counteracting the vanishing-gradient problem of deep products.
  • They normalize the activations.
    Normalization is LayerNorm’s job, a separate component. Residuals are about the gradient/identity path.
  • They compute attention.
    Attention is the sublayer the residual wraps around; the skip itself just adds the input back.
  • They reduce sequence length.
    Residuals don’t change shapes at all; the sequence length is preserved throughout.

Q3. What does layer normalization do in the block?

  • It masks future tokens.
    Masking is the causal mask in attention, not LayerNorm. LayerNorm doesn’t restrict attention.
  • It adds the residual.
    The residual add is a separate step. LayerNorm rescales activations; it doesn’t perform the skip.
  • It computes the query and key.
    Q and K come from the attention projections. LayerNorm operates on the token vectors before a sublayer, not on producing Q/K.
  • It re-centers and re-scales each token’s vector to keep activations stable across layers.
    Correct. LayerNorm normalizes each token to zero mean / unit variance (then learns a scale and shift), keeping the forward signal well-behaved through a deep stack.

Q4. Why does the transformer block preserve the input shape \((T, d)\)?

  • To save memory.
    Shape preservation isn’t a memory trick; it’s about composability.
  • So identical blocks can be stacked, each refining the same-shaped sequence.
    Correct. Because attention, MLP, and residuals all map \((T,d)\to(T,d)\), you can stack any number of identical blocks — the basis of building a deep GPT.
  • Because attention requires square inputs.
    Attention doesn’t require square inputs; \(T\) and \(d\) are independent. The score matrix is \(T\times T\) but the data is \(T\times d\).
  • It doesn’t — the shape changes each block.
    It does preserve the shape; that invariance is precisely what makes stacking work.

✍️ Practice

Problem 1 · shapes

A model has dimension \(d=512\) and \(h=8\) heads. What is each head’s dimension? After concatenating all heads, what dimension returns?

Show solution

Each head operates in dimension \(d/h = 512/8 = 64\). The eight \(64\)-dimensional head outputs concatenate back to \(8\times 64 = 512 = d\), so the multi-head output is again \(d\)-dimensional (then mixed by \(W_o\)). This is the standard design: total attention dimension is preserved, just partitioned across heads, so multi-head attention costs roughly the same as single-head but is far more expressive.

Problem 2 · reasoning

Use Track 4’s view of backprop (a long product of local derivatives) to explain precisely why a residual connection helps gradients reach early layers.

Show solution

Without a residual, the output of a sublayer is \(y=f(x)\), and the gradient to \(x\) is \(\frac{\partial L}{\partial x}=\frac{\partial L}{\partial y}\cdot f'(x)\) — multiplied by the (possibly tiny) local derivative \(f'(x)\). Over many layers these factors multiply and can vanish (Lesson 4.1). With a residual, \(y = x + f(x)\), so \(\frac{\partial L}{\partial x}=\frac{\partial L}{\partial y}\cdot(1 + f'(x)) = \frac{\partial L}{\partial y} + \frac{\partial L}{\partial y}f'(x)\). The crucial first term passes the upstream gradient through unchanged (the “+1” from the identity path — recall add distributes gradients). So even if \(f'(x)\approx 0\), the gradient still reaches \(x\) at full strength via the skip. Across a deep stack, this turns a fragile product into a robust sum-with-shortcut — exactly what lets transformers be dozens or hundreds of layers deep.

Problem 3 · code

Run attention.py’s multi-head function with n_heads=1 vs n_heads=2. Confirm the output shape is \((T, d)\) either way. What changes internally?

Show solution

The output is \((4, 8)\) for both, because the heads always concatenate back to \(d=8\). What changes internally is the head dimension: with n_heads=1 there is one attention over all 8 dimensions; with n_heads=2 there are two independent attentions, each over a 4-dimensional slice, computing two separate weight matrices that are then combined by \(W_o\). Same output shape, different (richer) computation — and in a trained model the two heads would specialize in different patterns, which you could visualize as two distinct attention heatmaps.

Problem 4 · reasoning

The transformer block alternates a “mixing” sublayer (attention, which moves information between tokens) and a “processing” sublayer (the MLP, which acts on each token independently). Why is having both important?

Show solution

They do complementary jobs. Attention mixes across positions — it’s the only part of the block where tokens communicate, gathering relevant context from elsewhere in the sequence. But attention’s output is a linear blend of values, so on its own it has limited per-token expressive power. The MLP processes each token independently with a nonlinearity, transforming the gathered context into richer features (and providing most of the block’s parameters and nonlinear capacity). Alternating them — gather context, then think about it, repeat — lets the network build up increasingly abstract representations layer by layer. Drop either one and the model is crippled: no attention means no communication between tokens; no MLP means little nonlinear processing of what was gathered. The block’s power comes from the rhythm of both.
