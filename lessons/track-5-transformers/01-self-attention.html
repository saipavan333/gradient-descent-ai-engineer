Self-attention, step by step

Attention sounds mysterious until you see it’s three dot products in a trench coat. Every token asks a question, every token offers a key, and the answer is a weighted blend of values — all built from the linear algebra of Track 1.

Track 5 · Lesson 1 Concept + derivation + code ~30 min Builds on: Lesson 5.0, Track 1 dot product

💡 Concept: query, key, value

Self-attention works by a soft retrieval. Think of a library. Each token produces three vectors by multiplying its embedding by three learned matrices: a query \(\mathbf{q}\) (“what am I looking for?”), a key \(\mathbf{k}\) (“what do I contain?”), and a value \(\mathbf{v}\) (“what will I contribute if selected?”). For the whole sequence \(X\in\mathbb{R}^{T\times d}\):

$$ Q = X W_q, \qquad K = X W_k, \qquad V = X W_v. $$

To decide how much token \(i\) should attend to token \(j\), we compare \(i\)’s query to \(j\)’s key with a dot product (Track 1 — large when they’re aligned, i.e. relevant). All pairwise comparisons at once are the matrix \(QK^\top\). We scale, normalize with softmax into weights that sum to 1, and use those weights to take a weighted average of the values. That is the entire mechanism:

$$ \mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right)V. $$
(★)

📐 Why the \(\sqrt{d}\), and reading the weights

Two details earn their keep. First, the scaling by \(1/\sqrt{d}\). A dot product of two \(d\)-dimensional vectors sums \(d\) terms, so its variance grows with \(d\); for large \(d\) the scores become huge, and softmax of huge numbers saturates to a one-hot vector (all weight on one token, near-zero gradients elsewhere). Dividing by \(\sqrt{d}\) rescales the scores back to a sane range, keeping softmax soft and gradients healthy (a direct echo of the variance-scaling argument from Track 4’s initialization). Second, the softmax output is a set of attention weights — for each query token, a probability distribution over which tokens to attend to. We can visualize the whole matrix as a heatmap:

🔑 The causal mask makes a language model

To train a model to predict the next token, each position must only use information from itself and earlier positions — otherwise it could “cheat” by peeking at the answer. We enforce this by adding \(-\infty\) to the scores of future positions before the softmax, so their weights become exactly 0. This single trick — the causal (autoregressive) mask — is what turns generic attention into a generative language model like GPT. Without the mask you get a bidirectional encoder (like BERT); with it, a generator.

One more piece of vocabulary: when \(Q\), \(K\), and \(V\) all come from the same sequence (as here), it’s self-attention — the sequence attending to itself. When the queries come from one sequence and the keys/values from another (e.g. a translation decoder attending to the source sentence), it’s cross-attention. GPT uses masked self-attention throughout.

🛠️ From scratch: scaled dot-product attention

code/track-5/attention.py
import numpy as np

def softmax(z, axis=-1):
    z = z - z.max(axis=axis, keepdims=True)   # stability: subtract the max
    e = np.exp(z)
    return e / e.sum(axis=axis, keepdims=True)

def attention(Q, K, V, causal=False):
    d = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d)            # (T,T): every query · every key
    if causal:                               # GPT: position i can't see j > i
        T = scores.shape[0]
        scores = scores + np.triu(np.full((T, T), -1e9), 1)   # -inf above diagonal
    weights = softmax(scores, axis=1)        # each row sums to 1
    return weights @ V, weights              # blended values, and the weights

# 4 tokens, dimension 8
rng = np.random.default_rng(0); T, d = 4, 8
X = rng.normal(size=(T, d))
Wq, Wk, Wv = (rng.normal(0, 1/np.sqrt(d), (d, d)) for _ in range(3))
Q, K, V = X @ Wq, X @ Wk, X @ Wv
out, weights = attention(Q, K, V, causal=True)
print(np.round(weights, 2))      # lower-triangular; rows sum to 1
print(weights.sum(axis=1))       # [1. 1. 1. 1.]

Run it: the weight matrix is lower-triangular (the causal mask at work), each row sums to 1, and the output is each token’s context — a learned blend of the values it chose to attend to. That’s self-attention, complete, in a dozen lines.

🔑 Key results to remember

Lock these in

  1. Each token yields a query, key, value: \(Q=XW_q,\ K=XW_k,\ V=XW_v\).
  2. Attention \(=\mathrm{softmax}(QK^\top/\sqrt{d})\,V\): query·key dot products → scaled → softmax weights → weighted sum of values.
  3. \(\sqrt{d}\) scaling keeps scores from blowing up so softmax stays soft and gradients flow.
  4. The causal mask (\(-\infty\) on future positions) prevents peeking ahead — turning attention into an autoregressive language model.
  5. Self-attention: Q, K, V from the same sequence. The whole thing is dot products and a softmax — Track 1 made powerful.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. In attention, how is the relevance of token \(j\) to token \(i\) measured?

  • The Euclidean distance between their embeddings.
    Attention uses an inner product, not a distance. It scores alignment via a dot product of query and key.
  • The difference of their values.
    Values are what get blended after the weights are computed; relevance itself is a query·key comparison.
  • The dot product of \(i\)’s query with \(j\)’s key.
    Correct. \(\mathbf{q}_i\cdot\mathbf{k}_j\) is large when the query and key are aligned — high relevance. This is the dot product from Track 1.
  • By counting how many words apart they are.
    Distance in the sequence isn’t how relevance is scored (that role belongs to positional encodings). Relevance is content-based via query·key.

Q2. Why divide the scores by \(\sqrt{d}\) before the softmax?

  • To stop large \(d\) from making scores huge and saturating the softmax.
    Correct. A dot product over \(d\) terms grows with \(d\); without scaling, softmax of large scores becomes one-hot with vanishing gradients. \(1/\sqrt{d}\) restores a healthy range.
  • To make the weights sum to 1.
    The softmax already guarantees the weights sum to 1, with or without scaling. The scaling addresses score magnitude, not normalization.
  • To apply the causal mask.
    The causal mask is a separate step (adding \(-\infty\) to future scores). Scaling is unrelated to masking.
  • To reduce the number of parameters.
    Dividing by a constant changes no parameters; it only rescales the scores.

Q3. What does the causal mask accomplish?

  • It makes attention faster.
    Masking doesn’t speed up the core computation; it changes what each token can attend to.
  • It normalizes the attention weights.
    Normalization is the softmax’s job. The mask instead forbids certain attentions.
  • It lets tokens attend to the future.
    The opposite — it prevents attending to future tokens, so the model can’t cheat when predicting the next one.
  • It prevents each position from attending to future tokens, enabling next-token prediction.
    Correct. By zeroing the weights on future positions (via \(-\infty\) scores), each token sees only itself and the past — exactly what a left-to-right generator requires.

Q4. Each row of the attention weight matrix (after softmax)…

  • sums to the number of tokens.
    Softmax outputs a probability distribution, which sums to 1, not to \(T\).
  • is a probability distribution over which tokens to attend to (sums to 1).
    Correct. For each querying token, softmax produces non-negative weights summing to 1 — how much attention to pay to each other token.
  • contains the raw dot-product scores.
    Those are the inputs to softmax. After softmax they become normalized weights, not raw scores.
  • is always one-hot.
    Only if the scores are extreme (which the \(\sqrt{d}\) scaling avoids). Normally the weights are a soft blend over several tokens.

✍️ Practice

Problem 1 · shapes

For a sequence of \(T=10\) tokens with model dimension \(d=64\), what are the shapes of \(Q\), \(K\), \(V\), the score matrix \(QK^\top\), and the attention output?

Show solution

\(X\) is \(10\times 64\). Each projection \(W\) is \(64\times 64\), so \(Q=XW_q\), \(K\), \(V\) are each \(10\times 64\). The scores \(QK^\top\) are \((10\times 64)(64\times 10) = 10\times 10\) — one score per (query, key) pair, which is the \(O(T^2)\) cost from Lesson 0. After softmax (still \(10\times 10\)), the output \(\text{weights}\cdot V\) is \((10\times 10)(10\times 64) = 10\times 64\) — same shape as the input, so layers can stack. Tracking these shapes is the single most useful habit for reading transformer code.

Problem 2 · reasoning

If two tokens have identical query and key vectors, and the scores were not scaled, what happens to the softmax as \(d\) grows very large? Why is that bad for training?

Show solution

The dot product of identical vectors is \(\lVert\mathbf{q}\rVert^2\), which grows with \(d\) (more terms in the sum). Without the \(1/\sqrt{d}\) scaling, one score dwarfs the others, and softmax pushes nearly all the weight onto that single token — effectively a hard, one-hot selection. That’s bad for training because softmax in its saturated region has near-zero gradient (Track 4’s vanishing-gradient story): the model can barely adjust the attention pattern, so learning stalls. Scaling by \(\sqrt{d}\) keeps the scores moderate, the softmax soft, and the gradients informative — a small constant with a big effect.

Problem 3 · code

In attention.py, set causal=False and inspect the weight matrix. How does it differ from the causal case, and what kind of model does that correspond to?

Show solution

With causal=False, the weight matrix is full (not lower-triangular): every token attends to every other token, including those after it. Each row still sums to 1, but now positions can use both past and future context. This corresponds to a bidirectional encoder like BERT — excellent for understanding tasks (classification, retrieval) where the whole input is available at once, but unusable for left-to-right generation, since predicting the next token while seeing it is cheating. The single flag flips the architecture between “understander” and “generator.”

Problem 4 · synthesis

Explain how self-attention is, at its core, an application of three ideas you already mastered: the dot product, matrix multiplication, and softmax.

Show solution

Self-attention is those three ideas composed. The projections \(Q=XW_q\) (etc.) are matrix multiplications (Track 1, Lesson 5) that learn what to look for, offer, and contribute. The scores \(QK^\top\) are a matrix of dot products (Track 1, Lesson 3) measuring query–key alignment — relevance as geometric similarity. The softmax (Track 1, Lesson 10) turns those scores into a probability distribution — the attention weights. And the output \(\text{weights}\cdot V\) is again a matrix multiply, blending values. So the headline idea behind every large language model is just: project, dot-product, softmax, blend — four operations you can now implement and differentiate yourself. The “magic” of transformers dissolves into linear algebra and probability the moment you write it down.
