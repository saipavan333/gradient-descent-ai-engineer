Positional encodings

Attention has a surprising blind spot: it treats a sentence as a bag of words, with no sense of order. “Dog bites man” and “man bites dog” look identical to it. Positional encodings fix this by stamping each token with where it sits — restoring the sequence to the sequence model.

Track 5 · Lesson 3 Concept + code ~22 min Builds on: Lessons 5.0–5.2

💡 Concept: attention can’t see order

Look again at attention: \(\mathrm{softmax}(QK^\top/\sqrt{d})V\). Every operation — the projections, the dot products, the weighted sum — treats the tokens symmetrically. If you shuffle the input tokens, the outputs just shuffle the same way; the computation itself has no notion of first, second, or last. In math terms, self-attention is permutation-equivariant. That’s a disaster for language, where order is meaning: “dog bites man” and “man bites dog” are different facts built from the same words. The model needs to know where each token sits.

The fix is elegant: give each position its own vector and add it to the token embedding before the first block. Now a token’s representation encodes both what it is (the token embedding) and where it is (the positional encoding), so attention can use position when it matters.

📐 Two ways to encode position

The original transformer used a fixed sinusoidal encoding: each dimension of the position vector is a sine or cosine of the position, at a different frequency. Low frequencies change slowly across the sequence (coarse position), high frequencies change quickly (fine position) — together they give every position a unique, smooth fingerprint:

$$ \mathrm{PE}(pos, 2i) = \sin\!\left(\frac{pos}{10000^{2i/d}}\right), \qquad \mathrm{PE}(pos, 2i{+}1) = \cos\!\left(\frac{pos}{10000^{2i/d}}\right). $$

The other approach — used by GPT-2 and by our own minigpt.py — is a learned positional embedding: simply a trainable table with one vector per position, learned like any other weights. It’s less elegant but works well and is dead simple. Either way, the principle is the same: position is added to content. Modern models often use rotary encodings (RoPE), which inject relative position directly into the attention dot product and extrapolate better to long contexts — but that’s a refinement of this same idea.

🔑 Content + position

The model’s input to the first block is token embedding + positional encoding. The token embedding answers “what is this token?”; the positional encoding answers “where is it?” Adding them lets every downstream attention operation be sensitive to both — without it, a transformer is just a powerful set-processor that can’t tell a sentence from its anagram.

🛠️ From scratch: positions, and the permutation test

code/track-5/positional.py
import numpy as np

def sinusoidal_encoding(T, d):
    pos = np.arange(T)[:, None]; i = np.arange(d)[None, :]
    angle = pos / np.power(10000, (2 * (i // 2)) / d)
    return np.where(i % 2 == 0, np.sin(angle), np.cos(angle))   # (T, d)

# Demonstrate that bare attention ignores order:
X = np.random.default_rng(0).normal(size=(3, 8))
perm = [2, 0, 1]
out1 = attention_out(X)            # attention over X
out2 = attention_out(X[perm])      # attention over a shuffled X
print(np.allclose(out1[perm], out2))   # True -> outputs just permute; order ignored!

# Add positions -> order now matters:
Xp      = X        + sinusoidal_encoding(3, 8)
Xp_perm = X[perm]  + sinusoidal_encoding(3, 8)
print(np.allclose(attention_out(Xp)[perm], attention_out(Xp_perm)))   # False

The first check prints True — proof that attention alone is permutation-equivariant (it can’t distinguish word orders). After adding positional encodings, the second check prints False — order now changes the result, exactly as language requires.

🔑 Key results to remember

Lock these in

  1. Self-attention is permutation-equivariant — by itself it has no notion of token order.
  2. Fix it by adding a positional encoding to each token embedding: representation = what (token) + where (position).
  3. Sinusoidal encodings use sines/cosines of many frequencies; learned encodings are a trainable position table (what GPT-2 and minigpt use).
  4. Modern models often use rotary (RoPE) encodings for better relative-position handling and long-context extrapolation.

Q1. Without positional information, how does self-attention treat “dog bites man” vs “man bites dog”?

  • It distinguishes them by word order automatically.
    It can’t — attention has no built-in sense of order. That’s precisely the problem positional encodings solve.
  • It can’t tell them apart (beyond a corresponding shuffle of outputs).
    Correct. Attention is permutation-equivariant: shuffle the inputs and the outputs just shuffle the same way. The two sentences are indistinguishable to bare attention.
  • It refuses to process them.
    It processes both fine — it just processes them identically, which is the issue.
  • It uses the causal mask to order them.
    The causal mask restricts attention to the past but doesn’t encode which position is which in a way that distinguishes content order; positions are still needed.

Q2. How is positional information combined with token information?

  • By multiplying the position into the attention scores.
    In the standard scheme, position isn’t multiplied into scores; it’s added to the input embeddings. (RoPE is a different, rotary mechanism.)
  • By concatenating positions as extra tokens.
    Positions aren’t added as separate tokens; they’re combined with each existing token’s vector.
  • By replacing the token embedding entirely.
    Position augments the token embedding; it doesn’t replace it — the model still needs to know what the token is.
  • By adding a positional vector to each token’s embedding.
    Correct. Input to the first block = token embedding + positional encoding, so each vector carries both content and position.

Q3. What distinguishes a learned positional embedding from a sinusoidal one?

  • The learned one is a trainable table of position vectors; the sinusoidal one is a fixed formula.
    Correct. Learned positions are parameters updated by training (like the token embedding table); sinusoidal positions are computed from a fixed sine/cosine formula and never trained.
  • The learned one ignores order.
    Both encode order; the difference is fixed-formula vs trained, not whether they capture position.
  • The sinusoidal one is trainable.
    Backwards — the sinusoidal encoding is the fixed one; the learned table is the trainable one.
  • They produce identical vectors.
    They generally differ (one is data-driven, one is a formula), though both serve the same role.

Q4. In positional.py, the permutation test prints True for bare attention and False after adding positions. What does this demonstrate?

  • That attention is broken.
    It’s not broken — being permutation-equivariant is correct behavior for a set operation. It just isn’t enough for ordered sequences.
  • That positional encodings slow the model down.
    The test is about order-sensitivity, not speed. Positional encodings don’t meaningfully change compute.
  • That bare attention ignores order, and adding positions makes order matter.
    Correct. True shows shuffling inputs only shuffles outputs (order-blind); False shows that once positions are added, the result depends on order — the fix working.
  • That the softmax is incorrect.
    The softmax is fine; the test isolates the order-sensitivity question, which is about positions, not softmax.

✍️ Practice

Problem 1 · reasoning

Why use many different frequencies in the sinusoidal encoding, rather than a single sine wave of position?

Show solution

A single sine wave can’t uniquely identify positions: it repeats every period, so positions a full wavelength apart get the same value (aliasing), and a single low-resolution signal can’t pin down position precisely. Using many frequencies — like the bits of a binary counter, or the hands of a clock (hour, minute, second) — lets the combination of values uniquely and precisely encode each position over a long range: high frequencies resolve nearby positions finely, low frequencies distinguish far-apart positions without repeating. Together they give every position a distinct, smoothly varying fingerprint.

Problem 2 · reasoning

Our minigpt.py uses a learned position table of shape \((T, d)\). What limitation does this impose at generation time compared to sinusoidal encodings?

Show solution

A learned table only has rows for positions \(0\) to \(T-1\) that it saw during training, so it cannot represent positions beyond \(T\) — the model can’t handle sequences longer than its trained context length (there’s simply no vector for position \(T\)). Sinusoidal encodings, being a formula, produce a vector for any position, so they can in principle extrapolate to longer sequences (though quality still degrades). This is exactly why long-context research favors methods like rotary/RoPE encodings that handle relative position and extrapolate more gracefully. The trade-off — simplicity of learned tables vs extrapolation of formulas — is a real design decision.

Problem 3 · code

Run positional.py and confirm the two boolean prints (True then False). In your own words, what would break in a language model if you forgot to add positional encodings?

Show solution

The prints confirm bare attention is order-blind (True) and that positions restore order-sensitivity (False). Forgetting positional encodings would make the model treat every input as an unordered bag of tokens: it could still learn which words co-occur, but it could never reliably model grammar, word order, or any meaning that depends on sequence — “the cat chased the dog” and “the dog chased the cat” would be identical to it. Generation would produce word-salad with the right vocabulary but no coherent structure. It’s a subtle bug because the model still trains and outputs something; the failure shows up as a ceiling on quality, which is why this “add positions” line is never optional.

Problem 4 · reasoning

Token embeddings and positional encodings are simply added together. Why doesn’t the model get “confused” mixing a token’s identity with its position into one vector?

Show solution

Because the model has plenty of dimensions and learns to separate the two signals as needed. In a high-dimensional space, the token-identity directions and the position directions can be (approximately) different subspaces, so a downstream linear projection (in the attention \(W_q, W_k\)) can read off “what” and “where” independently from the summed vector. The model isn’t handed a clean split — it learns to disentangle content and position during training because doing so helps it predict tokens. This is a recurring theme: neural networks are remarkably good at separating superimposed signals when the architecture gives them room (dimensions) and the task gives them reason. Empirically, adding (rather than concatenating) works well and saves dimensions, which is why it’s the standard.
