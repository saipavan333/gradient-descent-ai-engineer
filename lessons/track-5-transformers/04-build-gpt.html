💡 Concept: assembling a GPT

A GPT (Generative Pre-trained Transformer) stacks the pieces you’ve built into one autoregressive language model. The data flows straight up: tokens are embedded and given positions, passed through \(N\) transformer blocks (masked multi-head attention + MLP, each with residuals and normalization), normalized once more, then projected to a score for every word in the vocabulary and softmaxed into a probability distribution over the next token.

📐 Training: predict the next token

How do we train it? With the simplest, most powerful objective in modern AI: next-token prediction. Show the model a chunk of text and, at every position, ask it to predict the following token. The loss is the cross-entropy between its predicted distribution and the actual next token — which, from Track 1, is exactly the negative log-likelihood of the true tokens under the model. Minimizing it is maximum likelihood (Track 1), driven by the training loop and gradients of Track 4. The causal mask (Lesson 1) is what makes this honest: each position predicts its successor using only earlier tokens, so a single pass over a sequence gives \(T\) prediction problems at once. Train on enough text and the model must learn grammar, facts, and reasoning patterns — because all of them help predict the next word.

Once trained, the model generates autoregressively: feed it a prompt, sample from its next-token distribution, append the sampled token, and repeat — each new token becoming part of the context for the next:

It works. Starting from random weights, the loss falls from \(2.61\) to \(0.13\), and the model — using gradients you derived — learns to continue the text it was trained on. The core of the forward and backward pass is short enough to read in one sitting:

code/track-5/minigpt.py — the forward pass
def forward(self, x, y=None):
    T, d = len(x), self.d
    E = self.Wemb[x] + self.Wpos[:T]               # token embed + position
    Q, K, Vv = E@self.Wq, E@self.Wk, E@self.Wv     # project to Q, K, V
    A = softmax((Q@K.T)/np.sqrt(d) + self.mask[:T,:T], axis=1)  # masked attention
    C = A@Vv                                        # gather context
    logits = C@self.Wo                             # project to vocabulary
    P = softmax(logits, axis=1)                     # next-token probabilities
    if y is None: return logits, None, (x,E,Q,K,Vv,A,C,P)
    loss = -np.mean(np.log(P[np.arange(T), y] + 1e-12))   # cross-entropy = NLL
    return logits, loss, (x,E,Q,K,Vv,A,C,P)
🔑 You just built the architecture behind modern AI

This tiny model and a frontier model like GPT-4 are the same architecture. The differences are scale and engineering, not concept: billions of parameters instead of thousands; dozens of stacked blocks and many heads instead of one; subword tokenization over a huge vocabulary; trillions of tokens of training text; Adam with learning-rate schedules; and a fine-tuning stage (RLHF, Track 7) to make it follow instructions. But the heart — embed, add positions, masked multi-head attention in stacked residual blocks, predict the next token, generate autoregressively — is exactly what you just implemented and trained. You now understand large language models from the inside.

🔑 Key results to remember

Lock these in

  1. GPT = token+positional embeddings → \(N\) transformer blocks → final norm → linear to vocab → softmax = \(P(\text{next token})\).
  2. Training objective: next-token prediction via cross-entropy = negative log-likelihood = maximum likelihood (Track 1). The causal mask gives \(T\) predictions per sequence.
  3. Generation is autoregressive: sample next token, append, repeat.
  4. A from-scratch numpy GPT (gradient-checked) learns to generate — loss \(2.6\to0.1\) on a tiny corpus.
  5. Frontier LLMs are the same architecture scaled up (params, data, blocks, subwords, Adam) plus instruction fine-tuning.

🧩 Check your understanding
Answer from memory, then read why.

Q1. What is a GPT trained to do?

  • Classify whole sentences into categories.
    That’s a classification model (often an encoder like BERT). GPT is a generative model with a different objective.
  • Reconstruct masked-out words in the middle of text.
    That’s masked language modeling (BERT-style). GPT predicts the next token left-to-right, not masked ones.
  • Predict the next token given the previous ones.
    Correct. GPT is an autoregressive next-token predictor; its causal mask enforces left-to-right prediction, and generation samples one token at a time.
  • Translate between two fixed languages only.
    GPT is general next-token prediction; translation is just one of many tasks it can do, not its defining objective.

Q2. The cross-entropy loss used to train a GPT is equivalent to what, from Track 1?

  • The negative log-likelihood of the true tokens (maximum likelihood).
    Correct. Cross-entropy between the predicted distribution and the actual next token is exactly the negative log-likelihood; minimizing it is maximum likelihood estimation — the bridge from Track 1, Lesson 12.
  • The determinant of the weight matrix.
    Determinants are unrelated to the training loss. The loss measures how probable the data is under the model.
  • The dot product of two embeddings.
    Dot products appear inside attention, but the loss is cross-entropy / negative log-likelihood, not a single dot product.
  • The variance of the predictions.
    Variance isn’t the objective. The training target is the likelihood of the correct next tokens.

Q3. How does a trained GPT generate a sentence?

  • It outputs all tokens at once.
    It produces one token at a time; the whole sentence emerges from repeating the process, not from a single parallel output.
  • It picks tokens at random from the vocabulary.
    Tokens are sampled from the model’s learned next-token distribution, which is far from uniform random — it reflects what’s likely given the context.
  • It reverses the training loss.
    Generation doesn’t “reverse” a loss; it runs the trained forward pass and samples from the output distribution.
  • Predict the next token, append it, feed the longer context back, and repeat.
    Correct. Autoregressive generation: sample from \(P(\text{next token}\mid\text{context})\), append, and repeat with the extended context — exactly how the model writes word by word.

Q4. Our minigpt and GPT-4 share the same core architecture. Which is not primarily a difference of scale/engineering?

  • Number of parameters and layers.
    This is exactly a scale difference — GPT-4 has vastly more parameters and blocks, but the same kind of components.
  • The fundamental mechanism (attention over tokens to predict the next one).
    Correct. The core mechanism is identical — that’s the lesson’s point. It’s not a difference; everything else (size, data, optimizer, tokenizer, fine-tuning) is the scaling/engineering that differs.
  • Amount of training data.
    A real difference of scale — frontier models train on far more text.
  • Instruction fine-tuning (e.g. RLHF).
    A real added stage in production models (Track 7) that our minigpt lacks — an engineering/training difference, not a change to the core architecture.

✍️ Practice

Problem 1 · run it

Run minigpt.py and confirm the loss decreases and it generates text resembling the training corpus. Then change the corpus to a different short phrase and retrain — does it learn the new pattern?

Show solution

The loss should drop from ~2.6 toward ~0.1–0.3, and generation will echo the training text (it overfits this tiny corpus by design — that’s how we see it learned). Swap in a different phrase (say a nursery rhyme line) and it learns that instead, reproducing its structure after training. This hands-on confirmation — change the data, the model learns the new data — is the essence of a language model: it has no built-in knowledge; everything it “knows” comes from the text it predicts. Scale the corpus from one phrase to the internet and the same mechanism yields a model that has absorbed a huge swath of human text.

Problem 2 · reasoning

Our minigpt has a single attention head and a single layer. Name two specific upgrades from this track that would make it substantially more capable, and why.

Show solution

Two high-impact upgrades: (1) Stack multiple transformer blocks with residual connections and LayerNorm (Lesson 2) — depth lets the model build hierarchical representations (combine local patterns into higher-level structure), and residuals keep it trainable. (2) Multi-head attention (Lesson 2) — several heads capture different relationships simultaneously (syntax, reference, topic), which a single head cannot. A third would be adding the per-token MLP sublayer for nonlinear processing capacity. Each is exactly the difference between a toy and a real transformer, and each is something you now understand well enough to implement. The path from minigpt to a genuinely useful model is not a mystery — it’s these known upgrades plus scale and data.

Problem 3 · reasoning

Why does training on “just predict the next token” cause a model to learn things like grammar, facts, and even reasoning — capabilities nobody explicitly programmed?

Show solution

Because next-token prediction is a universal task that secretly requires understanding. To predict the next word well across billions of diverse texts, the model is forced to internalize whatever helps: grammar (to predict syntactically valid continuations), facts (to complete “the capital of France is ___”), arithmetic and logic (to continue “2+2=” or a step of reasoning), style, and more. There is no separate “grammar module” — these capabilities emerge as compressed regularities the model discovers because they reduce prediction loss. This is the surprising and profound lesson of large language models: a simple objective (maximum likelihood of the next token, Track 1) at enormous scale yields broad competence. Understanding why and when capabilities emerge — and where they fail — is one of the central open questions of modern AI research, and exactly the kind of question your first-principles foundation prepares you to investigate.

Problem 4 · synthesis

Trace a single forward pass of minigpt and name the track that gave you each component: embeddings, attention scores, softmax, the loss, and the gradients.

Show solution

The whole model is your curriculum, executing. Embeddings (token + position) are vectors and a lookup — Track 1. Q/K/V projections are matrix multiplies — Track 1, Lesson 5. The attention scores \(QK^\top\) are dot products measuring similarity — Track 1, Lesson 3. The softmax turning scores into a distribution — Track 1, Lesson 10. The loss (cross-entropy / negative log-likelihood) is maximum likelihood — Track 1, Lesson 12. The gradients come from backprop / the chain rule — Tracks 2 and 4. The weight updates are Adam / gradient descent — Track 2. And whether the trained model generalizes rather than merely memorizes is Track 3. A GPT is not a separate magical thing; it is the entire foundation you built, composed. That you can now name every part — and implement it — is precisely the goal we set out to reach.
