Start Here — how to use this curriculum

Read this firstYou are about to go from zero to job-ready. This page is your map and your method — how the curriculum is built, how to actually learn it (not just read it), and a realistic plan to finish. Five minutes here will save you weeks.

The one rule Follow the path in order

The 14 tracks are a true path: every lesson only uses ideas already taught. Start at Lesson 1.1 and use the Next → button at the bottom of each lesson. The sidebar is the whole map — a green badge means a track is complete. Don't skip ahead: the later tracks (LLMs, RAG, agents, system design) quietly rest on the earlier ones (gradients, generalization, transformers). If a lesson feels hard, the fix is almost always to revisit the earlier lesson it builds on.

Method How to actually learn it — not just read it

Reading is the weakest form of learning. The people who become engineers do the material. Five habits, in order of impact:

★ The five habits that make it stick
  • Build, don't just read. Every built lesson has a runnable notebook. Open it, run it, then change a number and predict what happens before you re-run. Tinkering beats re-reading every time.
  • Try the practice before peeking. Each lesson has practice problems with full solutions. Attempt your own answer first — even a bad one — then read the solution. The struggle is where the learning happens.
  • Use active recall. After a lesson, close it and say the key idea out loud in one sentence. Take the quiz. If you can't explain it without looking, you haven't learned it yet — that's useful information, not failure.
  • Teach it back (the Feynman test). Explain the lesson to an imaginary beginner (or a friend, or a rubber duck). The moment you get stuck is the exact gap to go fix.
  • Space it out. An hour a day for two months beats a 12-hour weekend. Sleep consolidates learning; spacing lets it. Revisit older tracks briefly as you go.

The test for every lesson is the one printed at the bottom of each: could you now explain this to someone else, and use it on the job? If yes, move on. If no, do the practice and re-run the notebook.

Anatomy How to use each part of a lesson

Every built lesson has the same shape. Here's how to spend your attention on each part:

  • Why this exists (top box) — read it; it tells you what problem the lesson solves so the details have a place to hang.
  • Concept — the plain-English idea, every term defined before use. Read actively: pause and predict the next step.
  • Worked example — real, runnable code with real numbers. Run it in the notebook. This is where understanding becomes concrete.
  • Key points — the few things worth memorizing. Come back to these for review.
  • Quiz — four questions with explained wrong answers. Do it honestly, no peeking; the explanations teach as much as the questions.
  • Practice — reason it out yourself first, then read the solution. This is the highest-value part.
  • Deep dive — optional extra rigor. Skip on the first pass if you're moving fast; return when you want depth.
  • Diagrams — every flow gets a picture. If a concept is fuzzy, study its diagram.

Plan A realistic schedule to finish

There are 117 lessons. Each takes roughly 30–45 focused minutes (reading + quiz + practice + running the notebook), so the whole path is about 60–90 hours of real work. Pick the pace that fits your life and protect the habit — consistency beats intensity:

★ Choose your pace
  • Steady (recommended): ~1 hour/day → about 2–3 months. Sustainable; the spacing helps retention.
  • Relaxed: 1 lesson/day → about 4 months. Great if you're working or studying alongside.
  • Intensive: 3–4 lessons/day → about 1 month. Feasible if you can go full-time, but build in rest days so it consolidates.

Your first week, concretely: finish Track 1 (Math Foundations) and start Track 2 (Python). Run every notebook. By the end of week one you'll have the two foundations everything else stands on — and proof that you can keep the habit. If the math feels intimidating: it's taught from zero, every term defined; trust the path.

Tools Using the runnable notebooks

Every built lesson links a ⬇ Runnable notebook (a .ipynb file). To use it: download it, then open it in Jupyter (on your computer) or upload it to Google Colab (free, in your browser — no setup). Run each cell, confirm the output matches the lesson, then experiment. If you're brand new to notebooks, Lesson 2.1 explains them. Running the code is not optional — it's how the ideas become real.

Milestones How you'll know you're getting there

  • Foundations solid — you finished Tracks 1–3 and can explain bias–variance, gradient descent, and precision vs recall without looking.
  • Modern AI fluent — you finished Tracks 4–8 and can explain what attention does, why LLMs hallucinate, and how RAG and agents work.
  • Production-ready — you finished Tracks 9–11 and can reason about serving, monitoring, drift, system design, and evaluation.
  • Interview-ready — you've worked the Interview Bank (Track 12) out loud and can hold your own on fundamentals, design, and behavioral questions.
  • Job-ready — you've built the Capstones (Track 13): a portfolio of real, reproducible projects you can deep-dive in an interview. That's the finish line — and the start of your career.
