Derivatives & gradients, geometrically

Training a model means rolling downhill on a loss surface. Before we can descend, we need the one tool that tells us which way is down and how steep it is: the derivative in 1D, and its multivariable big sibling, the gradient.

Track 2 · Lesson 0 Concept + code ~26 min Builds on: Track 1

Track 1 ended with a profound idea: training a model is maximum likelihood, which means minimizing a loss. But how do you minimize a function of millions of parameters that you can’t even picture? You feel your way downhill. This track makes “downhill” precise. It starts here, with slope.

💡 Concept: the derivative is a slope

Take a function of one variable, \(f(x)\). Its derivative at a point, written \(f'(x)\) or \(\frac{df}{dx}\), is the slope of the tangent line there — the rate at which \(f\) changes as you nudge \(x\). A positive slope means \(f\) is rising as \(x\) increases; a negative slope means it’s falling; a slope of zero means you’re momentarily flat — at a peak, a valley, or a plateau.

“As the run shrinks to zero” is the whole subtlety, and it’s captured by a limit. The slope between \(x\) and a nearby point \(x+h\) is \(\frac{f(x+h)-f(x)}{h}\) (rise over run). The derivative is what that ratio approaches as \(h\) becomes infinitesimally small:

$$ f'(x) = \lim_{h\to 0}\frac{f(x+h)-f(x)}{h}. $$
(★)

The symbol \(\lim_{h\to 0}\) means “the value this expression homes in on as \(h\) gets arbitrarily close to \(0\).” You rarely compute derivatives from (★) by hand — a few rules do it for you — but every rule traces back to this definition, and we’ll use (★) directly in code to check our work.

📖 The handful of rules you’ll reuse

Power rule: \(\frac{d}{dx}x^n = n x^{n-1}\) (so \(\frac{d}{dx}x^2 = 2x\)). Sum rule: the derivative of a sum is the sum of derivatives. Constant multiple: \(\frac{d}{dx}\,c\,f = c\,f'\). And the one that powers all of deep learning — the chain rule: if \(y = f(g(x))\), then \(\frac{dy}{dx} = f'(g(x))\cdot g'(x)\) (“derivative of outside times derivative of inside”). We’ll lean on the chain rule hard when we build backpropagation in Track 4.

📐 The gradient: slope in many directions at once

A loss depends on many parameters, not one. For a function \(f(x_1,\dots,x_n)\) of several variables we need a slope for each input direction. A partial derivative \(\frac{\partial f}{\partial x_i}\) is the ordinary derivative with respect to \(x_i\) while holding all other variables fixed — the slope you’d feel walking purely along the \(x_i\) axis. (The rounded \(\partial\) just signals “partial.”) Stack all \(n\) partial derivatives into a vector (Track 1!) and you get the gradient:

$$ \nabla f(\mathbf{x}) = \left(\frac{\partial f}{\partial x_1},\ \frac{\partial f}{\partial x_2},\ \dots,\ \frac{\partial f}{\partial x_n}\right). $$
(‡)

The upside-down triangle \(\nabla\) is read “nabla” or “grad.” The gradient is the single most important object in optimization because of one geometric fact: the gradient points in the direction of steepest ascent — the compass bearing in which \(f\) increases fastest — and its length \(\lVert\nabla f\rVert\) is how steep that climb is. Therefore \(-\nabla f\) points in the direction of steepest descent: the fastest way downhill. That arrow is the one we will follow, over and over, to train every model.

💡 Why perpendicular to the contours?

Along a contour line, \(f\) doesn’t change at all (that’s what “equal height” means), so the slope along the contour is zero. The direction of maximum change must therefore be the one with no component along the contour — i.e. exactly perpendicular to it. That’s the gradient.

Why steepest ascent? (the dot-product argument)

Here’s the quick reason, and it reuses the dot product from Track 1. If you take a tiny step in a unit direction \(\mathbf{u}\), the change in \(f\) is approximately \(\nabla f \cdot \mathbf{u}\) (the gradient dotted with your step direction — this is the “directional derivative”). From Lesson 3, \(\nabla f\cdot\mathbf{u} = \lVert\nabla f\rVert\cos\theta\), which is largest when \(\cos\theta = 1\), i.e. when \(\mathbf{u}\) points the same way as \(\nabla f\). So the steepest increase is in the direction of the gradient itself — and the steepest decrease is exactly opposite. Geometry settles it.

🤖 Why this is the engine of learning

A model’s loss \(L(\boldsymbol{\theta})\) is a function of its parameter vector \(\boldsymbol{\theta}\) (all its weights, stacked into one big vector). The gradient \(\nabla L\) is a vector with one entry per parameter, each saying “nudge me this way to increase the loss.” So \(-\nabla L\) says how to nudge every parameter to decrease the loss — to make the model better. Computing this gradient efficiently for networks with billions of parameters is exactly what backpropagation (Track 4) does, and following it downhill is gradient descent (the very next lesson). Everything in this track is built on the arrow \(-\nabla L\).

🛠️ From scratch: derivatives in code

We can approximate any derivative straight from the limit definition (★) by using a small but finite \(h\) — the finite-difference method. It’s slow and slightly inexact, but it’s invaluable for checking analytic gradients (a habit that has saved countless ML bugs).

code/track-2/derivatives.py
import numpy as np

def numerical_derivative(f, x, h=1e-5):
    """Approximate f'(x) using the centered difference (more accurate than (★) directly)."""
    return (f(x + h) - f(x - h)) / (2 * h)

f = lambda x: x**2            # f'(x) = 2x
print(numerical_derivative(f, 3.0))   # ~6.0   (analytic: 2*3 = 6)

def numerical_gradient(f, x, h=1e-5):
    """Gradient of a multivariable f at vector x, one partial derivative at a time."""
    x = np.asarray(x, float)
    grad = np.zeros_like(x)
    for i in range(x.size):
        step = np.zeros_like(x); step[i] = h
        grad[i] = (f(x + step) - f(x - step)) / (2 * h)   # vary x_i, hold others fixed
    return grad

# f(x, y) = x^2 + 2y^2  ->  grad = (2x, 4y)
g = lambda v: v[0]**2 + 2*v[1]**2
print(numerical_gradient(g, [3.0, 1.0]))   # ~[6. 4.]  matches (2*3, 4*1)

# -gradient is the downhill direction
x = np.array([3.0, 1.0])
print("downhill direction:", -numerical_gradient(g, x))   # [-6. -4.]  toward the minimum (0,0)
⚠️ Use the centered difference

The raw definition \(\frac{f(x+h)-f(x)}{h}\) has error proportional to \(h\); the centered version \(\frac{f(x+h)-f(x-h)}{2h}\) has error proportional to \(h^2\) — far more accurate for the same \(h\). And don’t make \(h\) too tiny (like \(10^{-12}\)): floating-point round-off then dominates and the estimate gets worse. Around \(10^{-5}\) is the sweet spot.

🔑 Key results to remember

Lock these in

  1. The derivative \(f'(x)\) is the slope of the tangent — the instantaneous rate of change, defined by the limit (★).
  2. Key rules: power \(\frac{d}{dx}x^n=nx^{n-1}\), sum, constant multiple, and the chain rule \( (f\circ g)' = f'(g)\,g'\).
  3. A partial derivative \(\frac{\partial f}{\partial x_i}\) is the slope along one axis, holding the rest fixed.
  4. The gradient \(\nabla f\) stacks all partials; it points uphill (steepest ascent), perpendicular to the contours; \(\lVert\nabla f\rVert\) is the steepness.
  5. \(-\nabla f\) is the steepest-descent direction — the arrow we follow to minimize a loss.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. At a point where \(f'(x) = 0\), what is happening?

  • \(f\) is increasing fastest there.
    Fastest increase happens where the slope is large and positive, not zero. Zero slope is the opposite of steep.
  • \(f\) equals zero there.
    The value \(f(x)\) and the slope \(f'(x)\) are unrelated; a function can be far from zero while momentarily flat.
  • The tangent is flat — a peak, valley, or plateau.
    Correct. Zero slope means a horizontal tangent: a local min, local max, or saddle/plateau. Minimization hunts for exactly these stationary points.
  • The function is undefined there.
    A zero derivative is perfectly well-defined; it just means the curve is momentarily level.

Q2. For \(f(x,y) = x^2 + 2y^2\), what is the gradient at \((3, 1)\)?

  • \((3, 1)\)
    That’s the point itself, not the gradient. You must differentiate: \(\partial f/\partial x = 2x\), \(\partial f/\partial y = 4y\).
  • \((6, 4)\)
    Correct. \(\partial f/\partial x = 2x = 6\) and \(\partial f/\partial y = 4y = 4\). Stack them: \(\nabla f = (6,4)\).
  • \((6, 2)\)
    The \(x\)-partial is right, but \(\partial/\partial y\) of \(2y^2\) is \(4y = 4\), not \(2\). The coefficient \(2\) and the power both contribute.
  • \((2, 4)\)
    These are the coefficients of the partials, not their values at \((3,1)\). Plug in: \(2x=6\), \(4y=4\).

Q3. To decrease \(f\) as fast as possible from a point, which direction do you step?

  • Along a contour line.
    Moving along a contour keeps \(f\) constant (zero change) — you wouldn’t decrease it at all.
  • In the direction of \(+\nabla f\).
    That’s steepest ascent — it increases \(f\) fastest, the wrong way for minimization.
  • Any direction with a negative slope works equally well.
    Many directions decrease \(f\), but not equally — the single fastest decrease is the steepest-descent direction specifically.
  • In the direction of \(-\nabla f\).
    Correct. The gradient is steepest ascent, so its negative is steepest descent — the fastest way down, and the basis of gradient descent.

Q4. Why is the gradient always perpendicular to the contour lines?

  • Because \(f\) doesn’t change along a contour, so the direction of maximum change has no component along it.
    Correct. Zero change along the contour forces the steepest-change direction to be entirely perpendicular to it.
  • Because contours are always circles.
    Contours are circles only for special functions; for \(x^2+2y^2\) they’re ellipses. The perpendicularity holds regardless of contour shape.
  • Because the gradient has unit length.
    The gradient’s length varies (it encodes steepness); it isn’t generally a unit vector, and length is unrelated to its direction being perpendicular.
  • It isn’t — that’s a coincidence of the picture.
    It’s a theorem, not a coincidence: it follows directly from \(f\) being constant along contours.

✍️ Practice

Problem 1 · compute

Find \(f'(x)\) for \(f(x) = 3x^2 - 4x + 7\), and the value of the slope at \(x=2\).

Show solution

Differentiate term by term using the power and constant-multiple rules: \(\frac{d}{dx}3x^2 = 6x\), \(\frac{d}{dx}(-4x) = -4\), \(\frac{d}{dx}7 = 0\). So \(f'(x) = 6x - 4\). At \(x=2\): \(f'(2) = 12 - 4 = 8\) — the curve is rising steeply there.

Problem 2 · chain rule

Use the chain rule to differentiate \(f(x) = (2x+1)^3\).

Show solution

Outside function \(u^3\) with inside \(u = 2x+1\). Derivative of outside: \(3u^2\); derivative of inside: \(2\). Multiply: \(f'(x) = 3(2x+1)^2 \cdot 2 = 6(2x+1)^2\). The chain rule is “outside′ × inside′,” and it’s exactly how a network passes gradients backward through its layers.

Problem 3 · gradient

For \(f(x,y) = x^2 y + 3y\), compute the gradient \(\nabla f\), then evaluate it at \((2, 1)\).

Show solution

Partial in \(x\) (treat \(y\) as constant): \(\frac{\partial f}{\partial x} = 2xy\). Partial in \(y\) (treat \(x\) as constant): \(\frac{\partial f}{\partial y} = x^2 + 3\). So \(\nabla f = (2xy,\ x^2+3)\). At \((2,1)\): \(\nabla f = (2\cdot2\cdot1,\ 4+3) = (4, 7)\). Stepping in \(-(4,7)\) would decrease \(f\) fastest from that point.

Problem 4 · code

Use numerical_gradient to check your hand answer to Problem 3 at \((2,1)\). Confirm it’s close to \((4, 7)\).

Show solution 
import numpy as np
def numerical_gradient(f, x, h=1e-5):
    x = np.asarray(x, float); g = np.zeros_like(x)
    for i in range(x.size):
        s = np.zeros_like(x); s[i] = h
        g[i] = (f(x+s) - f(x-s)) / (2*h)
    return g

f = lambda v: v[0]**2 * v[1] + 3*v[1]
print(numerical_gradient(f, [2.0, 1.0]))   # ~[4. 7.]

The output is approximately [4. 7.], matching the analytic gradient. This “gradient check” — comparing a hand-derived gradient against finite differences — is a standard sanity test you’ll run whenever you implement backprop yourself.
