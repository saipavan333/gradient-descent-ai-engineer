Convexity & why it matters

When is “roll downhill” guaranteed to find the best answer, not just a mediocre one? The answer is convexity — the single property that separates optimization problems we can fully trust from the wild, wrinkled landscapes of deep learning.

Track 2 · Lesson 2 Concept + derivation + code ~24 min Builds on: Lessons 2.0–2.1

💡 Concept: a bowl has only one bottom

Gradient descent only ever goes downhill, so where it ends up depends entirely on the shape of the surface. If the surface is a single smooth bowl, then no matter where you start, downhill leads to the one bottom — the global minimum. If instead the surface has multiple dips, downhill might dump you into a shallow one and strand you there. The property that guarantees the bowl shape is convexity.

The precise definition (chord above the graph)

“Bowl-shaped” has an exact meaning. A function \(f\) is convex if, for any two points on its graph, the straight line segment connecting them (the chord) lies on or above the graph. Formally, for all inputs \(x, y\) and every blend \(\lambda \in [0,1]\):

$$ f\big(\lambda x + (1-\lambda)y\big) \;\le\; \lambda f(x) + (1-\lambda) f(y). $$
(★)

Unpack it. The left side is the function’s value at a point between \(x\) and \(y\) (\(\lambda\) slides from \(y\) at \(\lambda{=}0\) to \(x\) at \(\lambda{=}1\)). The right side is the height of the chord at that same in-between point (the matching blend of the two endpoint heights). The “\(\le\)” says the curve never pokes above its own chords — the signature of a bowl.

🔑 The property that makes convexity priceless

For a convex function, every local minimum is a global minimum. There are no traps. So gradient descent (with a sensible learning rate) is guaranteed to converge to the best possible answer. This is why convex problems are considered “solved” in a way non-convex ones never are.

📐 A test you can compute: curvature ≥ 0

Checking (★) for all pairs is impractical. Luckily there’s a local test using the second derivative — the derivative of the derivative, \(f''(x)\), which measures curvature (how the slope itself is changing). A twice-differentiable function of one variable is convex exactly when

$$ f''(x) \ge 0 \quad\text{for all } x. $$

Intuitively, \(f'' \ge 0\) means the slope is non-decreasing — it only ever turns more upward as you move right, which is precisely what carves out a bowl. For \(f(x)=x^2\), \(f''=2>0\) everywhere, so it’s convex. For the wavy \(f(x)=x^4-4x^2\), \(f''=12x^2-8\) is negative near the origin, so it’s not convex — and indeed it has two separate dips.

🧮 The multivariable version (a bridge to Lesson 2.4)

In many variables, curvature is captured by the Hessian matrix \(H\) of all second partial derivatives, \(H_{ij} = \frac{\partial^2 f}{\partial x_i \partial x_j}\). The convexity test becomes “\(H\) is positive semidefinite” — meaning all its eigenvalues are \(\ge 0\) (Track 1!). Non-negative curvature in every direction = a bowl in every cross-section = convex. We’ll meet the Hessian properly in Lesson 2.4, where its eigenvalues also explain how fast gradient descent converges.

🤖 Why ML lives on both sides of this line

A surprising amount of classical machine learning is convex, and that’s a feature. Linear regression (squared-error loss) and logistic regression (cross-entropy loss) are both convex in their parameters, so training them has a unique best solution that gradient descent reliably finds — no random restarts, no luck required. Support vector machines are convex too. Neural networks are not: stacking nonlinear layers makes the loss a wrinkled, high-dimensional landscape riddled with local minima and saddle points. Remarkably, deep learning works anyway — in very high dimensions most local minima turn out to be nearly as good as the global one, and the bad traps are rare. Knowing whether your problem is convex tells you whether to trust the optimizer completely or to babysit it (multiple seeds, schedules, the momentum and adaptive methods of the next lesson).

🛠️ From scratch: testing convexity & its consequence

code/track-2/convexity.py
import numpy as np

def chord_test(f, a, b, n=50):
    """Check definition (★): does the chord stay above the curve between a and b?"""
    for lam in np.linspace(0, 1, n):
        mid = lam*a + (1-lam)*b
        if f(mid) > lam*f(a) + (1-lam)*f(b) + 1e-9:   # curve poked above the chord
            return False
    return True

convex     = lambda x: x**2
nonconvex  = lambda x: x**4 - 4*x**2
print(chord_test(convex,    -2, 3))   # True
print(chord_test(nonconvex, -2, 2))   # False -> dips below its chord

# Consequence: GD's destination on a non-convex function depends on the start
def gd_1d(grad, x0, lr=0.05, steps=200):
    x = x0
    for _ in range(steps): x = x - lr*grad(x)
    return x

grad_nc = lambda x: 4*x**3 - 8*x        # derivative of x^4 - 4x^2; minima at +-sqrt(2)
print(round(gd_1d(grad_nc,  0.1), 3))   # ->  1.414  (fell right)
print(round(gd_1d(grad_nc, -0.1), 3))   # -> -1.414  (fell left)  -- different minima!

# On a convex function, the destination is the SAME from anywhere
grad_c = lambda x: 2*x                   # x^2, min at 0
print(round(gd_1d(grad_c,  5.0), 3), round(gd_1d(grad_c, -5.0), 3))   # 0.0  0.0

The non-convex run lands in different minima depending on where it starts — the hallmark of a trap-filled surface. The convex run reaches the same global minimum from any start. That single difference is what “convex = trustworthy” means in code.

🔑 Key results to remember

Lock these in

  1. Convex = every chord lies on/above the graph: \(f(\lambda x+(1-\lambda)y) \le \lambda f(x)+(1-\lambda)f(y)\) — a single bowl.
  2. The payoff: for convex \(f\), every local minimum is global, so gradient descent finds the best answer.
  3. Test: 1D convex \(\iff f''(x)\ge 0\) everywhere; multivariable \(\iff\) Hessian positive semidefinite (eigenvalues \(\ge 0\)).
  4. Linear & logistic regression, SVMs are convex (trustworthy); neural networks are non-convex (but work in practice).

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. What is the practical superpower of a convex loss?

  • Gradient descent runs faster per step.
    Per-step cost is unrelated to convexity; it’s set by how you compute the gradient. Convexity is about where you end up.
  • It has no minimum at all.
    A convex loss like \(x^2\) certainly has a minimum. Convexity guarantees the minimum is unique-in-quality, not absent.
  • Any local minimum is also the global minimum.
    Correct. No traps exist, so wherever downhill leads is the best possible answer — the optimizer can be fully trusted.
  • It is always a straight line.
    Straight lines are convex, but so are \(x^2\), \(e^x\), and many curves. Convex ≠ linear.

Q2. A twice-differentiable \(f(x)\) has \(f''(x) = -2\) for some \(x\). What can you conclude?

  • \(f\) is not convex.
    Correct. Convexity requires \(f''\ge 0\) everywhere; a single point with negative curvature (here \(-2\)) breaks it.
  • \(f\) is convex.
    Negative second derivative anywhere means the curve bends downward there — a violation of convexity, not evidence for it.
  • \(f\) has no minimum.
    Non-convex functions can still have minima (often several). \(f''<0\) somewhere doesn’t remove minima; it allows multiple.
  • \(f\) is linear.
    A linear function has \(f''=0\). A value of \(-2\) means genuine downward curvature, not linearity.

Q3. Which is not convex in its parameters?

  • Linear regression with squared-error loss.
    This is the classic convex problem — a quadratic bowl in the weights.
  • Logistic regression with cross-entropy loss.
    Logistic regression’s loss is convex in its weights; it has a unique best fit.
  • Minimizing \(f(w) = w^2\).
    A parabola is the prototypical convex function.
  • A deep neural network’s training loss.
    Correct. Stacking nonlinear layers makes the loss non-convex — many local minima and saddle points — which is why deep learning needs careful optimizers and sometimes multiple seeds.

Q4. In the multivariable test, convexity corresponds to the Hessian having…

  • determinant equal to 1.
    The determinant being 1 is unrelated to convexity (it concerns volume scaling). Convexity is about the sign of curvature in every direction.
  • all eigenvalues \(\ge 0\) (positive semidefinite).
    Correct. Non-negative eigenvalues mean non-negative curvature along every direction — a bowl in every cross-section.
  • all entries positive.
    Positive entries is neither necessary nor sufficient; what matters is the eigenvalues (curvature), not the raw entries.
  • at least one negative eigenvalue.
    A negative eigenvalue means downward curvature in some direction — a saddle or hill — which violates convexity.

✍️ Practice

Problem 1 · test

Is \(f(x) = x^4\) convex? Use the second-derivative test.

Show solution

\(f'(x) = 4x^3\), \(f''(x) = 12x^2\). Since \(x^2 \ge 0\) for all real \(x\), we have \(f''(x) = 12x^2 \ge 0\) everywhere. So \(f(x)=x^4\) is convex. (It’s “flat” at the origin where \(f''=0\), but still convex — the test allows equality.)

Problem 2 · derivation

Prove from definition (★) that if \(f\) and \(g\) are both convex, then \(h(x) = f(x) + g(x)\) is convex. (This is why summing convex loss terms keeps a problem convex.)

Show solution

Take any \(x, y\) and \(\lambda\in[0,1]\). Apply (★) to each function and add the two inequalities:

$$ f(\lambda x+(1-\lambda)y) \le \lambda f(x)+(1-\lambda)f(y), $$

$$ g(\lambda x+(1-\lambda)y) \le \lambda g(x)+(1-\lambda)g(y). $$

Summing the left sides and the right sides (valid since both inequalities point the same way): \(h(\lambda x+(1-\lambda)y) \le \lambda h(x) + (1-\lambda)h(y)\). That’s exactly (★) for \(h\), so \(h\) is convex. \(\blacksquare\) Consequence: adding an \(L_2\) regularizer \(\lambda\lVert w\rVert^2\) (convex) to a convex loss keeps the whole objective convex — a fact you’ll use in Track 3.

Problem 3 · code

Use chord_test to check whether \(f(x) = |x|\) (absolute value) is convex on \([-3, 3]\). Is it differentiable everywhere?

Show solution 
import numpy as np
def chord_test(f, a, b, n=50):
    for lam in np.linspace(0, 1, n):
        mid = lam*a + (1-lam)*b
        if f(mid) > lam*f(a) + (1-lam)*f(b) + 1e-9: return False
    return True
print(chord_test(np.abs, -3, 3))   # True

It returns True: \(|x|\) is convex (its chords lie above the V-shaped graph). But it is not differentiable at \(x=0\) (the slope jumps from \(-1\) to \(+1\)). Convexity doesn’t require smoothness — which is why convex but non-smooth losses like the \(L_1\) penalty and the SVM hinge loss are still tractable, using subgradients.

Problem 4 · reasoning

If neural networks have non-convex losses full of local minima, why does training them with gradient descent work so well in practice?

Show solution

A few reasons, all active research but well-supported: (1) In very high dimensions, most critical points are saddle points, not bad local minima, and the local minima that exist tend to have similar, low loss — so “a local min” is usually “a good-enough answer.” (2) The noise in stochastic gradient descent helps escape saddles and shallow traps. (3) Over-parameterization (far more weights than data points) smooths the landscape and creates many equivalent good solutions. So while we lose the convex guarantee, the geometry of high-dimensional, over-parameterized networks is empirically forgiving. We don’t get a proof of optimality — we get something that reliably works, which is why honest evaluation (Track 9) matters so much for non-convex models.
