Newton’s method & second-order ideas

Gradients tell you which way is down. Curvature tells you how far to step. Using both gives the fastest-converging optimizer there is — and explains, by its very impracticality, why deep learning sticks with first-order methods like Adam.

Track 2 · Lesson 4 Concept + derivation + code ~28 min Builds on: Lessons 2.0–2.3, Track 1 eigen

💡 Concept: use the curvature, not just the slope

Gradient descent uses only first-order information — the slope. But a slope alone doesn’t tell you how far to step: a gentle valley and a sharp one can have the same gradient yet need very different step sizes. Second-order methods also use the curvature — the second derivative — to choose the step. The idea, due to Newton, is elegant: approximate your function near the current point by the parabola that matches its value, slope, and curvature, then jump straight to that parabola’s minimum.

In one dimension the recipe is short. The best local parabola is the second-order Taylor approximation, and its minimum sits at the current point minus slope-over-curvature:

$$ x_{t+1} = x_t - \frac{f'(x_t)}{f''(x_t)}. $$
(★)

Compare to gradient descent’s \(x - \alpha f'\): Newton has replaced the hand-tuned learning rate \(\alpha\) with \(1/f''\) — a step size chosen automatically by the curvature. Steep regions (large \(f''\)) get small steps; flat regions (small \(f''\)) get big ones. No learning rate to tune.

📐 Many dimensions: the Hessian

In \(n\) dimensions, curvature isn’t one number — it’s a whole matrix, the Hessian \(H\), holding every second partial derivative \(H_{ij} = \frac{\partial^2 f}{\partial x_i\partial x_j}\). It describes how the slope changes in every direction and every combination of directions. Newton’s step generalizes by replacing “divide by \(f''\)” with “multiply by the inverse Hessian”:

$$ \boldsymbol{\theta}_{t+1} = \boldsymbol{\theta}_t - H^{-1}\,\nabla f(\boldsymbol{\theta}_t). $$
(‡)

This has a remarkable property: on a quadratic function, Newton’s method reaches the exact minimum in a single step, from anywhere. (A quadratic is its own second-order Taylor approximation, so the fitted parabola is exact and its vertex is the true minimum.) More generally, near a minimum Newton converges quadratically — the number of correct digits roughly doubles each step, dramatically faster than gradient descent’s steady crawl. And because \(H^{-1}\) rescales every direction by its curvature, Newton is immune to ill-conditioning — the ravine that crippled plain GD is, to Newton, just another bowl.

🎯 Why ill-conditioning slows GD: the condition number

We can finally make “ill-conditioned” precise using Track 1’s eigenvalues. The Hessian’s eigenvalues are the curvatures along its principal axes (its eigenvectors). Their ratio is the condition number:

$$ \kappa = \frac{\lambda_{\max}}{\lambda_{\min}} \quad (\text{largest curvature} \,/\, \text{smallest curvature}). $$

When \(\kappa = 1\) the bowl is perfectly round and gradient descent flies to the center. As \(\kappa\) grows the bowl stretches into a ravine and GD’s progress degrades (its convergence rate depends on \(\frac{\kappa-1}{\kappa+1}\), which crawls toward \(1\) for large \(\kappa\)). Newton, by applying \(H^{-1}\), effectively reshapes any bowl back into a round one — which is exactly why it doesn’t care about \(\kappa\). This is the deepest reason momentum, RMSProp, and Adam exist: they’re cheap, approximate ways to buy some of Newton’s conditioning-robustness without paying for the full Hessian.

🤖 So why doesn’t deep learning use Newton?

If Newton converges so much faster, why does every large model train with first-order Adam instead? Cost. For \(n\) parameters the Hessian is an \(n\times n\) matrix: storing it takes \(O(n^2)\) memory and inverting it takes \(O(n^3)\) time. For a model with \(n = 10^9\) parameters, the Hessian alone would need \(10^{18}\) numbers — billions of gigabytes, utterly impossible. Even forming it is hopeless. So at deep-learning scale, second-order methods are off the table, and we use first-order optimizers whose cost is merely \(O(n)\). The compromise of choice, quasi-Newton methods like L-BFGS, approximate the inverse Hessian from a short history of gradients without ever forming it — excellent for smaller problems (thousands to millions of parameters), still too memory-hungry for the largest networks. Knowing this hierarchy tells you which optimizer to reach for at which scale.

🛠️ From scratch: Newton vs gradient descent

code/track-2/newton.py
import numpy as np

# 1D Newton: x <- x - f'/f''
def newton_1d(fp, fpp, x0, steps=8):
    x = x0
    for _ in range(steps):
        x = x - fp(x) / fpp(x)
    return x

# f(x) = x^2 - 2 has root structure; minimize g(x)=(x-3)^4 (min at 3)
gp  = lambda x: 4*(x-3)**3
gpp = lambda x: 12*(x-3)**2
# (quartic curvature vanishes at the min, so convergence is linear here, still fast)
print(round(newton_1d(gp, gpp, 0.0, 30), 4))   # ~3.0

# Multivariable Newton on a quadratic: ONE step reaches the minimum
A = np.array([[3.0, 1.0], [1.0, 2.0]])          # f = 0.5 x^T A x ; min at 0
grad = lambda x: A @ x
H    = A                                         # Hessian of a quadratic is constant
x = np.array([5.0, -4.0])
x_newton = x - np.linalg.solve(H, grad(x))       # solve H d = grad  (don't invert!)
print("Newton in ONE step:", np.round(x_newton, 6))   # [0. 0.] exactly

# Compare: gradient descent needs many steps and a tuned lr
x_gd = np.array([5.0, -4.0])
for _ in range(50):
    x_gd = x_gd - 0.2 * grad(x_gd)
print("GD after 50 steps:", np.round(x_gd, 4))   # close, but not exact

# Condition number of the Hessian (Track 1 eigenvalues) predicts GD's difficulty
eigs = np.linalg.eigvalsh(A)
print("eigenvalues:", np.round(eigs, 3), " condition number:", round(eigs.max()/eigs.min(), 2))
⚠️ Solve, don’t invert

Notice the code uses np.linalg.solve(H, grad) rather than np.linalg.inv(H) @ grad. Solving the linear system \(H\mathbf{d}=\nabla f\) for the step \(\mathbf{d}\) is faster and far more numerically stable than explicitly forming \(H^{-1}\). Forming an inverse just to multiply by it is a classic beginner inefficiency — and a real-world source of instability when \(H\) is near-singular.

🔑 Key results to remember

Lock these in

  1. Newton (1D): \(x \leftarrow x - f'/f''\) — curvature replaces the learning rate. (nD): \(\boldsymbol{\theta} \leftarrow \boldsymbol{\theta} - H^{-1}\nabla f\).
  2. On a quadratic, Newton reaches the minimum in one step; near a min it converges quadratically (digits double).
  3. The condition number \(\kappa = \lambda_{\max}/\lambda_{\min}\) of the Hessian sets how slowly GD converges; Newton is immune to it.
  4. Cost kills it at scale: the Hessian is \(O(n^2)\) memory, \(O(n^3)\) to invert — impossible for billion-parameter models, so we use first-order Adam.
  5. Quasi-Newton (L-BFGS) approximates \(H^{-1}\) cheaply from gradient history — great for medium-scale problems.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. In Newton’s 1D step \(x \leftarrow x - f'/f''\), what plays the role of the learning rate?

  • The gradient \(f'\).
    \(f'\) sets the direction (and is the analogue of the gradient in GD); it isn’t the step-size knob.
  • A fixed constant you tune.
    The whole point is that Newton has no learning rate to tune — the step size is computed, not chosen.
  • The reciprocal curvature \(1/f''\).
    Correct. Newton replaces the hand-set \(\alpha\) with \(1/f''\): big steps in flat regions (small \(f''\)), small steps in sharp ones (large \(f''\)).
  • The second derivative \(f''\) itself.
    It’s the reciprocal \(1/f''\) that scales the step. Large curvature should give a small step, so we divide by \(f''\).

Q2. How many steps does Newton’s method need to minimize a quadratic function?

  • One.
    Correct. A quadratic equals its own second-order Taylor approximation, so the fitted parabola is exact and Newton jumps straight to the true minimum in a single step.
  • About \(\kappa\) steps, where \(\kappa\) is the condition number.
    That’s the flavor of gradient descent’s dependence. Newton is condition-number-independent and finishes a quadratic in one step.
  • It never exactly converges.
    On a quadratic Newton is exact, not approximate — it lands on the minimum precisely.
  • The same number as gradient descent.
    Far fewer — GD may need many iterations and a tuned \(\alpha\); Newton needs one.

Q3. Why isn’t Newton’s method used to train large neural networks?

  • It only works on non-convex functions.
    If anything Newton is cleanest on convex functions; convexity isn’t the obstacle.
  • It can’t use gradients.
    Newton uses the gradient (in \(H^{-1}\nabla f\)) and the Hessian. Using gradients isn’t the problem.
  • It is less accurate than gradient descent.
    Newton is more accurate per step, not less. Accuracy isn’t the issue — cost is.
  • The Hessian is \(O(n^2)\) to store and \(O(n^3)\) to invert — impossible for billions of parameters.
    Correct. At deep-learning scale the Hessian can’t even be stored, let alone inverted, so we use \(O(n)\) first-order methods like Adam.

Q4. A loss has Hessian eigenvalues \(\{100, 1\}\) at the minimum. What does this tell you about gradient descent there?

  • GD will converge in one step.
    One-step convergence is Newton’s trait on quadratics, not GD’s. Unequal eigenvalues actually slow GD.
  • The problem is ill-conditioned (\(\kappa = 100\)), so GD will be slow and zig-zag.
    Correct. \(\kappa = \lambda_{\max}/\lambda_{\min} = 100/1 = 100\) — a stretched bowl. GD’s rate degrades with large \(\kappa\), producing the slow zig-zag of Lesson 2.3.
  • The loss is non-convex there.
    Both eigenvalues are positive, so the Hessian is positive-definite — the bowl is convex locally. It’s just stretched.
  • The learning rate doesn’t matter.
    It matters a lot: the steep direction (\(\lambda=100\)) caps the stable learning rate, which is precisely why the shallow direction (\(\lambda=1\)) crawls.

✍️ Practice

Problem 1 · by hand

Minimize \(f(x) = x^2 - 4x + 7\) with one Newton step from \(x_0 = 0\). Verify it lands exactly on the minimum.

Show solution

\(f'(x) = 2x - 4\), \(f''(x) = 2\). Newton step: \(x_1 = x_0 - f'(0)/f''(0) = 0 - (-4)/2 = 0 + 2 = 2\). The true minimum is where \(f'(x)=0\): \(2x-4=0 \Rightarrow x=2\). Newton nailed it in one step — as expected for a quadratic, whose Taylor parabola is exact.

Problem 2 · derivation

Derive the 1D Newton step (★) by minimizing the second-order Taylor approximation of \(f\) around \(x_t\).

Show solution

The second-order Taylor approximation is the parabola \(q(x) = f(x_t) + f'(x_t)(x-x_t) + \tfrac12 f''(x_t)(x-x_t)^2\). Minimize \(q\) by setting its derivative to zero:

$$ q'(x) = f'(x_t) + f''(x_t)(x - x_t) = 0 \;\Rightarrow\; x - x_t = -\frac{f'(x_t)}{f''(x_t)}. $$

So the minimizer is \(x = x_t - f'(x_t)/f''(x_t)\), which is exactly (★). \(\blacksquare\) The multivariable version (‡) comes from the same argument with the Hessian: minimizing \(f(\boldsymbol{\theta}_t) + \nabla f^\top \mathbf{d} + \tfrac12 \mathbf{d}^\top H \mathbf{d}\) over the step \(\mathbf{d}\) gives \(H\mathbf{d} = -\nabla f\), i.e. \(\mathbf{d} = -H^{-1}\nabla f\).

Problem 3 · code

Build the Hessian of \(f(x,y) = 2x^2 + 8y^2\), compute its condition number, and confirm Newton reaches \((0,0)\) in one step from \((3, 2)\).

Show solution 
import numpy as np
H = np.array([[4.0, 0.0], [0.0, 16.0]])     # second partials of 2x^2 + 8y^2
grad = lambda p: np.array([4*p[0], 16*p[1]])
eigs = np.linalg.eigvalsh(H)
print("condition number:", eigs.max()/eigs.min())   # 4.0
p = np.array([3.0, 2.0])
print("Newton step:", p - np.linalg.solve(H, grad(p)))   # [0. 0.]

The condition number is \(16/4 = 4\) (the bowl is 4× steeper in \(y\) than \(x\)), and one Newton step lands exactly on \((0,0)\). Gradient descent, by contrast, would need a careful \(\alpha\) and many steps to crawl in — the κ=4 stretch is what slows it.

Problem 4 · synthesis

Place GD, momentum, Adam, Newton, and L-BFGS on a spectrum from “cheap per step but many steps” to “expensive per step but few steps.” When would you pick each?

Show solution

From cheapest/most-steps to most-expensive/fewest-steps: GD → momentum → Adam → L-BFGS → Newton. GD, momentum, and Adam are all \(O(n)\) per step (first-order) and scale to billions of parameters — use Adam as the default for deep learning, SGD+momentum when generalization matters most (Lesson 2.3). L-BFGS approximates curvature from a gradient history — use it for smooth, medium-scale problems (thousands–millions of parameters) where you can afford a bit more per step for far fewer steps. Newton is \(O(n^3)\) per step and exact-ish — use it only for small problems (hundreds of parameters) or as a conceptual ideal. The right choice trades per-step cost against number of steps, governed by your parameter count \(n\) and how ill-conditioned the loss is. This judgment — matching the optimizer to the scale — is exactly the kind of decision a researcher makes routinely.
