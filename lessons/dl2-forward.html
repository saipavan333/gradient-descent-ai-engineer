Forward pass & activation functions

⏱ 23 min Lesson 4.2 ⬇ Runnable notebook
Why this existsThe forward pass is how a network turns inputs into predictions — matrix multiply, add bias, activate, repeat. Knowing the activations and shapes is the foundation for everything that trains.
By the end, you can
  • Compute a forward pass layer by layer
  • Track shapes through the network
  • Choose hidden (ReLU) and output activations
  • Build a network with nn.Sequential

Concept The forward pass: push data through, layer by layer

The forward pass is how a network turns an input into a prediction. You feed the input to the first layer, which computes a weighted sum, adds a bias, and applies an activation; its output becomes the next layer's input; and so on until the final layer emits the prediction. That's it — a forward pass is just this same three-step move (multiply, add, activate) repeated through the stack.

The shapes have to line up exactly as in Lesson 1.2. If the input is a length-3 vector and the first layer has 4 neurons, its weight matrix is $4\times 3$, producing a length-4 hidden vector; a final layer with 1 neuron has a $1\times 4$ matrix, producing the single output. In practice you push a whole batch of examples through at once — stack $n$ inputs into an $n\times 3$ matrix — and the same matrix multiplies handle them all in parallel, which is what makes GPUs so effective (Lesson 2.2's vectorization, at scale).

Concept The activation functions you'll actually use

The activation is the nonlinearity applied after each layer's weighted sum. A handful dominate practice, and they fall into two camps:

  • ReLU $= \max(0, z)$: the default for hidden layers. It's cheap (just a threshold), never saturates for positive inputs, and trains fast. Its quirk: neurons can "die" if they get stuck always outputting 0.
  • Sigmoid $= 1/(1+e^{-z})$: squashes to $(0,1)$. Great for a final probability (Lesson 3.3), but poor for hidden layers because it saturates — its slope flattens to nearly 0 for large $|z|$, starving gradients (the vanishing-gradient problem, Lesson 4.7).
  • tanh: like sigmoid but ranges $(-1,1)$ and is zero-centered; still saturates.
  • GELU: a smooth ReLU-like curve used in modern transformers (Track 5); often a touch better than ReLU in big models.

For the output layer the choice is dictated by the task, not preference: softmax for multi-class classification (probabilities summing to 1, Lesson 3.3), a single sigmoid for binary, and no activation (linear) for regression, where you want an unbounded number.

Worked example A two-layer network's forward pass, by hand then in PyTorch

We push one input through a network with a 4-neuron ReLU hidden layer and a 1-neuron linear output, computing every intermediate value explicitly:

import numpy as np
rng = np.random.default_rng(0)

x  = np.array([1.0, 2.0, -1.0])      # input, shape (3,)
W1 = rng.normal(0, 1, (4, 3)); b1 = np.zeros(4)   # hidden layer: 3 -> 4
W2 = rng.normal(0, 1, (1, 4)); b2 = np.zeros(1)   # output layer: 4 -> 1

h = np.maximum(0, W1 @ x + b1)       # hidden pre-activation, then ReLU
y = W2 @ h + b2                      # linear output (regression)
print("hidden (after ReLU):", h.round(3))
print("output:", y.round(3))
Outputhidden (after ReLU): [0. 0. 3.902 0. ] output: [-4.861]

Notice ReLU zeroed three of the four hidden neurons — their pre-activation sums were negative, so they output 0 and contributed nothing to this particular prediction. Only the third neuron "fired." Different inputs light up different neurons; that sparse, input-dependent pattern is part of what gives networks their flexibility. The whole forward pass was two matrix multiplies and one ReLU. In PyTorch you'd express the same architecture declaratively:

import torch.nn as nn

model = nn.Sequential(
    nn.Linear(3, 4),   # hidden layer: 3 -> 4
    nn.ReLU(),         # activation
    nn.Linear(4, 1),   # output layer: 4 -> 1 (no activation -> regression)
)
y = model(x_batch)     # forward pass for a whole batch at once

Calling model(x) is the forward pass — PyTorch runs exactly the multiply-add-activate chain above. (Keras: Sequential([Dense(4, "relu"), Dense(1)]).) The network doesn't know anything useful yet — its weights are random, so the output is meaningless. Making that output correct is what learning does, and that requires knowing how to adjust every weight — the job of backpropagation, next.

★ Key points to remember
  • The forward pass repeats multiply → add bias → activate through the layers to produce a prediction.
  • Shapes must chain: a layer from $m$ to $n$ units uses an $n\times m$ weight matrix; batches go through together.
  • ReLU ($\max(0,z)$) is the default hidden activation — cheap, no positive-side saturation.
  • Sigmoid/tanh saturate (flat slopes) and starve gradients, so they're avoided in hidden layers.
  • Output activation follows the task: softmax (multi-class), sigmoid (binary), linear (regression).

Practice Build the reasoning yourself

1A hidden layer maps 10 inputs to 64 neurons; the next maps 64 to 64; the output maps 64 to 3 classes. Give each layer's weight-matrix shape and the output activation.
Show solution & reasoning

Layer 1: $64 \times 10$. Layer 2: $64 \times 64$. Output layer: $3 \times 64$. (A layer from $m$ to $n$ units has an $n\times m$ matrix.) The output activation is softmax, because it's 3-class classification — you want three probabilities that sum to 1.

Total weights: $640 + 4096 + 192 = 4{,}928$ (plus $64+64+3=131$ biases). A batch of, say, 32 inputs enters as a $32\times 10$ matrix and exits as $32\times 3$ — every example processed in parallel by the same matrices.

2A colleague uses sigmoid activations in all 20 hidden layers of a deep network and finds the early layers barely train. Explain the likely cause and the fix.
Show solution & reasoning

The cause is saturation → vanishing gradients. Sigmoid's slope is near 0 when its input is even moderately large or small. During backprop (next lesson) the gradient for an early layer is a product of many such slopes; multiply twenty near-zero numbers and you get essentially zero, so the early layers receive almost no learning signal (Lesson 4.7 covers this in depth).

The fix: use ReLU (or GELU) in the hidden layers. ReLU's slope is exactly 1 for positive inputs, so it doesn't shrink the gradient as it propagates back — this single change was one of the key unlocks that made deep networks trainable. Reserve sigmoid for the output layer when you need a probability.

3Without any activation functions, the two-layer worked example would compute $W_2(W_1 x)$. Why is that a problem, and what single change fixes it?
Show solution & reasoning

$W_2(W_1 x) = (W_2 W_1)x = W_{\text{combined}}\,x$ — the two layers collapse into one linear map (Lesson 4.1). The "deep" network would be no more expressive than a single linear layer, unable to fit anything nonlinear.

The fix is the ReLU sitting between them: $W_2\,\text{ReLU}(W_1 x)$ cannot be rewritten as a single matrix multiply, because ReLU bends the signal. That one nonlinearity is precisely what makes the forward pass compute something a linear model can't — which is why the worked example applied np.maximum(0, ...) before the second layer.

★ Deep dive: dead ReLUs, leaky variants, and numerical care▸

ReLU's one weakness is the dying ReLU problem: if a neuron's weights drift so that its pre-activation is negative for every input, it always outputs 0, and since ReLU's gradient is also 0 there, it receives no signal to recover — the neuron is dead for good. Variants fix this by leaking a small slope for negative inputs: Leaky ReLU ($0.01z$ for $z<0$), PReLU (learns the leak), and ELU/GELU (smooth curves). In practice plain ReLU works fine for most networks with good initialization (Lesson 4.8), and GELU has become standard in transformers, but it's worth knowing the failure mode and the menu of fixes.

A subtlety you'll meet in code: softmax and sigmoid are usually folded into the loss function, not applied as a separate output layer. PyTorch's nn.CrossEntropyLoss expects raw scores ("logits") and applies softmax internally; nn.BCEWithLogitsLoss does the same with sigmoid. This isn't a quirk — doing the squash and the log together is numerically stabler (it avoids computing $\log$ of a number that underflowed to 0). So a classification network often ends with a bare nn.Linear and no visible final activation, with the softmax living inside the loss. Knowing this saves you from the common bug of applying softmax twice, which silently wrecks training.

A fresh network's forward pass gives nonsense, because its weights are random. Next, in 4.3 Backpropagation, step by step, we learn how the network measures its error and computes how to adjust every weight — the algorithm that makes learning possible.
