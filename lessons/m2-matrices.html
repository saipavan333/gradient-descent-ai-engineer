Track 1 · Math Foundations for ML

Matrices & matrix multiplication

⏱ 28 min Lesson 1.2 ▶ Open in Colab
Why this existsA single layer of a neural network, a batch of data, an image, a linear model — all of them are matrices, and the one operation they all run is matrix multiplication. Understand this and the machinery of ML stops being a black box.
By the end, you can
  • See a matrix as a stack of vectors AND as a transformation
  • Multiply a matrix by a vector and by another matrix, by hand
  • Connect matrix multiply to 'a layer of a network'
  • Vectorize it in NumPy and feel why it is fast

Concept From one vector to a whole grid

In Lesson 1.1 one thing (a house) became one vector (a list of numbers). But you never have just one house — you have a spreadsheet full of them. If you stack many vectors on top of each other, you get a grid of numbers, and a grid of numbers is called a matrix (plural: matrices). That's the whole idea: a matrix is just a rectangle of numbers arranged in rows and columns.

A few words we'll use constantly. A row runs left-to-right (one row = one house's feature vector). A column runs top-to-bottom (one column = one feature measured across all houses). Each individual number in the grid is an entry (or "element"). And the shape of a matrix is the pair of counts rows × columns — for example a grid with 3 rows and 4 columns has shape "3 by 4," written $3 \times 4$.

We'd write that example as $X \in \mathbb{R}^{3 \times 4}$ — read "X is in R, three by four," meaning "X is a grid of ordinary numbers with 3 rows and 4 columns." (We tend to use a capital letter like $X$ for a matrix and a small letter like $x$ for a single vector, so you can tell them apart at a glance.) Matrices are the containers that hold almost everything in ML: a dataset is a matrix, a black-and-white image is a matrix of pixel brightnesses, a batch of inputs fed to a model is a matrix, and the connection strengths inside a neural network are a matrix.

i
Two ways to read the same matrix

(1) As stored data — a stack of example vectors (the view in the picture above). (2) As a transformation — a machine that takes a vector in and pushes a (usually different) vector out. Both readings are correct, and the wonderful thing is that one operation — matrix multiplication — serves both. Let's build up to it slowly, because it's the most important operation in this whole track.

Concept Matrix × vector: the move at the center of ML

Here is the operation everything depends on. To multiply a matrix $A$ by a vector $x$, you do the following for each row of $A$: line that row up against $x$, multiply the matching entries (first-with-first, second-with-second, …), and add up all those products into a single number. Each row of $A$ produces one number, so the result is a new vector with one entry per row.

That "multiply matching entries and add them up" step is so common it has its own name, the dot product — but we don't need the name yet; we'll define it properly in Lesson 1.3. For now just trust the mechanic: row meets vector → one number.

Let's grind through the exact numbers from the picture, slowly. The matrix $A$ has 2 rows and 3 columns (shape $2 \times 3$), and the input vector is $x = (4, 1, 2)$:

$$ \text{row 1 of } A = (1, 0, 2): \quad (1)(4) + (0)(1) + (2)(2) = 4 + 0 + 4 = 8, $$ $$ \text{row 2 of } A = (0, 3, 1): \quad (0)(4) + (3)(1) + (1)(2) = 0 + 3 + 2 = 5. $$

Read row 1: one times four is four, zero times one is zero, two times two is four; add them, $4+0+4 = 8$. Row 2 the same way gives 5. So $Ax = (8, 5)$. Notice the result has 2 numbers, even though the input had 3 — a $2 \times 3$ matrix turned a 3-number vector into a 2-number vector. Matrices can change the dimension, which is exactly how a network layer reshapes data as it flows through.

★
The intuition: each row is a recipe

Think of each row of $A$ as a list of weights — a recipe for mixing the input's features. Row 1, $(1, 0, 2)$, says "take 1 part of feature 1, ignore feature 2, take 2 parts of feature 3." The output number is what that recipe produces. A model learns good recipes from data; matrix-vector multiplication is simply how it applies them. That's the entire trick.

Concept Matrix × matrix: many recipes at once

Multiplying two matrices is the same move, just repeated. The entry in row $i$, column $j$ of the result comes from pairing row $i$ of the left matrix with column $j$ of the right matrix (multiply matching entries, add them up — the same dot-product step).

There's one rule you must internalize, because mismatched shapes are the single most common error in all of practical machine learning — the shape rule:

$$ (m \times \underline{n}) \times (\underline{n} \times p) \;=\; (m \times p). $$

In words: write the two shapes side by side. The two inner numbers (the ones that touch in the middle — both $n$ here) must be equal; that's what makes "row meets column" line up, since the row's length must match the column's length. The two outer numbers ($m$ and $p$) become the shape of the answer. So a $2 \times 3$ times a $3 \times 2$ works (inner 3s match) and gives a $2 \times 2$; but a $2 \times 3$ times another $2 \times 3$ does not work at all (inner 3 and 2 disagree).

!
Order matters: $AB$ is usually not $BA$

With ordinary numbers $3 \times 5 = 5 \times 3$. With matrices this is not true — the order you multiply in changes the answer, and often $BA$ isn't even a legal shape. (The fancy word is "matrix multiplication is not commutative," where "commutative" just means "order doesn't matter.") Think of it like getting dressed: socks-then-shoes is not the same as shoes-then-socks. Always keep your order and your shapes straight.

Concept The payoff: this is a neural-network layer

Remember linear combinations from Lesson 1.1 (scale some vectors and add them)? A matrix-vector multiply is a batch of linear combinations. That's why a single layer of a neural network — one processing step — is, at its core, exactly one matrix multiply plus a vector add:

Let's name the parts of $y = f(Wx + b)$, since these exact words return in Track 4:

  • $x$ is the input vector; $y$ is the output vector.
  • $W$ (a matrix) holds the learned weights — the recipes from before. Multiplying by $W$ mixes the inputs.
  • $b$ (a vector) is the bias — a fixed amount added on, which lets the layer shift its output up or down (like the "+ c" that lets a line not pass through zero).
  • $f$ is an activation function — a simple function applied to each number that bends the result, adding nonlinearity ("nonlinear" = "not a straight line"). Without it, stacking layers would just collapse into one big straight-line map and gain nothing (we prove this in the deep dive).

So, three lessons in, you already understand the skeleton of deep learning. Everything that follows is about finding good numbers for $W$ and $b$ — which is what gradients (later in this track) and training (Track 4) are for.

Worked example Pricing a whole batch of houses in one line

Let's price three houses at once. We put the houses in a matrix $X$ (one house per row; features = size, bedrooms, age) and apply a single set of weights $w$. One matrix-vector multiply scores the entire batch — no loop needed.

Three predictions from one expression — that's the power of vectorized matrix math. Let's trace the first house by hand to prove there's no magic: $1800(0.18) + 3(15) + 12(-1.2) = 324 + 45 - 14.4 = 354.6$. The weights here are just guesses (a real model would learn them), but the machinery is already exactly what a trained linear model or neural layer uses. Add a bias and an activation and you have the full layer from the diagram:

b = 20.0                       # one bias number added to every prediction
W = np.array([[0.18, 15, -1.2],
              [0.05,  8, -0.3]])   # TWO recipes (rows) -> two output numbers per house
out = X @ W.T + b              # (3x3) @ (3x2) + bias -> shape (3x2)
print(out.shape)               # (3, 2): three houses, two outputs each
Output(3, 2)

(W.T is the transpose — it flips a matrix's rows and columns so the shapes line up for the multiply; just a bookkeeping step.) In PyTorch the operator is the same, and this exact pattern is what torch.nn.Linear runs under the hood:

import torch
X = torch.tensor([[1800., 3, 12], [950., 2, 40], [2400., 4, 5]])
W = torch.randn(2, 3)          # 2 output features, 3 input features
b = torch.randn(2)
y = X @ W.T + b                # a linear layer, written by hand
# equivalently:  torch.nn.Linear(3, 2)(X)

TensorFlow: tf.matmul(X, tf.transpose(W)) + b, or just tf.keras.layers.Dense(2)(X). Different spellings, identical idea.

★ Key points to remember
  • A matrix is a grid of numbers with a shape (rows × columns). Everyday reading: rows = examples, columns = features.
  • Matrix × vector: each output entry is one row "mixed with" the input — multiply matching entries, then add. It can change the dimension.
  • Matrix × matrix: entry $(i,j)$ pairs row $i$ of the left with column $j$ of the right.
  • Shape rule: $(m\times n)(n\times p) = (m\times p)$ — the inner numbers must match; this catches most bugs.
  • Order matters: $AB \neq BA$ (matrix multiply is not commutative).
  • A neural layer is $y = f(Wx + b)$: weights $W$ mix, bias $b$ shifts, activation $f$ bends. You already understand its core.

Practice Build the reasoning yourself

1Compute $\begin{bmatrix}1 & 2\\ 3 & 0\end{bmatrix}\begin{bmatrix}2\\ 1\end{bmatrix}$ by hand, showing the work for each row.
Show solution & reasoning

Row 1 meets the column: $1(2) + 2(1) = 2 + 2 = 4$. Row 2 meets the column: $3(2) + 0(1) = 6 + 0 = 6$.

Result: $(4, 6)$. The habit to lock in: one row at a time, multiply position-by-position, sum, write one number. A $2\times2$ times a $2\times1$ gives a $2\times1$ — outer dimensions, exactly as the shape rule predicts.

2You have a batch of 64 images, each flattened to a 784-number vector, stored as a matrix $X$ of shape $64 \times 784$. A layer should turn each image into 10 scores (one per digit 0–9). What shape must the weight matrix $W$ be so that $X\,W$ is valid, and what's the output shape?
Show solution & reasoning

For $X\,W$ with $X$ being $64 \times 784$, the inner numbers must match, so $W$ needs 784 rows. We want 10 outputs per image, so $W$ has 10 columns: $W$ is $784 \times 10$.

Then $(64 \times 784)(784 \times 10) = 64 \times 10$: 64 images, 10 scores each. That's exactly the final layer of a handwritten-digit classifier. Reasoning the shapes out before writing code is the single best habit for avoiding the errors that trip up every beginner (and plenty of experts).

3In NumPy, A @ B raises a ValueError about "mismatched" dimensions, where A.shape == (3, 4) and B.shape == (3, 4). Explain the error and give one fix that makes a legal multiply.
Show solution & reasoning

Write the shapes side by side: $(3 \times \underline{4})(\underline{3} \times 4)$. The inner numbers (the left's column count, 4, and the right's row count, 3) must be equal — but they're 4 and 3, so it's illegal.

One fix: transpose $B$ so it becomes $4 \times 3$. Then $(3\times4)(4\times3) = 3\times3$ is valid: A @ B.T. (Whether that's the meaningful thing to compute depends on your intent — but it's now shape-legal.) The lesson: read shape errors literally; they're almost always telling you the inner dimensions disagree.

★ Deep dive: a matrix as a transformation you can see — and why depth needs $f$▸

The "rows are examples" view is great for data, but the deeper view is that a matrix is a function that moves space. If you multiply every point on a flat sheet by a fixed $2\times2$ matrix, the whole sheet rotates, stretches, or shears — but grid lines stay straight and evenly spaced and the origin stays put. That "straight lines stay straight" property is what the word linear really means.

A few worth recognizing on sight:

MatrixWhat it does to the plane
$\begin{bmatrix}1&0\\0&1\end{bmatrix}$ (the identity)nothing — every vector maps to itself. It's the "1" of matrix multiply.
$\begin{bmatrix}2&0\\0&2\end{bmatrix}$scales everything by 2 (zoom in).
$\begin{bmatrix}0&-1\\1&0\end{bmatrix}$rotates the plane 90°.

Why depth needs the nonlinear $f$. Here's the proof promised in the lesson. Suppose you stack two plain (no-$f$) layers: first $y = W_1 x$, then $z = W_2 y$. Substitute: $z = W_2(W_1 x) = (W_2 W_1)x$. But $W_2 W_1$ is itself just one matrix — call it $W_3$ — so the two layers together do exactly what the single layer $z = W_3 x$ does. No matter how many plain layers you stack, they always collapse to one. The nonlinear activation $f$ breaks this collapse: $f(W_2\,f(W_1 x))$ cannot be folded into a single matrix, so each extra layer genuinely adds expressive power — the ability to bend, curve, and carve up space. That single fact is why deep networks are "deep" and why activation functions are non-negotiable.

Why hardware loves matrices. A matrix multiply is a huge pile of identical multiply-and-add operations with no dependencies between them, so they can all run at the same time. That is precisely what a GPU (a chip with thousands of small parallel cores) is built for, and it's the reason modern AI — which is billions of matrix multiplies — became practical. When people say a model "has 175 billion parameters," they mean the count of numbers sitting inside its weight matrices.
