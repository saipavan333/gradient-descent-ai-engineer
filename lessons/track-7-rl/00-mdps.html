MDPs & the reinforcement-learning problem

Every model so far learned from a fixed dataset. Reinforcement learning is different: an agent acts, the world responds, and it must learn — from reward alone — how to behave. First we need the language to state that problem precisely.

Track 7 · Lesson 0 Concept ~22 min Builds on: Track 1 (probability)

Supervised learning is told the right answer. Reinforcement learning is only told how good the outcome was, often much later, and must discover good behaviour by trial and error. This is how we train agents to play games, control robots, and — as you will see in Lesson 4 — align language models. It all starts with one formal object: the Markov decision process.

💡 Concept: agent, environment, reward

Picture an agent interacting with an environment in a loop of discrete timesteps. At step \(t\) the agent observes a state \(s_t\), chooses an action \(a_t\), and the environment responds with a new state \(s_{t+1}\) and a scalar reward \(r_{t+1}\). The agent’s only goal is to collect as much reward as possible over time. It is never told which action was “correct” — only how much reward followed.

📐 The Markov decision process

We formalise the environment as a Markov decision process (MDP), the tuple \((\mathcal{S}, \mathcal{A}, P, R, \gamma)\):

  • \(\mathcal{S}\) — the set of states; \(\mathcal{A}\) — the set of actions.
  • \(P(s' \mid s, a)\) — the transition probability of landing in \(s'\) after taking \(a\) in \(s\).
  • \(R(s, a, s')\) — the reward for that transition.
  • \(\gamma \in [0,1)\) — the discount factor: how much future reward is worth now.

The word Markov means the future depends only on the present state, not the full history: once you know \(s_t\), how you got there is irrelevant to what happens next. This assumption is what makes the problem tractable — the state is a sufficient summary of the past.

The agent’s behaviour is a policy \(\pi(a \mid s)\): a (possibly random) rule for choosing actions given the state. What it tries to maximise is the return — the discounted sum of future rewards from time \(t\):

\[ G_t = r_{t+1} + \gamma\, r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k\, r_{t+k+1}. \] (0.1)

The discount \(\gamma\) does two jobs: it keeps the sum finite, and it encodes how far-sighted the agent is. With \(\gamma\) near \(0\) it grabs immediate reward; near \(1\) it plans far ahead. A reward of \(1\) received \(k\) steps from now is worth only \(\gamma^k\) today — the reinforcement-learning version of “a dollar tomorrow is worth less than a dollar today.”

💡 Value functions: grading states and actions

Because the return is random (the environment and policy both involve chance), we work with its expectation. The state-value function of a policy \(\pi\) is the expected return from a state if you follow \(\pi\) thereafter, and the action-value function (or “\(Q\)-function”) conditions also on the first action:

\[ V^\pi(s) = \mathbb{E}_\pi\!\left[G_t \mid s_t = s\right], \qquad Q^\pi(s,a) = \mathbb{E}_\pi\!\left[G_t \mid s_t = s,\, a_t = a\right]. \] (0.2)

These are the central objects of the next two lessons. \(V^\pi\) tells you how good it is to be in a state; \(Q^\pi\) tells you how good it is to take an action there. Solving an MDP means finding an optimal policy \(\pi^\star\) whose value is at least as high as any other policy’s, in every state. Everything in reinforcement learning is a strategy for estimating these values, or the policy directly, from experience.

Why this is harder than supervised learning. Two new difficulties appear here that Tracks 1–6 never faced. Delayed reward: the action that mattered may have happened long before the reward arrived (the credit-assignment problem). Exploration: the agent must try uncertain actions to discover good ones, trading off against exploiting what it already knows. Keep both in mind — every algorithm in this track is, at heart, a way to cope with them.

🔑 Key results to remember

  • RL = learning from reward by interaction, not from labelled examples. The signal is evaluative (“how good”), not instructive (“the answer”).
  • MDP \((\mathcal{S},\mathcal{A},P,R,\gamma)\); the Markov property says the state summarises all the past you need.
  • Return \(G_t=\sum_k \gamma^k r_{t+k+1}\); the discount \(\gamma\) bounds the sum and sets the planning horizon.
  • Policy \(\pi(a\mid s)\) is what we learn; value functions \(V^\pi,Q^\pi\) grade states and actions by expected return.
  • Two core challenges: delayed reward (credit assignment) and the exploration–exploitation trade-off.

🧩 Check your understanding
Answer from memory, then read why.

Q1. What does the “Markov” property in an MDP assert?

  • Rewards are always positive.
    Rewards can be any sign. Markov is about state, not reward magnitude.
  • The next state and reward depend only on the current state and action, not the full history.
    Correct. The current state is a sufficient statistic of the past, which is what makes value functions well-defined and the problem solvable.
  • The policy must be deterministic.
    Policies may be stochastic; the Markov property concerns the environment’s dynamics, not the policy’s form.
  • The discount factor must equal 1.
    \(\gamma\) is typically \(<1\). It is unrelated to the Markov assumption.

Q2. Why is a discount factor \(\gamma < 1\) used in the return?

  • To make all rewards equal.
    It does the opposite — it weights near rewards more than far ones.
  • To force the agent to ignore the future entirely.
    Only \(\gamma=0\) does that. A typical \(\gamma\) like 0.9 still values the future, just less than the present.
  • To keep the infinite sum of rewards finite and set how far-sighted the agent is.
    Correct. Geometric weighting bounds the return for ongoing tasks and tunes the planning horizon: higher \(\gamma\) means more long-term planning.
  • To guarantee the policy is optimal.
    Optimality comes from the learning algorithm; \(\gamma\) only defines the objective being optimised.

Q3. The difference between \(V^\pi(s)\) and \(Q^\pi(s,a)\) is that…

  • \(Q^\pi\) also fixes the first action taken, while \(V^\pi\) averages over the policy’s action choice.
    Correct. \(V^\pi(s)=\sum_a \pi(a\mid s)Q^\pi(s,a)\): the state value is the policy-weighted average of action values.
  • \(V^\pi\) is for deterministic policies only.
    Both are defined for stochastic and deterministic policies alike.
  • \(Q^\pi\) ignores future rewards.
    Both are expected returns, summing discounted future rewards; \(Q\) just conditions on the first action.
  • They are always equal.
    They coincide only when the action is forced; in general they differ because \(Q\) pins the first action.

Q4. The “credit-assignment problem” in RL refers to…

  • deciding how much compute to spend.
    That is a resource question, not credit assignment.
  • normalising rewards to zero mean.
    A useful trick (you’ll meet baselines later), but not what credit assignment means.
  • choosing the learning rate.
    Unrelated; credit assignment is about which actions deserve credit, not optimiser settings.
  • figuring out which earlier actions deserve credit (or blame) for a reward that arrives much later.
    Correct. Because reward is delayed, the agent must trace a late reward back to the decisions that caused it — the heart of what makes RL hard.

✍️ Practice

1. An agent gets reward \(+1\) at every step forever. With \(\gamma=0.9\), what is the return \(G_t\)? With \(\gamma=0.99\)?

Show solution

It is a geometric series: \(G_t=\sum_{k=0}^\infty \gamma^k = \frac{1}{1-\gamma}\). For \(\gamma=0.9\), \(G_t=\frac{1}{0.1}=10\); for \(\gamma=0.99\), \(G_t=\frac{1}{0.01}=100\). This shows concretely how \(\gamma\) sets the effective horizon: \(\frac{1}{1-\gamma}\) is roughly “how many steps ahead the agent cares about” — 10 steps vs 100 steps here.

2. Show that \(V^\pi(s)=\sum_a \pi(a\mid s)\,Q^\pi(s,a)\).

Show solution

By definition \(V^\pi(s)=\mathbb{E}_\pi[G_t\mid s_t=s]\). Condition on the first action using the law of total expectation: \(\mathbb{E}[G_t\mid s]=\sum_a \Pr(a_t=a\mid s_t=s)\,\mathbb{E}[G_t\mid s_t=s,a_t=a]\). The first factor is \(\pi(a\mid s)\) and the second is \(Q^\pi(s,a)\), giving \(V^\pi(s)=\sum_a \pi(a\mid s)Q^\pi(s,a)\). In words: a state’s value is the average of its action-values, weighted by how often the policy picks each action.

3. Conceptual: give a real task and write it as an MDP — name \(\mathcal{S},\mathcal{A},R\). Where might the Markov assumption be violated?

Show solution

Example — a thermostat: \(\mathcal{S}\) = current temperature (and maybe time of day), \(\mathcal{A}\) = {heat, cool, off}, \(R\) = comfort minus energy cost. The Markov assumption is violated if the next temperature depends on history not in the state — e.g. thermal mass means the recent trend matters, not just the instantaneous temperature. The usual fix is to enrich the state (include recent temperatures or their rate of change) until it again summarises everything relevant — a recurring practical theme in applying RL.
