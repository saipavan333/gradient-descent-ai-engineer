Value iteration & Q-learning

Two ways to solve an MDP. If you know the rules, dynamic programming computes the optimal values exactly. If you don’t, Q-learning discovers them from raw experience — the first real learning agent you’ll build.

Track 7 · Lesson 1 Concept + code ~28 min Builds on: Lesson 0

Lesson 0 defined value functions; now we compute them. We start with value iteration, which needs a full model of the environment and finds the optimum by repeated one-step lookahead. Then we drop the model entirely and build Q-learning, which learns the same optimal behaviour purely by trial and error — and watch it solve a GridWorld it was never given the rules to.

📐 The Bellman optimality equation

The key fact that makes MDPs solvable is that value functions are self-referential: the value of a state is the immediate reward plus the discounted value of where you land next. For the optimal value \(V^\star\), you act greedily — take the action with the best expected outcome — which gives the Bellman optimality equation:

\[ V^\star(s) = \max_{a}\ \sum_{s'} P(s'\mid s,a)\,\Big[\,R(s,a,s') + \gamma\,V^\star(s')\,\Big]. \] (1.1)

Value iteration turns Equation (1.1) into an algorithm: start with \(V\) all zeros and apply the right-hand side as an update over every state, again and again. Each sweep is a Bellman backup. The update is a contraction (each sweep shrinks the error by a factor \(\gamma\)), so \(V\) is guaranteed to converge to \(V^\star\); reading off the greedy action in each state then gives the optimal policy \(\pi^\star\).

🛠️ From scratch: value iteration on a GridWorld

Our gridworld.py defines a 4×4 grid with a goal (\(+1\)), a pit (\(-1\)), a wall, and “slippery” moves (you go where you intend with probability 0.8, else slip sideways). The backup is Equation (1.1) line-for-line: 

def value_iteration(theta=1e-6):
    V = {s: 0.0 for s in states}
    while True:
        delta = 0.0
        for s in V:
            if is_terminal(s): V[s] = 0.0; continue
            best = max(                                   # max over actions
                sum(p * (reward(ns) + GAMMA * (0 if is_terminal(ns) else V[ns]))
                    for p, ns in transitions(s, a))       # expectation over s'
                for a in A_LIST)
            delta = max(delta, abs(best - V[s])); V[s] = best
        if delta < theta: break                          # converged
    return V

💡 Q-learning: solving the MDP without the rules

Value iteration needed \(P\) and \(R\). Real agents rarely have them — a robot doesn’t know the physics of every contact. Q-learning learns the optimal action-value \(Q^\star\) from sampled experience alone, using the temporal-difference (TD) update. After taking \(a\) in \(s\) and observing reward \(r\) and next state \(s'\):

\[ Q(s,a) \;\leftarrow\; Q(s,a) + \alpha\,\Big[\underbrace{r + \gamma \max_{a'} Q(s',a')}_{\text{TD target}} - Q(s,a)\Big]. \] (1.2)

The bracket is the TD error: the gap between our current estimate \(Q(s,a)\) and a better, one-step-bootstrapped target built from real reward \(r\) plus the best next value. We nudge \(Q\) toward that target by a learning rate \(\alpha\). Crucially the target uses \(\max_{a'}\) — the best possible next action — so Q-learning learns the optimal policy even while behaving exploratorily (it is off-policy). To explore, the agent acts \(\varepsilon\)-greedy: usually greedy, but with probability \(\varepsilon\) it picks a random action to keep discovering. 

a = random_action() if rng.random() < eps else argmax_a Q[(s, a)]   # epsilon-greedy
ns, r = step(s, a)                                                   # sample the world
best_next = 0 if is_terminal(ns) else max(Q[(ns, a2)] for a2 in A_LIST)
Q[(s, a)] += alpha * (r + GAMMA * best_next - Q[(s, a)])             # TD update (1.2)

🔑 Key results to remember

  • Bellman optimality: \(V^\star(s)=\max_a \sum_{s'}P(s'\mid s,a)[R+\gamma V^\star(s')]\) — value is reward plus discounted next value.
  • Value iteration repeats the backup to convergence; it is a \(\gamma\)-contraction, so it always converges to \(V^\star\). Needs the model \((P,R)\).
  • Q-learning is model-free: \(Q\leftarrow Q+\alpha[r+\gamma\max_{a'}Q(s',a')-Q]\), learned from samples.
  • TD error = target − estimate; the \(\max\) makes Q-learning off-policy (learns optimal while exploring).
  • \(\varepsilon\)-greedy balances exploration and exploitation; decaying \(\varepsilon\) converges to greedy as \(Q\) sharpens.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. The single biggest difference between value iteration and Q-learning is that Q-learning…

  • always converges faster.
    Not necessarily — using a known model, value iteration can be much faster. Speed isn’t the defining difference.
  • does not require knowing the transition and reward model.
    Correct. Q-learning learns from sampled \((s,a,r,s')\) experience, so it works when \(P\) and \(R\) are unknown — the usual real-world case.
  • cannot find the optimal policy.
    It can — given enough exploration and decaying step sizes, Q-learning converges to \(Q^\star\).
  • only works without discounting.
    Q-learning uses the same discount \(\gamma\); it is unrelated to whether discounting is present.

Q2. In the Q-learning update, what is the “TD error”?

  • The difference between two consecutive states.
    It is a difference between value estimates, not states.
  • The total reward of the episode.
    That is the Monte-Carlo return; TD uses a one-step bootstrap, not the full episode.
  • The learning rate times the reward.
    The learning rate multiplies the TD error; it is not the error itself.
  • The gap \(r+\gamma\max_{a'}Q(s',a') - Q(s,a)\) between the bootstrapped target and the current estimate.
    Correct. We move \(Q(s,a)\) a fraction \(\alpha\) toward this better target; when the error is zero everywhere, \(Q\) satisfies the Bellman equation.

Q3. Q-learning is called “off-policy” because…

  • its target uses \(\max_{a'}Q(s',a')\) — the optimal next action — regardless of the (exploratory) action it actually takes.
    Correct. The behaviour policy explores (\(\varepsilon\)-greedy), but the learning target evaluates the greedy/optimal policy, so the two policies differ — hence “off-policy.”
  • it never explores.
    It does explore via \(\varepsilon\)-greedy; that exploration is exactly why “off-policy” matters.
  • it discards old experience.
    Off-policy actually enables reusing old experience (e.g. replay buffers); it is about the target, not memory.
  • it requires a model of the environment.
    It is model-free. “Off-policy” describes the relationship between behaviour and target policies.

Q4. If you set \(\varepsilon=0\) (pure greedy) from the very start of Q-learning, what is the risk?

  • The learning rate becomes unstable.
    \(\varepsilon\) and \(\alpha\) are independent; greedy action selection doesn’t destabilise the step size.
  • The discount factor is ignored.
    \(\gamma\) still applies in the target regardless of \(\varepsilon\).
  • The agent may lock onto a mediocre action and never discover better ones, because it never explores.
    Correct. With no exploration, early lucky-but-suboptimal estimates get reinforced and untried actions stay unknown. Some exploration (\(\varepsilon>0\), often decaying) is needed to guarantee finding the optimum.
  • Q-values can never be negative.
    Q-values can be any sign; this has nothing to do with \(\varepsilon\).

✍️ Practice

1. Run both files. Confirm Q-learning’s greedy policy matches value iteration’s along the path from the start. Then raise the step cost (make each move more negative). What changes?

Show solution

Both should agree on the route from the start toward the goal, skirting the pit. Increasing the step penalty makes the agent more impatient: it prefers the shortest path even if that path passes closer to the pit, and state values everywhere drop (each step now costs more). If you make the step cost large enough, the optimal policy can change qualitatively — taking a riskier shortcut becomes worth it. This is reward shaping in action: the reward function is the specification of the task, and small changes can flip the optimal behaviour.

2. Why does value iteration use \(\max_a\) inside the sum but Q-learning uses \(\max_{a'}\) on the next state? Relate the two.

Show solution

They are the same Bellman optimality principle applied at different objects. Value iteration computes \(V^\star(s)\), so it maximises over the action taken now. Q-learning estimates \(Q^\star(s,a)\), where the current action \(a\) is already fixed (it is an argument of \(Q\)); the optimality “max” therefore moves to the next state, \(\max_{a'}Q(s',a')\), which is exactly \(V^\star(s')\). So Q-learning’s target \(r+\gamma\max_{a'}Q(s',a')\) is the sampled version of the Bellman backup, with the model expectation replaced by a single sampled \((r,s')\).

3. The GridWorld has ~14 states. Real problems (a camera image as state) have astronomically many. Why does the tabular \(Q\) here not scale, and what replaces the table?

Show solution

A table stores one number per \((s,a)\) pair, which is impossible when states are images (more states than atoms in the universe) — and you would never revisit the exact same state to learn its value. The fix is function approximation: replace the table \(Q(s,a)\) with a neural network \(Q_\theta(s,a)\) that generalises across similar states (Track 4). That is exactly “Deep Q-Networks” (DQN), which combined Q-learning with a CNN to play Atari from pixels. The TD update becomes a gradient step on \(\big(r+\gamma\max_{a'}Q_\theta(s',a')-Q_\theta(s,a)\big)^2\).
