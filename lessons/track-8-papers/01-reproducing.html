Reproducing results faithfully

Reading tells you what a paper claims; reproducing tells you whether it’s true — and teaches you more than any amount of reading. We’ll reproduce one of the most surprising results in modern ML, from scratch, and watch it appear on our own screen.

Track 8 · Lesson 1 Method + code ~28 min Builds on: Lesson 0, Track 3

A Pass-3 read is a reproduction. Rebuilding a result is how you find the undocumented detail, catch your own misunderstanding, and earn the right to believe — or doubt — a claim. We’ll reproduce double descent, a phenomenon that overturned the textbook bias–variance story, using nothing but the tools from Track 3.

💡 The reproduction workflow

Reproduction is a discipline, not a guess-and-check. Isolate one concrete claim, find the exact method and settings, build the smallest version that could show the effect, compare to the reported result, and — the step amateurs skip — reconcile every gap. A number that is “close” but unexplained usually hides a bug or a misunderstanding.

📐 The claim: test error is not U-shaped

Track 3 taught the classic bias–variance story: as a model grows more complex, test error falls, bottoms out, then rises as the model overfits — a U-shape. Double descent (Belkin et al., 2019; Nakkiran et al., 2019) says that story is incomplete. Keep growing the model past the interpolation threshold — the point where it has just enough parameters to fit the training data exactly (parameters \(=\) data points) — and test error, after spiking at that threshold, falls again. Bigger-than-necessary models can generalize better, not worse.

The mechanism, in one line: right at the threshold there is exactly one way to fit the data, and it is wildly contorted (huge variance). With more parameters there are many interpolating solutions, and the least-squares fit picks the smoothest (minimum-norm) one — which generalizes well. We can show this with random features and linear regression, no deep network required.

🛠️ From scratch: reproducing double descent

Our reproduce_double_descent.py fits a linear model on \(P\) random ReLU features of the input, sweeping \(P\) across the interpolation threshold \(P=n\) (here \(n=40\) training points). For each \(P\) we solve least squares and measure test error on clean data: 

def fit_predict(Xtr, ytr, Xte, yte, P, d, ridge=1e-6):
    W = rng.normal(size=(d, P)) / np.sqrt(d); b = rng.normal(size=P)
    Ftr = relu(Xtr @ W + b); Fte = relu(Xte @ W + b)     # P random features
    A = Ftr.T @ Ftr + ridge * np.eye(P)                  # tiny ridge = min-norm
    w = np.linalg.solve(A, Ftr.T @ ytr)                  # least-squares fit
    return np.mean((Fte @ w - yte) ** 2)                 # test MSE

for P in [2, 8, 20, 40, 52, 80, 160, 360, 520]:          # sweep across P = n = 40
    err = mean(fit_predict(..., P, ...) for _ in range(25))
Why this reproduction is convincing. It is minimal (random features + least squares), it uses only ideas you already verified in Track 3, and the effect is dramatic and in the predicted place (the spike sits exactly at \(P/n=1\)). A faithful reproduction isn’t just “I saw a similar trend” — it’s “the effect appears, at the predicted location, for the predicted reason, and disappears when I remove the mechanism.” Try setting the ridge penalty large and watch the peak shrink: you’ve just identified the mechanism.

🔑 Key results to remember

  • Reproduction workflow: isolate the claim → find method/settings → minimal implementation → compare → reconcile every gap.
  • Double descent: beyond the interpolation threshold (\(P=n\)), test error spikes then descends again — the classic U-shape is incomplete.
  • Mechanism: over-parameterized least squares picks the minimum-norm interpolant, which is smooth and generalizes.
  • A faithful reproduction shows the effect at the predicted location, for the predicted reason — and vanishes when you remove the mechanism.
  • Unexplained gaps are bugs in disguise; “close enough” is where errors hide.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. The “interpolation threshold” in double descent is where…

  • training error first starts to fall.
    Training error falls throughout; the threshold is defined by capacity vs data, not by where training error begins dropping.
  • the learning rate is largest.
    It has nothing to do with the optimizer’s learning rate; it’s a property of model size relative to dataset size.
  • the model has just enough parameters to fit the training data exactly (parameters ≈ data points).
    Correct. At \(P\approx n\) there is essentially a unique, highly contorted interpolating solution, so test error spikes; this is the peak of the double-descent curve.
  • test and training error are equal.
    They’re generally never equal here; the threshold is about capacity matching dataset size, not error curves crossing.

Q2. Why does test error fall again when the model grows well past the interpolation threshold?

  • With many interpolating solutions available, least squares selects the minimum-norm (smoothest) one, which generalizes well.
    Correct. Over-parameterization gives the optimizer freedom to pick a smooth interpolant among many, reducing variance — the second descent.
  • The model memorizes the test set.
    The test set is never seen during fitting; memorizing it is impossible and not the mechanism.
  • Training error increases, balancing it out.
    Training error stays ~0 in the over-parameterized regime; it doesn’t rise.
  • The noise in the data disappears.
    The data noise is fixed; what changes is which interpolating solution gets chosen.

Q3. In a reproduction, you get a number “in the right ballpark” but a few points off the paper’s. Best next step?

  • Report it as a successful reproduction and move on.
    An unexplained gap may be a bug; declaring success prematurely propagates errors.
  • Assume the paper is wrong.
    Possible, but far more often the gap is in your setup. Investigate before concluding.
  • Tune your method until the numbers match.
    Tuning to hit a target number is how you fool yourself; you might match by coincidence while still being wrong.
  • Track down the cause: seed, data split, preprocessing, or an undocumented detail.
    Correct. Reconciling the gap is the most educational step — it reveals hidden assumptions and either fixes your bug or genuinely flags a problem with the paper.

Q4. Why is a minimal reproduction (random features + least squares) valuable, even though it’s simpler than the original deep-network experiments?

  • Because simpler models always generalize better.
    Generalization isn’t the point here, and that claim is false in general — recall the double-descent curve itself.
  • It isolates the mechanism cheaply and lets you toggle it on and off, building strong causal evidence.
    Correct. A minimal setup removes confounds, runs in seconds, and lets you show the effect appears for the predicted reason and disappears when you remove it — stronger evidence than a single big, hard-to-control experiment.
  • It exactly matches the original paper’s numbers.
    It won’t match exact numbers (different setup); it reproduces the phenomenon, which is the goal.
  • It avoids needing any math.
    It uses linear algebra and least squares throughout; minimality is about controlling confounds, not avoiding math.

✍️ Practice

1. Run reproduce_double_descent.py, then increase the ridge penalty from \(10^{-6}\) to, say, \(1.0\). What happens to the peak, and what does that tell you about the mechanism?

Show solution

The sharp peak at \(P=n\) shrinks or vanishes. A large ridge penalty prevents the model from interpolating exactly — it regularizes toward small weights — so the “one wildly contorted fit” at the threshold never arises. This is strong evidence that the peak is caused by unregularized interpolation at the threshold: remove the mechanism (allow regularization) and the effect disappears. Demonstrating that an effect vanishes when you disable its proposed cause is one of the most convincing things a reproduction can do — it’s a controlled ablation, the subject of the next track.

2. The script averages test error over 25 random feature draws per \(P\). Why average, and what would a single draw look like?

Show solution

A single random-feature draw is noisy — the exact features you sample affect the fit, especially near the threshold where the problem is ill-conditioned. A single run’s curve would be jagged and the peak’s height would vary run to run, making it hard to see the clean phenomenon. Averaging over draws estimates the expected behaviour and reveals the smooth double-descent shape. This is a preview of Track 9: results that depend on randomness must be averaged over seeds (with error bars) to be trustworthy — a single run can mislead.

3. Conceptual: double descent contradicts the simple “more parameters ⇒ more overfitting” rule from Track 3. Is the bias–variance tradeoff wrong?

Show solution

Not wrong — incomplete as usually taught. The classical U-shape is exactly the left portion of the double-descent curve (the under-parameterized regime), where the bias–variance tradeoff holds as described. Double descent adds a second regime past the interpolation threshold that the classical analysis didn’t consider, where adding parameters reduces variance by enabling smoother minimum-norm solutions. The lesson for a researcher: textbook rules have regimes of validity, and a faithful reproduction can reveal exactly where a familiar rule stops applying — which is how fields advance.
