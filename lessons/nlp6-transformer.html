Track 5 · NLP & Transformers

The Transformer architecture

⏱ 30 min Lesson 5.6 ⬇ Runnable notebook
Why this existsThe transformer block — attention + FFN wrapped in residuals and layer norm — is the universal unit of modern AI. Stack it and you have GPT, BERT, and everything since.
By the end, you can
  • Describe the transformer block's sub-layers
  • Explain the roles of attention vs the FFN
  • Know why residuals/layer norm enable depth
  • Assemble a block and stack it

Concept The transformer block: attention + feed-forward, wrapped well

Self-attention mixes information across tokens, but on its own it's all linear blending. The transformer block combines it with a per-token feed-forward network (a small MLP, Lesson 4.2) that adds nonlinear processing, and wraps both in the two stabilizing tricks from Track 4: residual connections and layer normalization. That's the entire unit — and a transformer is just a stack of these identical blocks.

The two sub-layers play complementary roles. Attention moves information between tokens (each token gathers context, Lesson 5.5). The feed-forward network then processes each token's gathered representation independently, adding the nonlinearity and capacity to transform it (it's typically much wider than the model dimension — e.g., 4×). The residual connections (add the input back) give gradients a clean path through deep stacks, and layer normalization keeps activations well-scaled — both essential for training depth (Lesson 4.8). Crucially, input and output have the same shape, which is what lets you stack blocks indefinitely.

Concept Stacking blocks into the full model

The complete transformer is: turn tokens into embeddings (Lesson 5.2), add positional information (Lesson 5.7), pass through $N$ identical transformer blocks, and finish with an output head that produces next-token probabilities (Lesson 5.3). GPT-3 has 96 such blocks; small models have a handful. Each block refines the representation a little more — early blocks capture local/syntactic patterns, later ones capture meaning and long-range structure.

Worked example A transformer block in NumPy

Let's assemble one full block: self-attention, residual + layer norm, feed-forward, residual + layer norm — and confirm the shape is preserved and layer norm does its job. (3 tokens, dimension 8.)

import numpy as np
np.random.seed(0)
softmax = lambda x: np.exp(x - x.max(-1, keepdims=True)) / np.exp(x - x.max(-1, keepdims=True)).sum(-1, keepdims=True)
layernorm = lambda x: (x - x.mean(-1, keepdims=True)) / (x.std(-1, keepdims=True) + 1e-5)

X = np.random.randn(3, 8)                       # 3 token vectors, dim 8

# 1) self-attention sub-layer (single head; Q=K=V=X for brevity)
A = softmax(X @ X.T / np.sqrt(8))
x1 = layernorm(X + A @ X)                        # residual + norm

# 2) feed-forward sub-layer (8 -> 16 -> 8, with ReLU)
W1, W2 = np.random.randn(8, 16), np.random.randn(16, 8)
ffn = np.maximum(0, x1 @ W1) @ W2
out = layernorm(x1 + ffn)                        # residual + norm

print("input shape:", X.shape, "-> output shape:", out.shape, "(same!)")
print("output mean per token (~0):", out.mean(-1).round(3))
print("output std  per token (~1):", out.std(-1).round(3))
Outputinput shape: (3, 8) -> output shape: (3, 8) (same!) output mean per token (~0): [ 0. -0. -0.] output std per token (~1): [1. 1. 1.]

The block took 3 token vectors and returned 3 token vectors of the same shape — which is exactly why you can stack 96 of them. The two residual additions (X + ... and x1 + ...) let the original signal flow through untouched if a sub-layer isn't helpful, and the layer norms forced each token's output to mean ≈ 0, std ≈ 1, keeping the activations well-behaved no matter how deep the stack. Notice how little code it is: attention is a few matrix multiplies, the FFN is two more, and the wrapping is addition plus normalization. That deceptively simple unit, repeated and scaled, is the architecture behind every large language model — the elegance is the point.

★ Key points to remember
  • A transformer block = multi-head self-attention + a per-token feed-forward network, each wrapped with residual + layer norm.
  • Attention moves information between tokens; the FFN transforms each token (adding nonlinearity and capacity).
  • Residuals give gradients a clean path; layer norm keeps activations well-scaled — both enable deep stacks (Lesson 4.8).
  • Input and output have the same shape, so blocks stack; the full model is embeddings + position → N blocks → output head.
  • Depth ($N$ blocks) is the main scale knob; each block refines the representation further.

Practice Build the reasoning yourself

1In the worked example, the output had mean ≈ 0 and std ≈ 1 for every token. Which operation forced this, and why is it helpful in a deep stack?
Show solution & reasoning

Layer normalization forced it — by definition it subtracts each token vector's mean and divides by its standard deviation, producing mean ≈ 0, std ≈ 1. (Real layer norm then applies learnable scale/shift parameters, but the normalization is the core.)

It's helpful because it keeps activations in a consistent, well-scaled range as they pass through dozens of blocks — preventing the explode/vanish drift of Lesson 4.8. Without it, small scale differences would compound across 96 layers and destabilize training. Layer norm (rather than batch norm) is used in transformers because it normalizes each token independently, so it doesn't depend on batch statistics — convenient when sequence lengths and batch compositions vary.

2Attention is permutation-equivariant and the FFN is applied per token. So where in the whole transformer does word order actually get encoded?
Show solution & reasoning

Neither the attention nor the FFN encodes order on its own — attention sees a set (Lesson 5.5), and the FFN treats each token identically regardless of position. Order enters before the blocks, via positional encoding (Lesson 5.7) added to the token embeddings at the input. After that addition, each token vector carries "what" (the word) plus "where" (its position), so the order information is baked into the representations the blocks then process.

This cleanly separates concerns: the blocks handle relationships and transformations, while a one-time positional injection handles order. It's why the diagram shows "token embeddings + positional encoding" as the very first step, before any block.

3A small transformer has 12 blocks; GPT-3 has 96. Beyond "more layers," what does increasing the number of blocks buy, and what's the cost?
Show solution & reasoning

More blocks give the model more capacity and a deeper hierarchy of representations — early blocks tend to capture local syntax, middle blocks phrase- and entity-level meaning, and later blocks abstract, long-range, and task-relevant structure. Depth lets the model compose simple relationships into complex ones, much like depth in a CNN (Lesson 4.6). More blocks (and wider ones) is a large part of why bigger models are more capable (Track 6's scaling laws).

The cost is compute, memory, and data: each block adds parameters to train and store, and inference runs through every block for every token, so latency and cost scale with depth. You also need proportionally more training data to fit the extra parameters without overfitting. The residual/layer-norm wrapping is what makes 96 blocks trainable at all — without those Track-4 tricks, such depth would suffer vanishing gradients. So the architecture scales gracefully, but not freely.

★ Deep dive: reading the original diagram, and why this design endures▸

If you open the original "Attention Is All You Need" paper, the famous diagram can look intimidating, but you now have every piece. The boxes labeled "Multi-Head Attention" are Lesson 5.5; "Feed Forward" is a per-token MLP; "Add & Norm" is a residual connection plus layer normalization; "Positional Encoding" is Lesson 5.7; and the stack of these is just $N$ blocks. The one new wrinkle in the full paper is that it shows an encoder–decoder for translation (Lesson 5.8), where the decoder has an extra "cross-attention" sub-layer that attends to the encoder's output — but each piece is something you've now built. The whole architecture is genuinely just attention + MLP + residual + norm, repeated. The intimidation comes from seeing it all at once; the understanding comes from the bottom-up path you took.

Why has this exact design proven so durable, dominating NLP and now spreading to vision, audio, biology, and beyond? A few reasons. It's parallelizable (no recurrence), so it trains efficiently on modern hardware and scales to enormous size. It has excellent gradient flow (residuals everywhere), so it trains stably at great depth. It's general — attention makes no assumption about grid or sequence structure beyond "things relate to other things," so the same block works for text tokens, image patches, or amino acids. And it scales predictably: make it bigger, feed it more data, and it reliably gets better (Track 6). Few architectures combine all of these. The remarkable thing is how little is in the design — there's no clever task-specific structure, just a uniform stack of attention-and-MLP blocks, and yet that uniformity is precisely what lets it absorb scale and generality. Understanding this block deeply means you understand the computational core of essentially all of modern AI; everything in Tracks 6 through 8 is about what you do with this architecture — how you train it, align it, retrieve for it, and let it act.

The block treats its input as a set, so we owe it one missing ingredient: a sense of order. Next, in 5.7 Positional encoding, we see how a clever addition of vectors tells the otherwise order-blind transformer which word came where.
