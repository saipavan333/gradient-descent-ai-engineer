Autoregressive models

The most direct way to model \(p(x)\): break it into a chain of one-step predictions, each conditioned on everything generated so far. It is exact, it is simple, and you have already trained one — GPT.

Track 6 · Lesson 1 Concept + code ~24 min Builds on: Lesson 0, Track 5

Autoregressive models are the family that needs no approximation: they write \(p(x)\) exactly using a rule from probability you already know, then generate by sampling one coordinate at a time. The GPT you built in Track 5 is exactly this idea applied to text. Here we strip it to its core on 2-D data so the whole mechanism fits on one screen.

💡 Concept: the chain rule turns one hard problem into many easy ones

Modelling a joint distribution over several variables at once, \(p(x_1,x_2,\dots,x_n)\), is daunting. But the chain rule of probability — true for any distribution, with no assumptions — lets us factor it into a product of one-variable conditionals:

\[ p(x_1,x_2,\dots,x_n) \;=\; p(x_1)\,\prod_{i=2}^{n} p\!\left(x_i \mid x_1,\dots,x_{i-1}\right). \] (1.1)

Read it left to right: the probability of the whole is the probability of the first coordinate, times the probability of the second given the first, times the third given the first two, and so on. The shorthand \(x_{<i}\) means “all coordinates before \(i\),” so the model is a collection of predictors \(p(x_i\mid x_{<i})\). “Autoregressive” literally means regressing on itself — each output becomes part of the input for the next.

You already did this. In Track 5, GPT modelled \(p(\text{token}_i \mid \text{tokens}_{<i})\) and generated text by sampling one token at a time, appending each to the context. That is an autoregressive model over sequences — Equation (1.1) with “token” for \(x_i\). Everything here is the same skeleton with a tiny model in place of a transformer.

📐 Why the factorization is exact (and what we must choose)

Equation (1.1) is not an approximation — it is the definition of conditional probability applied repeatedly. From \(p(a,b)=p(a)\,p(b\mid a)\), peel off one variable at a time:

\[ p(x_1,x_2,x_3) = p(x_1)\,p(x_2,x_3\mid x_1) = p(x_1)\,p(x_2\mid x_1)\,p(x_3\mid x_1,x_2). \] (1.2)

Because it holds for any ordering of the variables, you are free to pick one (left-to-right for text, raster order for images). What the model must learn is each conditional \(p(x_i\mid x_{<i})\). The art of autoregressive modelling is choosing a good predictor for these conditionals: a lookup table for tiny problems, a causal convolution for images (PixelCNN), or a masked transformer for text (GPT). The training objective is always the same — maximise the likelihood (1.1) of real data, which, taking logs, is just the sum of per-coordinate log-probabilities. That sum is the cross-entropy / next-token loss you already minimised in Track 5.

🛠️ From scratch: an autoregressive model with no neural network

To see the mechanism with nothing hidden, we model 2-D points with the simplest possible conditionals — histograms. Discretize the plane into a \(K\times K\) grid. Then \(p(x_1)\) is a histogram over columns, and \(p(x_2\mid x_1)\) is, for each column, a histogram over rows. That is the entire model; there is no gradient descent at all, yet it generates convincing two-moons by ancestral sampling. From autoregressive.py: 

K = 24                                   # bins per axis
b1, b2 = binize(X[:,0]), binize(X[:,1])  # which cell each point falls in

# p(x1): how often each column is occupied
p1 = np.bincount(b1, minlength=K).astype(float); p1 /= p1.sum()

# p(x2 | x1): for each column, the distribution over rows
joint = np.zeros((K, K))
for i, j in zip(b1, b2): joint[i, j] += 1
cond = joint / joint.sum(1, keepdims=True)

# ANCESTRAL SAMPLING: draw x1 ~ p(x1), then x2 ~ p(x2 | x1)
col = rng.choice(K, p=p1)                 # first coordinate
row = rng.choice(K, p=cond[col])          # second, conditioned on the first

The two rng.choice calls are Equation (1.1) in code: sample the first factor, then the second factor conditioned on what you drew. Scale \(K\) up and add more coordinates and this is literally how PixelCNN paints an image pixel by pixel.

🔑 Key results to remember

  • Chain rule (exact): \(p(x)=\prod_i p(x_i\mid x_{<i})\) for any distribution and any ordering — no approximation.
  • Ancestral sampling: generate one coordinate at a time, feeding each back in. Sequential by construction.
  • Training = maximum likelihood, which factors into a sum of per-coordinate log-losses — the next-token cross-entropy from Track 5.
  • Exact likelihood is the headline strength; slow, sequential sampling is the headline weakness.
  • GPT, PixelCNN, WaveNet are all this same recipe with a powerful learned conditional in place of a histogram.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. The chain-rule factorization \(p(x)=\prod_i p(x_i\mid x_{<i})\) holds…

  • only when the coordinates are independent.
    Independence is the opposite case — it would let you drop the conditioning entirely. The chain rule’s power is that it needs no independence.
  • only for Gaussian distributions.
    No — it is distribution-free. Gaussians are one special case among all distributions it covers.
  • for any distribution and any ordering of the variables.
    Correct. It follows from the definition of conditional probability applied repeatedly, so it is exact universally; the ordering is a free choice.
  • only approximately, with error that grows with dimension.
    It is exact, not approximate. The difficulty is modelling each conditional well, not any factorization error.

Q2. What is the main practical drawback of autoregressive models at generation time?

  • They cannot compute the likelihood of data.
    Backwards — exact likelihood is their strength. The product (1.1) gives \(p(x)\) directly.
  • Sampling is sequential: each coordinate waits for all previous ones.
    Correct. You must draw \(x_1\), then \(x_2\mid x_1\), etc. For a 1024-token sequence that is 1024 sequential steps — slow compared with models that emit all coordinates at once.
  • They require adversarial training that often diverges.
    That describes GANs (Lesson 3). Autoregressive models train by stable maximum likelihood.
  • They can only model discrete data.
    They handle continuous data too (e.g. WaveNet on audio). Sequentiality, not data type, is the limitation.

Q3. Training an autoregressive model by maximum likelihood is equivalent to…

  • minimising the distance between generated and real images pixel-by-pixel.
    That is a reconstruction loss, not the AR objective. AR maximises the probability of the real data under the factorized model.
  • a two-player game between a generator and a critic.
    Again that is the GAN setup. AR training is a single, stable optimisation.
  • maximising only \(p(x_1)\), the first coordinate.
    All factors matter; the objective sums log-probabilities across every coordinate.
  • minimising the summed cross-entropy of each next-coordinate prediction.
    Correct. \(\log p(x)=\sum_i \log p(x_i\mid x_{<i})\), so maximising likelihood means minimising the total next-step cross-entropy — exactly the GPT training loss.

Q4. In the histogram demo, why does raising \(K\) (more bins) eventually hurt?

  • With too many bins, many cells receive zero or one training point, so \(p(x_2\mid x_1)\) becomes noisy or undefined.
    Correct. Finer grids mean less data per cell — the classic bias–variance trade-off (Track 3). Empty conditioning columns have no distribution to sample from. Real models replace the table with a smooth learned function to share statistical strength.
  • The chain rule stops being exact for large \(K\).
    The factorization is exact regardless of \(K\); the problem is estimating each conditional from finite data.
  • Sampling becomes adversarial.
    There is no adversary here; sampling stays a simple two-step draw. The issue is data sparsity per bin.
  • The model can no longer represent the gap between the moons.
    More bins represent the gap better, not worse; the failure is statistical (empty cells), not representational.

✍️ Practice

1. Run autoregressive.py and change the factor order to model \(p(x_2)\,p(x_1\mid x_2)\) instead. Should the generated cloud look the same? Why?

Show solution

It should look essentially the same. The chain rule is exact for any ordering, so both factorizations describe the identical joint \(p(x_1,x_2)\). You are just conditioning along the other axis first. Tiny differences appear only from binning and finite-sample noise, not from the math. (In high dimensions the order can still matter for a limited model, because some orderings make the conditionals easier to learn — but the target distribution is unchanged.)

2. Compute the model’s log-likelihood on the data: \(\sum \log p(x_1)+\log p(x_2\mid x_1)\) over all points. Add a small constant to empty bins first. Why is this number a fair score for an autoregressive model but not for a GAN?

Show solution

Because an autoregressive model defines an explicit, normalised \(p(x)\), you can read off the probability of any real point and sum the logs — a genuine likelihood you can compare across models. The “small constant on empty bins” is Laplace smoothing; without it a single zero makes the whole log-likelihood \(-\infty\). A GAN (Lesson 3) never defines \(p(x)\) explicitly — it only knows how to sample — so there is no density to evaluate, and likelihood simply is not available as a score. Different families, different yardsticks.

3. Conceptual: a 256×256 colour image has 196,608 coordinates. Estimate how many sequential steps a pixel-by-pixel autoregressive model needs to generate one image, and explain why this motivated later parallel approaches.

Show solution

One step per coordinate (often three, one per colour channel), so on the order of \(2\times10^5\) sequential network evaluations for a single image — each waiting on the last. Even at a millisecond per step that is minutes per image, and it cannot be parallelised across pixels because pixel \(i\) needs pixels \(<i\). This sequential bottleneck is a major reason the field explored VAEs and GANs (which emit a whole image in one forward pass) and later diffusion (a fixed, modest number of refinement steps). Exactness is wonderful; waiting 200,000 steps is not.
