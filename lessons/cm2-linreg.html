Linear regression from scratch

⏱ 28 min Lesson 3.2 ▶ Open in Colab
Why this existsLinear regression is the simplest real model and the template for all of them: define a prediction, define a loss, minimize it. Master this and every later model is a variation on the theme.
By the end, you can
  • Write the linear model y = w·x + b
  • Define mean squared error and why it squares errors
  • Solve it two ways: normal equations and gradient descent
  • Interpret the weights — and the scale caveat

Concept The simplest real model: a straight line

The first model in machine learning is the one you already know from school: a straight line. In linear regression we predict a number (a "continuous" target like price or temperature) by taking a weighted sum of the features. With one feature it's the line $\hat{y} = w\,x + b$, where the prediction is written $\hat{y}$ ("y-hat" — the hat means "estimated"), $w$ is the weight (the slope: how much $y$ moves per unit of $x$), and $b$ is the bias (the intercept: the prediction when $x = 0$).

With many features it's the same idea written with a dot product (Lesson 1.3): $\hat{y} = w_1 x_1 + w_2 x_2 + \cdots + w_d x_d + b = \mathbf{w}\cdot\mathbf{x} + b$. Each feature gets its own weight, and the model adds them up. That's it — linear regression is just "a weight per feature, summed, plus an offset." Despite its simplicity it's still a workhorse: fast, interpretable, and a strong baseline on tabular data.

Concept Measuring wrongness: mean squared error

To "fit the line" we need a number that says how bad a given $(w, b)$ is. For each example, the residual is the miss: prediction minus truth, $\hat{y}_i - y_i$. We square each residual (so positive and negative misses both count as badness, and big misses count a lot more), then average over all $n$ examples. That average is the mean squared error:

$$\text{MSE} = \frac{1}{n}\sum_{i=1}^{n} \left(\hat{y}_i - y_i\right)^2$$

This is exactly the loss function from Lesson 1.8, now with a name and a purpose. Training the model means choosing $w$ and $b$ to make MSE as small as possible — the lowest point of the bowl. Two more reasons squaring is the default: it's smooth (so we can take gradients), and minimizing squared error has a clean statistical meaning (it's the maximum-likelihood answer under Gaussian noise, from Lesson 1.10).

Concept Two routes to the best line

Because the MSE bowl is convex (a single valley, no false bottoms), linear regression is one of the rare models where you can jump straight to the exact best answer with a formula — no iteration needed. That formula is the normal equations: stacking a column of 1s onto $X$ (to absorb the bias) and writing the parameters as $\boldsymbol{\beta}$, the minimizer is $\boldsymbol{\beta} = (X^\top X)^{-1} X^\top y$. It uses exactly the matrix machinery from Lessons 1.2 and 1.4.

The other route is gradient descent (Lesson 1.8): start anywhere and repeatedly step downhill. Why bother, if a formula exists? Because $(X^\top X)^{-1}$ costs too much to compute when you have millions of examples or features, and because every model after this one (logistic regression, neural nets) has no closed form — gradient descent is the universal method. Linear regression is the gentle place to see both side by side and confirm they agree.

Worked example Fitting a line from scratch, then with scikit-learn

We'll generate data with a known true relationship — slope 2.5, intercept 4, plus noise — then recover it two ways and check they match. First the closed-form normal equations, in pure NumPy:

It recovered intercept $\approx 4.07$ and slope $\approx 2.46$ — close to the true 4 and 2.5, with the small gap caused by the random noise. Now the one-liner everyone actually uses, scikit-learn's LinearRegression, which should agree exactly:

from sklearn.linear_model import LinearRegression

model = LinearRegression().fit(X, y)
print("scikit-learn  -> intercept, slope:", round(model.intercept_, 3), round(model.coef_[0], 3))

pred = model.predict(X)
mse = ((pred - y) ** 2).mean()
print("MSE:", round(mse, 3))
Outputscikit-learn -> intercept, slope: 4.071 2.462 MSE: 2.345

Identical answer — because LinearRegression solves the very same normal equations under the hood. The MSE of 2.345 is close to the noise variance we baked in ($1.5^2 = 2.25$), which is the best any model could do here: the leftover error is irreducible noise, not a flaw in the line. That's the whole loop — define a model, define a loss, minimize it — and you just did it from scratch.

★
Reading the coefficients

A weight $w_j$ says: "holding the other features fixed, increasing feature $j$ by one unit changes the prediction by $w_j$." That interpretability is linear regression's superpower. But beware of comparing raw weights to judge feature importance — a feature measured in dollars and one measured in millions will have wildly different-sized weights for reasons of scale, not importance. Standardize features first (Lesson 2.5) if you want comparable weights.

★ Key points to remember
  • Linear regression predicts a number as a weighted sum of features: $\hat{y} = \mathbf{w}\cdot\mathbf{x} + b$.
  • It's trained by minimizing mean squared error, the average of squared residuals.
  • Squaring punishes big misses and makes the loss a smooth, convex bowl with one global minimum.
  • You can solve it in closed form (normal equations) or by gradient descent; both give the same line.
  • Weights are interpretable ("per-unit effect"), but only comparable across features if the features are scaled.

Practice Build the reasoning yourself

1A model is $\hat{y} = 1.5x + 2$. For three points $(x, y) = (0, 3),\ (2, 4),\ (4, 9)$, compute the residuals and the MSE.
Show solution & reasoning

Predictions: at $x=0$, $\hat{y}=2$; at $x=2$, $\hat{y}=5$; at $x=4$, $\hat{y}=8$.

Residuals ($\hat{y}-y$): $2-3=-1$; $5-4=+1$; $8-9=-1$.

Squared: $1,\ 1,\ 1$. MSE $= (1+1+1)/3 = \mathbf{1.0}$. Notice the residuals don't cancel here only because we squared them — their plain sum is $-1$, which would have hidden two of the three misses.

2You fit linear regression and the training MSE is tiny but the test MSE is large. What's happening, and does the convexity of MSE protect you from it?
Show solution & reasoning

This is overfitting: the model fit the training data's noise, not just its signal — usually because there are too many features relative to examples (each feature is another knob to memorize with). The model generalizes poorly.

Convexity does not save you. Convexity guarantees you found the global minimum of the training loss — but a low training loss is exactly the problem when the model is too flexible. The cure isn't better optimization; it's controlling complexity, which is what regularization (Lesson 3.6) does by adding a penalty that discourages large weights.

3The relationship between $x$ and $y$ is clearly a curve (a U-shape), but you fit a straight line. Describe what goes wrong, and one simple way to let a linear model still capture the curve.
Show solution & reasoning

A straight line can't bend, so it will underfit: it splits the difference, with large residuals that are systematically positive in some regions and negative in others (a tell-tale pattern when you plot residuals). The model has high bias (Lesson 3.5).

A simple fix: add a polynomial feature. Include $x^2$ as an extra column, so the model becomes $\hat{y} = w_1 x + w_2 x^2 + b$. It's still "linear" in the weights (so all the same math applies), yet it can now fit a parabola. This trick — engineering nonlinear features for a linear model — is a recurring and powerful move, and the conceptual seed of what kernels do for SVMs (Lesson 3.9).

★ Deep dive: multiple features, R², and where linear models break▸

With $d$ features the picture lifts from a line to a hyperplane in $d$-dimensional space, but nothing else changes: still a weighted sum, still MSE, still the same normal equations $\boldsymbol{\beta} = (X^\top X)^{-1}X^\top y$. Geometrically, that formula is a projection (Lesson 1.3): it finds the point in the space spanned by your feature columns that is closest to $y$, and the residual vector is exactly perpendicular to that space. Linear regression is orthogonal projection in disguise.

To report quality in a scale-free way, people use R² (the "coefficient of determination"): the fraction of the target's variance the model explains, where 1.0 is perfect and 0 means "no better than predicting the mean." It's just a normalized version of MSE that's easier to compare across datasets.

Two failure modes worth knowing. Multicollinearity: when features are strongly correlated (say "height in cm" and "height in inches"), $X^\top X$ becomes nearly singular (Lesson 1.4), the inverse blows up, and the individual weights become huge and unstable — even though predictions may still be fine. Outliers: because MSE squares errors, a single wild point can drag the whole line toward it; robust losses (like the absolute error, which grows linearly) resist this. Both problems motivate regularization, the next big idea after we detour through classification — adding a penalty on weight size that stabilizes the solution and tames overfitting.

You've built a model that predicts numbers. But a huge share of real problems ask a yes/no question instead — spam or not, fraud or not. Next, in 3.3 Logistic regression & classification, we bend the line into an S-curve that outputs probabilities.
