Sequence models: RNNs, LSTMs, GRUs

⏱ 27 min Lesson 4.7 ⬇ Runnable notebook
Why this existsSequences need memory. RNNs carry a hidden state through time; LSTMs/GRUs add a gated highway that fixes the vanishing gradients plain RNNs suffer — the ideas that motivate transformers.
By the end, you can
  • Explain the RNN hidden-state recurrence
  • Describe the vanishing/exploding gradient problem
  • Explain how LSTM/GRU gates fix it
  • Know why transformers replaced RNNs

Concept Data with an order: sequences need memory

CNNs assume grid structure; but language ("the movie was not good"), audio, stock prices, and sensor streams are sequences, where order carries meaning and each element depends on what came before. A plain network has no memory — it treats each input independently. A Recurrent Neural Network (RNN) adds memory: it processes a sequence one step at a time, maintaining a hidden state that summarizes everything seen so far and is updated at each step.

At each step the cell combines the new input with the previous hidden state to produce the new hidden state: $h_t = \tanh(W_h h_{t-1} + W_x x_t + b)$. The same weights ($W_h$, $W_x$) are reused at every step — parameter sharing across time, just as a CNN shares across space. That's why the diagram shows one cell "unrolled": it's literally the same cell applied repeatedly, with the hidden state threading them together.

Concept The vanishing-gradient problem

RNNs have a serious flaw on long sequences. To learn, gradients must flow backward through every time step (called backpropagation through time). But that means multiplying many local slopes together — and if those slopes are slightly less than 1, the product shrinks exponentially toward zero; if slightly more than 1, it explodes.

The consequence: a plain RNN effectively forgets the distant past. It can't connect "not" at the start of a sentence to "good" twenty words later, because the gradient linking them has vanished. This is the same vanishing-gradient issue that plagues deep feed-forward nets (Lesson 4.2), but amplified because a long sequence is like a very deep network in time.

Concept LSTMs and GRUs: a gated memory highway

The fix is a smarter cell. The LSTM (Long Short-Term Memory) adds a separate cell state — a "memory highway" that runs straight through with only minor, controlled edits, so gradients can flow across many steps without vanishing. Small neural gates decide what to do: a forget gate (what to erase), an input gate (what new information to store), and an output gate (what to expose). All are learned.

The GRU (Gated Recurrent Unit) is a simpler, faster variant with two gates instead of three; it often matches the LSTM. Both made it possible to learn long-range dependencies and powered a decade of sequence modeling — translation, speech, text generation — before transformers (Track 5) largely replaced them. But the core ideas here (memory, gating, the vanishing problem) are exactly what motivate the transformer, so they're worth understanding even as the architecture evolves.

Worked example An RNN's memory, and why gradients vanish

First, watch a tiny RNN carry a hidden state through a sequence. Each step blends the new input with the running memory through a tanh:

import numpy as np
seq = [1.0, 0.5, -1.0, 2.0]      # a length-4 input sequence
Wh, Wx, h = 0.8, 0.5, 0.0        # recurrent weight, input weight, initial state

for t, x in enumerate(seq):
    h = np.tanh(Wh * h + Wx * x)
    print(f"step {t+1}: input {x:+.1f}  ->  hidden state {h:+.3f}")
Outputstep 1: input +1.0 -> hidden state +0.462 step 2: input +0.5 -> hidden state +0.551 step 3: input -1.0 -> hidden state -0.059 step 4: input +2.0 -> hidden state +0.741

The hidden state at each step depends on the new input and the previous state — it's a running memory. Step 3's negative input pulls it down; step 4 recovers. Now the vanishing problem, in one line: backprop through $T$ steps multiplies $T$ local slopes, so a slope below 1 shrinks the signal exponentially:

for slope in [0.5, 1.5]:
    grad_factor = slope ** 20            # gradient through 20 time steps
    print(f"slope {slope} over 20 steps -> gradient x {grad_factor:.2e}")
Outputslope 0.5 over 20 steps -> gradient x 9.54e-07 slope 1.5 over 20 steps -> gradient x 3.33e+03

A per-step slope of 0.5 makes the gradient $\sim$10$^{-6}$ after 20 steps — vanished, so the early inputs get essentially no learning signal. A slope of 1.5 explodes to 3,330×. Plain RNNs live on this knife's edge, which is exactly why LSTMs/GRUs (with their additive cell-state highway, slope ≈ 1) were invented. In PyTorch you use them directly:

import torch.nn as nn
lstm = nn.LSTM(input_size=10, hidden_size=64, num_layers=2, batch_first=True)
# output, (h_n, c_n) = lstm(x)  for x of shape (batch, seq_len, 10)

One line gives you a stacked LSTM; nn.GRU and nn.RNN have the same interface. You rarely implement the gates by hand — but knowing what they fix (the exploding/vanishing factors above) is what lets you reason about sequence models.

★ Key points to remember
  • RNNs process sequences step by step, carrying a hidden state (memory); the same weights are reused across time.
  • Backprop through time multiplies many slopes, so plain RNNs suffer vanishing/exploding gradients and forget the distant past.
  • LSTMs/GRUs add a gated cell-state highway so gradients flow across many steps — learning long-range dependencies.
  • Gates (forget/input/output) are small learned networks deciding what to keep, add, and emit.
  • Transformers (Track 5) have largely replaced RNNs, but these ideas — memory, gating, the vanishing problem — motivate them.

Practice Build the reasoning yourself

1Using the worked example's RNN update $h_t = \tanh(0.8\,h_{t-1} + 0.5\,x_t)$ with $h_0 = 0$, compute $h_1$ for input $x_1 = 1.0$. (Use $\tanh(0.5) \approx 0.462$.)
Show solution & reasoning

$h_1 = \tanh(0.8 \times 0 + 0.5 \times 1.0) = \tanh(0.5) \approx \mathbf{0.462}$ — matching the worked example's step 1.

The previous state contributed nothing here because $h_0 = 0$, so only the input mattered. At later steps the $0.8\,h_{t-1}$ term carries the accumulated memory forward, which is why step 2 (with input 0.5) still rose to 0.551 — the prior state added to the new input. That recurrence is the entire mechanism of an RNN.

2You're classifying sentiment, and the key word "not" appears 30 words before the word it negates. Why might a plain RNN miss this, and what would you use instead?
Show solution & reasoning

A plain RNN would likely forget "not" by the time it reaches the negated word. Linking them requires a gradient to flow back ~30 steps, and through 30 multiplications the gradient vanishes (as the $0.5^{20}$ demo showed), so the network never learns to connect them — it sees "good" and predicts positive, missing the negation.

Use an LSTM or GRU, whose gated cell-state highway preserves information and gradients across long spans, so "not" can persist and influence the later prediction. Even better for long-range dependencies is a transformer (Track 5), whose attention mechanism connects any two positions directly in one step — no multiplying through 30 intermediate states at all, which is a big part of why transformers overtook RNNs.

3An RNN's training loss suddenly jumps to NaN after a few steps. Which gradient pathology is this, and name a standard quick fix.
Show solution & reasoning

This is exploding gradients — the per-step slope exceeded 1, so backprop through time blew the gradient up exponentially (the $1.5^{20} \approx 3330$ case, but worse over a long sequence), producing updates so large the weights overflow to NaN.

The standard quick fix is gradient clipping: cap the gradient's norm at some threshold (e.g., rescale it if its magnitude exceeds 5) before the optimizer step — torch.nn.utils.clip_grad_norm_(model.parameters(), 5.0). This prevents the catastrophic giant updates while keeping the gradient's direction. Clipping is near-universal in RNN training. (Vanishing, the opposite failure, isn't fixed by clipping — it needs LSTM/GRU gating.)

★ Deep dive: backprop through time, GRU vs LSTM, and why transformers won▸

Backpropagation through time (BPTT) is just backprop applied to the unrolled RNN: you treat the sequence of shared cells as one deep network and propagate gradients back across all time steps, summing the gradient contributions to the shared weights. For long sequences this is expensive and memory-heavy, so in practice people use truncated BPTT — backprop only a fixed window of steps — which bounds cost but also caps how far back the model can learn dependencies. The vanishing problem and truncation together explain why even LSTMs have practical limits on context length.

GRU vs LSTM: the GRU merges the forget and input gates into a single "update" gate and ditches the separate cell state, giving two gates instead of three. It has fewer parameters, trains a bit faster, and usually performs comparably — so it's a reasonable default when you want speed, while LSTMs sometimes edge it out on tasks needing very precise long-term memory. Empirically the choice rarely dominates results; both are solid.

So why did transformers (Track 5) replace them? Two reasons. First, RNNs are inherently sequential — step $t$ needs step $t-1$ — so they can't parallelize across the sequence, making them slow to train on modern hardware. Transformers process all positions at once. Second, attention connects any two positions in a single step regardless of distance, sidestepping the multiply-through-many-steps vanishing problem entirely, so they capture long-range dependencies far better. RNNs taught the field that sequence models need memory and that gradient flow is the central obstacle; the transformer is, in a sense, the answer to "what if we could give a model memory of the whole sequence without recurrence?" — which makes this lesson the perfect on-ramp to Track 5.
