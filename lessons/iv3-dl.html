Deep learning & transformers — Q&A

⏱ 26 min Lesson 12.3
Why this existsDeep learning questions probe how modern networks actually work: backpropagation, vanishing gradients, CNN vs RNN, and — above all — self-attention and transformers. Interviewers reward mechanistic understanding, not memorized definitions.
By the end, you can
  • Explain backprop via the chain rule
  • Explain vanishing gradients and fixes
  • Contrast CNNs, RNNs, and transformers
  • Explain self-attention and why it won

Concept How modern neural networks actually work

Deep learning questions test whether you understand the machinery behind modern AI — how networks learn (backpropagation), why they sometimes don't (vanishing gradients), which architecture fits which data, and above all the transformer and attention that power today's LLMs. As with fundamentals, interviewers probe the why, so intuition beats memorization. These questions cluster into a few groups.

Model answer "Explain backpropagation."

"Backpropagation is how a neural network computes the gradient of the loss with respect to every weight, so gradient descent can update them. A network is a stack of functions (layers), and to train it you need to know how much each weight contributed to the final error. Backprop does this efficiently using the chain rule: it computes the loss at the output (the forward pass), then works backward layer by layer, multiplying local derivatives to propagate the error signal from the output back to each weight. The key efficiency is that it reuses computed gradients — the error signal at a layer is built from the layer after it — so instead of computing each weight's effect separately (which would be astronomically expensive), one backward pass gets them all. So: forward pass computes the prediction and loss, backward pass computes every gradient via the chain rule, then gradient descent steps the weights downhill. It's the literal engine that makes training deep networks tractable." Follow-up: "Why is it efficient? — Dynamic-programming-style reuse: each layer's gradient is computed once from the layer downstream and reused, turning an exponential problem into a linear one." Naming the chain rule and the reuse is what shows you understand it, not just that it exists.

Model answer "What are vanishing/exploding gradients, and how are they addressed?"

"In a deep network, backprop multiplies many local gradients together as it propagates backward. If those values are consistently less than 1, their product shrinks toward zero — vanishing gradients — so early layers barely update and the network can't learn long-range or deep patterns. If they're consistently greater than 1, the product explodes, causing unstable, diverging training. This was a major obstacle to training deep and recurrent networks. The fixes: better activations (ReLU instead of sigmoid/tanh, since ReLU's gradient is 1 for positive inputs, not a shrinking fraction); careful weight initialization (schemes like Xavier/He that keep signal variance stable across layers); normalization (batch/layer norm, which keep activations well-scaled); residual (skip) connections (which give gradients a direct path backward, the key to very deep networks); and for exploding gradients specifically, gradient clipping (cap the gradient magnitude). For sequences, LSTMs/GRUs were designed with gating to preserve gradients over time." Why it lands: connecting the cause (product of many terms) to each fix (which addresses that product) shows mechanistic understanding, and residual connections are the insight that unlocked modern deep networks.

Model answer "CNN vs RNN — what's each for, and why?"

"They encode different inductive biases matched to different data. A CNN (convolutional neural network) is built for grid-like data with local spatial structure — images especially. It slides small filters across the input, so it (a) exploits locality (nearby pixels are related) and (b) reuses the same filter everywhere (parameter sharing / translation invariance — a cat is a cat wherever it appears), making it efficient and effective on images. An RNN (recurrent neural network) is built for sequential data — text, time series — processing elements one at a time while maintaining a hidden state that carries information forward, so it models order and context over a sequence. The reason to match them to data is that the architecture's built-in assumptions (local spatial patterns for CNNs, sequential dependencies for RNNs) let it learn efficiently from the right data and generalize well. That said, I'd add that for sequences, transformers have largely replaced RNNs because they handle long-range dependencies better and parallelize, and even in vision, transformers now compete with CNNs — so these classic architectures are foundational but the field has moved toward attention." Framing it as inductive bias matched to data structure, then noting the transformer shift, shows both depth and currency.

Model answer "Explain self-attention and why transformers replaced RNNs."

"Self-attention lets every position in a sequence directly look at every other position and weigh how much each one matters for it. Concretely, each token produces a query, a key, and a value; a token's new representation is a weighted sum of all tokens' values, where the weights come from how well its query matches each key. So 'it' in a sentence can attend strongly to the noun it refers to, no matter how far away — the model learns these relationships. Transformers stack layers of this (plus feed-forward layers), with positional encodings to inject word order since attention itself is order-agnostic. Transformers replaced RNNs for two decisive reasons. Long-range dependencies: an RNN passes information step-by-step through a hidden state, so distant tokens are connected only through a long, lossy chain (the vanishing-gradient problem over time), whereas attention connects any two tokens directly in one step. Parallelism: an RNN must process tokens sequentially (each step depends on the previous), but a transformer processes all tokens at once, which is vastly more efficient on modern hardware (GPUs) — this is what made training on massive data feasible and enabled today's LLMs. So attention solved both the modeling limitation and the training-efficiency limitation of RNNs at once." Why it lands: explaining the query-key-value mechanism and the two concrete reasons (direct long-range connections + parallelism) is exactly the depth transformer questions demand — and it's the most important architecture to nail.

Model answer "What does batch normalization do, and why does it help?"

"Batch normalization normalizes the activations within each layer — for each mini-batch, it standardizes the layer's inputs to roughly zero mean and unit variance (then applies learnable scale and shift so the network can undo it if needed). It helps in several ways. It stabilizes and speeds training: by keeping activations well-scaled across layers, it makes the loss landscape smoother and gradients better-behaved, so you can use higher learning rates and train faster and more reliably. It reduces sensitivity to initialization and mitigates the vanishing/exploding issues by controlling activation scale. And it has a mild regularizing effect (the batch statistics add noise). The original motivation was reducing 'internal covariate shift' (layers constantly adapting to shifting input distributions), though the exact reason it helps is debated — the smoother-optimization explanation is now favored. The practical upshot is that it's a widely-used trick that makes deep networks much easier to train." Follow-up: "How does it behave at inference? — It uses running averages of the statistics collected during training, not batch statistics, so predictions don't depend on other items in the batch." Knowing the train/inference difference signals hands-on experience.

★ Rapid-fire answers to have ready
  • Backprop? Chain rule computing every weight's gradient in one efficient backward pass; reuses downstream gradients.
  • Vanishing gradients? Product of many small gradients → zero; fix with ReLU, good init, normalization, residual connections.
  • CNN vs RNN? CNN for spatial/grid data (locality + parameter sharing); RNN for sequences (hidden state) — now often transformers.
  • Self-attention? Every token attends to every other via query-key-value; direct long-range links + full parallelism.
  • Why transformers > RNNs? Direct long-range dependencies and parallel processing (RNNs are sequential and lossy over distance).
  • Dropout? Randomly zero units during training to prevent co-adaptation — a regularizer; off at inference.
  • Embeddings? Dense vectors where similar items are close; learned representations that replace sparse one-hots.
  • Why deep (many layers)? Each layer builds higher-level features from lower ones — hierarchical representation learning.

Practice Mock interview prompts

1"Walk me through what happens, step by step, in one training iteration of a neural network."
Show a strong answer

"One training iteration has four steps. (1) Forward pass: feed a batch of inputs through the network — each layer transforms its input (a linear step plus a nonlinear activation) — producing predictions at the output. (2) Compute the loss: compare the predictions to the true labels with a loss function (e.g., cross-entropy for classification), giving a single number that measures how wrong the network was on this batch. (3) Backward pass (backpropagation): starting from the loss, apply the chain rule backward through the layers to compute the gradient of the loss with respect to every weight — how much each weight contributed to the error. (4) Update the weights: the optimizer (e.g., SGD or Adam) nudges each weight a small step in the direction that reduces the loss — opposite its gradient, scaled by the learning rate. Then repeat with the next batch. Over many iterations, the weights settle into values that make good predictions. The whole loop is: predict, measure error, compute how to reduce it, take a step — gradient descent powered by backprop." This lands because it names all four phases precisely (forward, loss, backward, update), correctly places backprop as computing gradients (not updating), and connects to the optimizer and learning rate — showing you understand the training loop as a coherent mechanism, not a black box.

2"Your deep network's training loss won't go down at all. What could be wrong, and how would you debug it?"
Show a strong answer

"A loss that won't decrease at all points to a few common culprits, which I'd check systematically. Learning rate: too high and it diverges or bounces (loss may spike or stay flat/erratic); too low and it barely moves — I'd try a range, since LR is the most common cause. Bugs in the setup: a wrong loss function, labels misaligned with inputs, a data-loading error feeding garbage, or forgetting to zero gradients between steps — I'd verify by trying to overfit a tiny batch (a healthy network should drive the loss to near zero on a handful of examples; if it can't, there's a bug, not a capacity issue). Vanishing gradients: a very deep network with poor activations/init may have no gradient signal reaching early layers — I'd check gradient magnitudes and consider ReLU, better init, normalization, or residual connections. Data issues: unnormalized inputs, or no learnable signal in the features. Dead ReLUs / bad initialization: units stuck outputting zero. My first move is always the overfit-a-single-batch test — it cleanly separates 'the model/plumbing is broken' from 'the model can't generalize,' and it's the fastest diagnostic. Then I'd check the learning rate, inspect a few inputs/labels for correctness, and watch gradient norms." This answer stands out because it offers a concrete diagnostic protocol (especially the overfit-tiny-batch test that experienced practitioners rely on), rather than a vague list — showing you've actually debugged networks.

3"Explain the intuition for why very deep networks became trainable — what was the key breakthrough?"
Show a strong answer

"The central obstacle to depth was that gradients degrade as they propagate back through many layers — vanishing (or exploding) gradients meant early layers in a very deep network barely learned, so adding layers stopped helping and often hurt. Several advances chipped at this (ReLU activations, better initialization, batch normalization), but the decisive breakthrough for very deep networks was residual connections (skip connections), introduced with ResNets. The idea is simple and powerful: instead of each block having to learn a full transformation, it learns a residual — a small change added to its input, which is passed through unchanged via a skip connection. This does two things. First, it gives gradients a direct highway backward: the skip path lets the gradient flow to early layers without being repeatedly multiplied down, so vanishing gradients are dramatically mitigated and networks of hundreds of layers become trainable. Second, it makes optimization easier — a block can trivially represent 'do nothing' (output its input) by driving the residual to zero, so adding layers can't easily make things worse. This unlocked the depth that modern architectures (including transformers, which use residual connections throughout) rely on. So the key insight to name is that residual connections let gradients and information skip past layers, solving the degradation that had capped network depth." This lands because it identifies the specific breakthrough (residual connections), explains the mechanism (a gradient highway + easy identity mapping), and connects it to transformers — demonstrating you understand not just that deep networks work but why they became possible.

★ Deep dive: reasoning about architectures from first principles▸

The strongest deep-learning answers treat architectures not as a list of names to memorize but as inductive biases matched to data structure — assumptions baked into the model about what patterns matter. A CNN assumes locality and translation invariance, which is exactly right for images; an RNN assumes sequential dependence, right for ordered data; attention assumes any element might relate to any other, a very general bias that lets transformers learn the relevant structure from data rather than having it hardcoded. Understanding architectures this way lets you reason about why each fits its domain, predict where it will struggle (a CNN on data with no spatial locality, an RNN on very long-range dependencies), and even understand the trend toward transformers: attention's minimal, general inductive bias means that with enough data and compute, transformers can learn structure that more specialized architectures had hardcoded, which is why they've spread from language to vision to other domains. When an interviewer asks "why this architecture?", the deep answer is always about the match between the model's built-in assumptions and the data's actual structure — that framing turns architecture questions from recall into reasoning.

The other theme running through deep-learning questions is that training deep networks is a battle against gradient pathologies, and most of the field's key tricks exist to win it. Vanishing/exploding gradients, addressed by activations (ReLU), initialization (Xavier/He), normalization (batch/layer norm), and architecture (residual connections), are the recurring enemy — and recognizing that these seemingly-separate techniques all attack the same underlying problem (keeping gradient signal healthy through depth and time) is a sign of real understanding. This connects directly to why the transformer, which combines attention (direct connections, no long gradient chains), residual connections, and layer normalization, was such an effective design: it's engineered to keep gradients flowing while modeling long-range structure and parallelizing. For interviews, the meta-lesson is to always push past "what" to "why": why does this component exist, what problem does it solve, how does it work mechanically, and where does it break? A candidate who can explain that backprop is the chain rule with reuse, that vanishing gradients come from a product of small terms, that residual connections are a gradient highway, and that attention gives direct long-range connections with parallelism, demonstrates the mechanistic understanding that separates someone who has used deep learning frameworks from someone who understands what those frameworks are doing. That understanding is what lets you debug real networks, choose and adapt architectures, and reason about the LLMs that build on all of it — which is the next lesson.
