CNN Feature-Map Lab

A convolutional layer isn't one operation — it's a whole bank of small filters, each sliding over the image and lighting up wherever it finds its pattern. Here are six classic filters running on one image, so you can literally see what a CNN sees.

The magic of CNNs is that these filters aren't programmed — they're learned. But the mechanism is exactly what you see here: each 3×3 filter produces a "feature map" that's bright where its pattern appears (a vertical edge, a corner, a texture) and dark elsewhere. Stack layers, and later filters combine these into eyes, wheels, faces.

▶ One image, six filters, six feature maps

Top-left is the input. Each other tile is that image after one 3×3 convolution filter. Bright = the filter found its pattern there.

i
Why edges first? Trained CNNs almost always learn edge and color-blob detectors in their first layer — because edges are the most reusable building block of every image. The filters here are hand-set versions of exactly that. ReLU throws away negative responses (a filter either fires or it doesn't); pooling shrinks the map, making the network care whether a feature appeared, not exactly where.

Try these

1. Match filter to feature. Look at the vertical-edge map: it's bright only along the up-down borders of the shape, dark on flat areas and horizontal edges. The horizontal-edge filter does the opposite. Each filter is tuned to one orientation.

2. Toggle ReLU. With ReLU off you see both bright and dark responses (positive and negative). With it on, the negatives are clipped to black — the map now says "my pattern is here" or "it isn't", which is what the next layer wants.

3. Pool it. Turn on max-pooling and the maps get chunkier and smaller. The network keeps the strongest response in each little region and drops the rest — cheaper, and slightly position-invariant.

What this lab proves
  • A conv layer = many filters. Each produces its own feature map highlighting one pattern.
  • Weight sharing. The same 3×3 filter slides everywhere, so a pattern is detected wherever it appears.
  • ReLU makes it a detector. Keep positive responses, drop the rest — "found it / didn't".
  • Pooling adds robustness. Downsampling keeps the strongest signal and a little position-invariance.
