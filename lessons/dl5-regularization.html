Track 4 · Deep Learning

Regularization: dropout, weight decay, batchnorm

⏱ 26 min Lesson 4.5 ⬇ Runnable notebook
Why this existsDeep nets have the capacity to memorize — regularization keeps them generalizing. Dropout, weight decay, batch norm, and early stopping are the everyday tools.
By the end, you can
  • Explain dropout and inverted-dropout scaling
  • Connect weight decay to L2 regularization
  • Describe batch norm and early stopping
  • Toggle train/eval mode correctly

Concept Deep nets have huge capacity — so they overfit

A network with millions of weights can memorize its training set perfectly, including its noise (the overfitting of Lesson 3.5, now with far more capacity to misbehave). The cure is the same instinct as Track 3.6 — penalize or constrain complexity — but deep learning has its own toolkit for it. We'll cover the four you'll use constantly: dropout, weight decay, batch normalization, and early stopping. They're often combined.

Concept Dropout: train a different thinned network each step

Dropout randomly switches off a fraction of neurons (say 50%) on each training step — their outputs are forced to 0 for that step. Each step therefore trains a different, smaller "thinned" network, and crucially no neuron can rely on any specific other neuron being present, so the network is forced to learn redundant, robust features instead of brittle co-dependencies.

At test time you switch dropout off and use all neurons, which acts like averaging the predictions of all those thinned networks — a built-in ensemble (Lesson 3.8), and a big part of why dropout fights overfitting. One detail makes the math line up: since training used only a fraction of neurons, the activations are scaled up during training (called inverted dropout) so their average size matches what test time will see. Frameworks handle this for you, but it's worth seeing why.

Concept Weight decay, batch norm, and early stopping

Weight decay is simply L2 regularization (Lesson 3.6) applied to a network: add a penalty on the sum of squared weights, which pulls them toward zero and discourages the large weights that overfitting needs. In PyTorch it's one argument: optim.Adam(..., weight_decay=1e-4).

Batch normalization normalizes each layer's activations (to mean 0, variance 1) within each mini-batch before passing them on. Its main job is to stabilize and speed up training by keeping activations in a good range (Lesson 4.8), but it also has a mild regularizing side effect from the batch-to-batch noise.

Early stopping is the simplest of all: watch the validation loss during training and stop when it starts rising, keeping the weights from the best epoch. It directly targets the moment overfitting begins.

Worked example Why inverted dropout keeps activations honest

Dropout zeros half the neurons during training — wouldn't that shrink the signal the next layer sees, creating a mismatch with test time (when nothing is dropped)? Inverted dropout fixes this by dividing the survivors by the keep probability. Let's verify that this keeps the average activation unchanged, by simulating many random dropout masks:

import numpy as np
rng = np.random.default_rng(0)
h = rng.normal(0, 1, 8)        # the 8 activations a layer would output
p = 0.5                         # drop probability

# inverted dropout: zero each unit w.p. p, scale survivors by 1/(1-p)
masks = rng.random((20000, 8)) > p          # 20k random keep/drop patterns
dropped = h * masks / (1 - p)               # apply dropout + inverted scaling
print("original h        :", h.round(2))
print("avg over droppings:", dropped.mean(axis=0).round(2))
Outputoriginal h : [ 0.13 -0.13 0.64 0.1 -0.54 0.36 1.3 0.95] avg over droppings: [ 0.12 -0.13 0.63 0.1 -0.53 0.36 1.31 0.95]

The average activation after dropout matches the original almost exactly — that's the point of the $1/(1-p)$ scaling. On any single step, half the values are 0 (a thinned network), but on average the signal passed forward is unchanged, so the network the next layer "expects" during training matches the full network used at test time. Without the scaling, activations would be half-sized during training and the model would misbehave when you turn dropout off. In PyTorch you never code this — you just declare it:

import torch.nn as nn
model = nn.Sequential(
    nn.Linear(784, 256), nn.ReLU(),
    nn.Dropout(0.5),                       # drop 50% of these activations in training
    nn.Linear(256, 10),
)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)  # + weight decay
# model.train() enables dropout; model.eval() turns it off for validation/test

The one gotcha: call model.train() and model.eval() to toggle dropout (and batch norm) between training and evaluation — forgetting eval() at test time is a classic bug that adds random noise to your predictions.

★ Key points to remember
  • Deep nets overfit because of huge capacity; regularization keeps them generalizing.
  • Dropout randomly zeros neurons during training → forces redundancy; at test it averages like an ensemble.
  • Weight decay = L2 penalty on weights (Lesson 3.6); one optimizer argument.
  • Batch norm normalizes activations → faster, more stable training (plus mild regularization).
  • Early stopping halts at the validation-loss minimum; toggle dropout/batchnorm with model.train()/eval().

Practice Build the reasoning yourself

1A network gets 99% training accuracy and 80% validation accuracy. Name three regularization techniques you'd add, each with a one-line reason.
Show solution & reasoning

The 19-point gap is overfitting. Dropout: forces redundant features so the net can't memorize via brittle co-adaptations. Weight decay (L2): penalizes large weights, shrinking the model's effective capacity. Early stopping: halt at the validation-loss minimum before memorization dominates.

Beyond these three: get more data or use data augmentation (the most effective cure when feasible — more examples leave less room to memorize), reduce model size, or add batch norm. You'd typically combine several and tune their strengths on the validation set. The diagnosis (large train–val gap → high variance) and the toolkit are the deep-learning version of Lesson 3.5/3.6.

2A model performs great during training but gives noticeably worse, slightly random predictions at test time. You used dropout. What's the likely bug?
Show solution & reasoning

You probably forgot to call model.eval() before evaluating, so dropout stayed active at test time — randomly zeroing neurons on every prediction, which both degrades and randomizes the output.

model.train() enables dropout (and uses batch statistics for batch norm); model.eval() disables dropout (and uses running statistics for batch norm). Always switch to eval() for validation/test and back to train() for training. This train/eval-mode bug is one of the most common in practice precisely because the code still "runs" — it just silently underperforms.

3Explain in one or two sentences why dropout is sometimes described as "training an ensemble of networks for the price of one."
Show solution & reasoning

Each training step zeros a different random subset of neurons, so it effectively trains a different "thinned" sub-network; over training you've optimized an enormous family of overlapping sub-networks that share weights. At test time, using all neurons (with the dropout scaling) approximates averaging the predictions of all those sub-networks — which is exactly what an ensemble does (Lesson 3.8), and ensembling reduces variance.

So you get the variance-reduction benefit of an ensemble without training and storing many separate models — the "ensemble" lives inside one network's shared weights. That connection to bagging is the deepest explanation for why such a simple trick (randomly zeroing activations) regularizes so effectively.

★ Deep dive: batch norm's real story, and data augmentation▸

Batch normalization was introduced to fix "internal covariate shift" — the idea that as early layers update, the distribution of inputs to later layers keeps shifting, making training unstable. By normalizing each layer's activations to mean 0, variance 1 within each mini-batch (then applying two learnable scale/shift parameters so the layer can undo it if needed), batch norm keeps activations in a healthy range, which lets you train deeper networks faster and with higher learning rates. Interestingly, later research suggested its main benefit may actually be smoothing the loss landscape rather than fixing covariate shift — a reminder that in deep learning, techniques often work for reasons subtler than the story that motivated them. Its regularizing effect comes from the noise of using batch statistics, which is also why batch norm and dropout are sometimes not combined (their interaction can hurt), and why alternatives like layer normalization are preferred in transformers (Track 5), where batch statistics are awkward.

The single most powerful regularizer, though, is often not on this list: data augmentation. Instead of constraining the model, you expand the data by generating realistic variations — for images, random crops, flips, rotations, and color shifts; for text, paraphrases or token masking; for audio, time-shifts and noise. Each augmentation is a free new training example that preserves the label, so the model sees far more variety and has much less room to memorize. A well-augmented dataset frequently beats any amount of dropout or weight decay, because the cleanest way to stop a model from memorizing the training set is to make the training set effectively much larger. This is why state-of-the-art vision and speech systems lean heavily on augmentation — and it's the regularizer to reach for first whenever your domain allows realistic transformations.
