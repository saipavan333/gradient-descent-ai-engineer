Track 4 · Deep Learning

From a neuron to a network

⏱ 24 min Lesson 4.1 ⬇ Runnable notebook
Why this existsDeep learning's atom is the neuron — a weighted sum through a nonlinearity, exactly the logistic regression you already know. Stack them and you get networks; understand one neuron and the rest is composition.
By the end, you can
  • Define a neuron: weighted sum + bias + activation
  • Stack neurons into layers and networks
  • Explain why nonlinearity is essential
  • Compute a neuron's output in NumPy and PyTorch

Concept The artificial neuron: a weighted sum, then a squash

Deep learning sounds mysterious, but its atom is something you already built in Track 3. A neuron (or "unit") takes several inputs, multiplies each by a weight, adds them up, adds a bias, and passes the result through a nonlinear activation function. That weighted sum is exactly a dot product (Lesson 1.3), and a single neuron with a sigmoid activation is logistic regression (Lesson 3.3). A neural network is just many of these tiny units wired together.

In symbols, a neuron computes $a = f(\mathbf{w}\cdot\mathbf{x} + b)$, where $\mathbf{w}$ are the weights, $\mathbf{x}$ the inputs, $b$ the bias, $z = \mathbf{w}\cdot\mathbf{x}+b$ the pre-activation score, and $f$ the activation. The weights say how much each input matters; the bias shifts the threshold; the activation introduces the nonlinearity that makes everything that follows possible.

Concept Stacking neurons into layers makes a network

Put several neurons side by side, all reading the same inputs, and you have a layer. Feed that layer's outputs into another layer, and another, and you have a neural network (also called a multi-layer perceptron, or MLP). The first layer reads the raw features (the input layer), the last produces the prediction (the output layer), and everything between is a hidden layer — "hidden" only because you don't directly observe its values.

Each layer is just a matrix multiply plus a bias plus an activation, applied to the layer before it (Lesson 1.2's matrix machinery, now doing real work). "Deep" learning simply means stacking many such layers. More layers (depth) and more neurons per layer (width) give the network more capacity — more adjustable weights, so it can represent more complex functions.

Concept Why the nonlinearity is non-negotiable

Here's the subtle point that justifies the whole architecture. If you remove the activation functions, every layer is a linear map — and a stack of linear maps collapses into a single linear map (matrix times matrix is just another matrix). A hundred linear layers would be no more powerful than one: still just a straight line/plane. All the depth would buy you nothing.

The activation function — applied between layers — is what breaks that collapse. By bending the signal at each layer, nonlinearities let a deep network compose simple pieces into arbitrarily complex functions. In fact, a network with just one hidden layer and enough neurons can approximate any continuous function (the "universal approximation theorem"). Depth is what makes that practical. Without the humble activation, there is no deep learning.

Worked example One neuron, by hand in NumPy — then in PyTorch

Let's compute a single neuron's output for a concrete input, both activations. This is the entire forward computation of a neuron — nothing hidden:

import numpy as np

x = np.array([2.0, -1.0, 3.0])     # three inputs
w = np.array([0.5, -1.0, 0.25])    # one weight per input
b = 0.5                            # the bias

z = w @ x + b                      # weighted sum (a dot product!) plus bias
print("z =", z)
print("ReLU(z)    =", max(0.0, z))
print("sigmoid(z) =", round(1 / (1 + np.exp(-z)), 4))
Outputz = 3.25 ReLU(z) = 3.25 sigmoid(z) = 0.9627

The score is $z = (0.5)(2) + (-1)(-1) + (0.25)(3) + 0.5 = 3.25$. ReLU passes it through unchanged (it's positive); sigmoid squashes it to 0.96 — read as "96% confident," exactly the logistic-regression output from Track 3. Now the same operation in PyTorch, where a whole layer of neurons is one object:

import torch
import torch.nn as nn

layer = nn.Linear(in_features=3, out_features=4)   # 4 neurons, each reading 3 inputs
x = torch.tensor([2.0, -1.0, 3.0])
output = torch.relu(layer(x))                       # w@x + b for all 4 neurons, then ReLU
print(output.shape)                                 # torch.Size([4])

nn.Linear(3, 4) holds a 4×3 weight matrix and a length-4 bias — it computes four neurons at once with a single matrix multiply, and PyTorch fills the weights with sensible random values for you. (In Keras the equivalent is layers.Dense(4, activation="relu").) That one line, stacked a few times, is a neural network — which is exactly what we assemble next.

★ Key points to remember
  • A neuron computes $a = f(\mathbf{w}\cdot\mathbf{x} + b)$: a weighted sum (dot product) plus bias, through an activation.
  • A single sigmoid neuron is logistic regression — deep learning generalizes what you already know.
  • Stacking neurons into layers (input → hidden → output) makes a network; depth + width = capacity.
  • The nonlinear activation is essential: without it, stacked layers collapse to one linear map.
  • In PyTorch a layer of neurons is nn.Linear(in, out) — one matrix multiply for the whole layer.

Practice Build the reasoning yourself

1A neuron has weights $\mathbf{w} = [1, -2]$, bias $b = 1$, and ReLU activation. Compute its output for inputs $\mathbf{x} = [3, 1]$ and for $\mathbf{x} = [0, 2]$.
Show solution & reasoning

For $\mathbf{x}=[3,1]$: $z = (1)(3) + (-2)(1) + 1 = 3 - 2 + 1 = 2$. ReLU$(2) = \mathbf{2}$.

For $\mathbf{x}=[0,2]$: $z = (1)(0) + (-2)(2) + 1 = 0 - 4 + 1 = -3$. ReLU$(-3) = \mathbf{0}$ — the neuron is "off" for this input. Notice the same neuron fires for one input and stays silent for another; a layer of such neurons, each tuned to different patterns, is how a network builds up a rich representation of its input.

2Someone builds a 5-layer network but forgets all activation functions (every layer is just $\mathbf{W}\mathbf{x}+\mathbf{b}$). How powerful is this network, really?
Show solution & reasoning

It's exactly as powerful as a single linear layer — i.e., plain linear/logistic regression. Composing linear maps gives another linear map: $\mathbf{W}_5(\mathbf{W}_4(\cdots \mathbf{W}_1\mathbf{x})) = \mathbf{W}_{\text{combined}}\,\mathbf{x}$ for one combined matrix. The five layers collapse into one.

So despite having five times the parameters, it can only draw straight decision boundaries and fit linear relationships — it would fail on XOR or any curved pattern. This is the concrete reason activations are mandatory: they're the only thing preventing this collapse. The fix is one line per layer — wrap each in a ReLU.

3A network has an input layer of 3 features, one hidden layer of 4 neurons, and an output layer of 1 neuron. How many weights (ignore biases) does it have, and where do they live?
Show solution & reasoning

Weights live on the connections between layers. Input→hidden: every one of the 3 inputs connects to every one of the 4 hidden neurons → $3 \times 4 = 12$ weights (a 4×3 matrix). Hidden→output: each of the 4 hidden neurons connects to the 1 output → $4 \times 1 = 4$ weights. Total $= 12 + 4 = \mathbf{16}$ weights (plus $4 + 1 = 5$ biases).

The pattern: a layer mapping $m$ inputs to $n$ neurons holds an $n \times m$ weight matrix. This is why parameter counts explode in big networks — a single layer mapping 1,000 to 1,000 units already has a million weights, and modern models stack hundreds of far larger layers.

★ Deep dive: universal approximation, depth vs width, and the brain analogy▸

The universal approximation theorem says a network with a single hidden layer and enough neurons can approximate any continuous function to arbitrary accuracy. So why go deep instead of just very wide? Because "enough neurons" can mean an exponentially large number for a shallow net, while a deep net can represent the same function far more efficiently. Depth lets the network build a hierarchy — early layers learn simple features (edges, in an image), later layers compose them into complex ones (shapes, objects). The same function that needs a billion neurons in one layer might need only a few thousand across ten layers. Depth is about efficient representation, which is why "deep" learning, not "wide" learning, took over.

The "neuron" name comes from a loose analogy to biological brains — real neurons also sum inputs and "fire" past a threshold. But the analogy is loose: artificial neurons are vastly simpler, the brain doesn't do backpropagation as we know it, and chasing biological realism has rarely helped deep learning progress. It's better to think of a neuron as a tunable mathematical function — a weighted sum with a switch — than as a tiny brain cell. The power comes from composing millions of these simple functions and tuning them all with gradient descent, which is exactly the machinery the next three lessons build: the forward pass (how a network computes), backpropagation (how it learns), and the training loop (how it all runs).

You can build a network out of neurons. Next, in 4.2 Forward pass & activation functions, we push data all the way through one — layer by layer, as matrix multiplies and activations — and meet the activation functions you'll actually use.
