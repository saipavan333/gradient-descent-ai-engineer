PAC learning & VC dimension

We’ve seen generalization happen. Now the theory: when can we mathematically guarantee that low training error implies low test error? The answer connects how much data you have to how much capacity your model has — and gives the deepest reason “more data helps.”

Track 3 · Lesson 3 Concept + code ~28 min Builds on: Lessons 3.0–3.2, Track 1 probability

💡 Concept: probably approximately correct

Generalization is a probabilistic promise, and the framework that makes it precise is PAC learning — Probably Approximately Correct (Valiant, 1984). We can’t guarantee a model is exactly right (noise makes that impossible) or right with certainty (an unlucky sample could mislead any learner). So we settle for the two relaxations baked into the name: with high probability (at least \(1-\delta\)), the model is approximately correct (test error within \(\varepsilon\) of training error). Formally, a learner PAC-learns if, given enough samples,

$$ \Pr\big[\,\text{test error} \le \text{training error} + \varepsilon\,\big] \;\ge\; 1-\delta. $$
(★)

The two small numbers are knobs: \(\varepsilon\) is how close (accuracy) and \(\delta\) is how sure (confidence). The whole game is figuring out how many training examples \(n\) you need to make this promise — the sample complexity — and the answer turns out to depend on the capacity of your model class.

📐 Measuring capacity: shattering and VC dimension

“Capacity” is how many different patterns a model can fit. For a finite set of candidate models the natural measure is just the count, but most model classes are infinite (every setting of real-valued weights is a different model). The right capacity measure for them is the VC dimension (Vapnik–Chervonenkis), built on a vivid idea called shattering. A model class shatters a set of points if, no matter how you label them (every possible assignment of + and −), some model in the class can separate them perfectly. The VC dimension is the size of the largest set the class can shatter.

VC dimension formalizes the complexity axis from Lessons 3.0–3.1: a higher VC dimension means the model can fit more arbitrary patterns, which means more flexibility, which means more capacity to overfit. A straight line (VC = 3 in 2D) is simple; a high-degree polynomial or a big neural network has enormous VC dimension.

🎯 The generalization bound: data vs capacity

Here is the payoff theorem, stated in spirit. With \(n\) training examples and a model class of VC dimension \(d\), the generalization gap is bounded (with probability \(1-\delta\)) by roughly:

$$ \text{test error} \;\lesssim\; \text{training error} \;+\; O\!\left(\sqrt{\frac{d\,\log n + \log(1/\delta)}{n}}\right). $$

Don’t memorize the formula — read its shape, which is what matters. The extra term (the worst-case gap) grows with capacity \(d\) and shrinks with data \(n\), like \(1/\sqrt{n}\). Two immediate, profound consequences: (1) more data always tightens the guarantee, and (2) a more complex model needs proportionally more data to earn the same guarantee. This is the rigorous version of “capacity and data size must balance” — and it’s why throwing data at a problem is so reliably effective.

⚠️ Where classical theory and deep learning collide

Be honest about the limits. Modern neural networks have astronomically large VC dimension — the classical bound above would predict terrible generalization, yet they generalize well. The worst-case VC bound is true but loose: it’s pessimistic because it assumes the adversary picks the hardest possible data. Real data and the implicit biases of SGD (Track 2) make the effective capacity far smaller than the raw VC dimension. Reconciling this is active research (margin theory, the double-descent curve from Lesson 3.0, implicit regularization). The VC framework gives the right intuition — more capacity needs more data — even where its exact numbers don’t bind.

🛠️ From scratch: watch the gap shrink with data

We can’t compute VC dimension by brute force, but we can directly measure the generalization gap as a function of \(n\) and confirm the \(1/\sqrt{n}\) shape — and verify a line shatters 3 points but not the XOR 4.

code/track-3/pac_vc.py
import numpy as np
rng = np.random.default_rng(0)

# A linear classifier trained by least squares; measure the train/test gap vs n.
def gap_for_n(n, d=20, trials=40):
    gaps = []
    w_true = rng.normal(size=d)
    for _ in range(trials):
        Xtr = rng.normal(size=(n, d)); ytr = np.sign(Xtr @ w_true)
        Xte = rng.normal(size=(2000, d)); yte = np.sign(Xte @ w_true)
        w = np.linalg.lstsq(Xtr, ytr, rcond=None)[0]      # fit
        tr = np.mean(np.sign(Xtr @ w) != ytr)
        te = np.mean(np.sign(Xte @ w) != yte)
        gaps.append(te - tr)
    return np.mean(gaps)

print(f"{'n':>6} {'gap':>8}")
for n in [25, 50, 100, 400, 1600]:
    print(f"{n:>6} {gap_for_n(n):>8.3f}")   # gap shrinks ~ 1/sqrt(n)

# Shattering check: can a linear threshold realize ALL labelings of k points?
import itertools
def shatters(points):
    P = np.c_[points, np.ones(len(points))]               # add bias column
    for labels in itertools.product([0, 1], repeat=len(points)):
        y = np.array(labels)*2 - 1                          # {0,1} -> {-1,+1}
        w, *_ = np.linalg.lstsq(P, y, rcond=None)
        if not np.all(np.sign(P @ w) == y):                # this labeling failed
            return False
    return True

three = np.array([[0,0],[1,0],[0,1]])                      # generic 3 points
xor4  = np.array([[0,0],[1,1],[0,1],[1,0]])                # XOR square
print("shatters 3 points:", shatters(three))              # True
print("shatters XOR 4   :", shatters(xor4))               # False  -> VC dim = 3

🔑 Key results to remember

Lock these in

  1. PAC: with enough data, test error ≤ training error + \(\varepsilon\) with probability \(\ge 1-\delta\) (Probably Approximately Correct).
  2. Shattering: a class shatters points if it can realize every labeling. VC dimension = size of the largest shattered set = capacity.
  3. A 2D linear classifier has VC dimension 3.
  4. The bound: generalization gap \(\lesssim \sqrt{(d\log n)/n}\) — grows with capacity \(d\), shrinks like \(1/\sqrt{n}\) with data.
  5. Classical VC bounds are loose for deep nets (huge VC, yet generalize) — right intuition, not tight numbers. Active research.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. What does the “Probably” in “Probably Approximately Correct” refer to?

  • The model is probably the best one.
    PAC makes no claim about being best — it bounds generalization. “Probably” is about confidence, not optimality.
  • The predictions are probabilities.
    The outputs needn’t be probabilities; “Probably” qualifies the guarantee, not the output format.
  • The guarantee holds with high probability (≥ 1−δ), not with certainty.
    Correct. An unlucky training sample could fool any learner, so we only promise the bound holds with probability at least \(1-\delta\). “Approximately” handles the \(\varepsilon\) slack.
  • The data is probably noisy.
    Noise is a separate consideration. “Probably” is specifically about the confidence \(1-\delta\) of the bound.

Q2. A model class “shatters” a set of points when…

  • it classifies them all correctly for the true labels.
    Shattering is stronger: it must handle every possible labeling, not just the one true labeling.
  • for every possible labeling of those points, some model in the class separates them.
    Correct. Shattering means all \(2^k\) labelings are realizable — the class is flexible enough to fit any pattern on those points.
  • the points are linearly separable.
    Linear separability concerns one labeling; shattering demands all labelings, which is far stronger.
  • the points all have the same label.
    That’s a single trivial labeling. Shattering requires realizing every mix of labels.

Q3. The generalization bound says the worst-case gap shrinks with \(n\) at what rate?

  • Like \(1/\sqrt{n}\).
    Correct. The bound scales as \(\sqrt{d/n}\), so quadrupling the data roughly halves the gap. Diminishing returns, but reliable.
  • Like \(1/n^2\).
    That would be far faster than the theory gives; the standard VC bound decays as \(1/\sqrt{n}\), not \(1/n^2\).
  • It doesn’t depend on \(n\).
    Data size is central — more data is precisely what tightens the bound. The dependence on \(n\) is the whole point.
  • It grows with \(n\).
    More data shrinks the gap, never grows it. The bound improves with \(n\).

Q4. Deep nets have enormous VC dimension yet generalize well. What does this tell us about the classical VC bound?

  • The bound is wrong.
    The bound is a valid upper bound — it’s not false, just not tight. Reality can be much better than the worst case it describes.
  • Deep nets don’t really generalize.
    They demonstrably do generalize well in practice; that’s the empirical fact prompting the question.
  • VC dimension is meaningless.
    VC dimension still captures the right intuition (capacity vs data); it just isn’t the tight predictor for over-parameterized nets.
  • It’s a valid but loose (pessimistic) worst-case bound; real data and SGD make effective capacity smaller.
    Correct. The bound assumes a worst-case adversary; real data structure and the implicit bias of SGD mean the effective capacity is far below the raw VC dimension, so actual generalization beats the bound.

✍️ Practice

Problem 1 · reasoning

A threshold classifier on the real line (“predict + if \(x > \theta\)”) has VC dimension 1. Show it can shatter 1 point but not 2.

Show solution

One point: for a single point you need to realize both labelings (+ and −). Put \(\theta\) below the point to label it +, or above it to label it −. Both work, so 1 point is shattered. Two points \(x_1 < x_2\): there are 4 labelings, but the threshold can only produce labels that are “− then +” as you move right (everything left of \(\theta\) is −, right is +). It can never make \(x_1\) be + while \(x_2\) is − (that needs + on the left, − on the right — impossible for a single rightward threshold). One of the 4 labelings is unrealizable, so 2 points can’t be shattered. Hence VC dimension = 1.

Problem 2 · reasoning

You want to halve your model’s generalization gap. Using the \(1/\sqrt{n}\) rate, by what factor must you increase the training data?

Show solution

The gap scales as \(1/\sqrt{n}\). To halve it you need \(1/\sqrt{n'} = \tfrac12\cdot 1/\sqrt{n}\), i.e. \(\sqrt{n'} = 2\sqrt{n}\), so \(n' = 4n\). You must quadruple the data to halve the gap. This is the sobering arithmetic of the \(1/\sqrt{n}\) rate: data buys generalization, but with steeply diminishing returns — which is also why architectural and regularization improvements (that lower effective capacity) are so valuable, since they shift the whole curve down rather than crawling along it.

Problem 3 · code

Run pac_vc.py. Confirm the gap roughly follows \(1/\sqrt{n}\) (e.g. compare \(n=100\) to \(n=400\)) and that the shattering check gives True for 3 points, False for XOR’s 4.

Show solution

Going from \(n=100\) to \(n=400\) (4×) should roughly halve the gap (since \(1/\sqrt{4}=1/2\)), and you’ll see it shrink steadily across \(25 \to 1600\). The shattering function returns True for the three generic points (a linear threshold with a bias can realize all 8 labelings) and False for the XOR square (the labeling that puts opposite corners together has no separating line). This is the VC dimension of the 2D linear classifier — exactly 3 — verified in code rather than asserted.

Problem 4 · synthesis

Tie the whole track together: how do generalization (3.0), bias–variance (3.1), regularization (3.2), and VC/PAC (3.3) describe the same phenomenon from four angles?

Show solution

They’re four lenses on one thing — the tension between fitting your data and generalizing beyond it. Generalization (3.0) names the goal and shows the U-shaped test curve empirically. Bias–variance (3.1) decomposes that curve into two competing forces, explaining why it’s U-shaped (bias falls, variance rises). Regularization (3.2) is a practical lever to move along the curve — shrink capacity to trade a little bias for a lot less variance. PAC/VC (3.3) gives the theoretical backbone: it defines capacity precisely (VC dimension) and proves the gap depends on capacity-over-data, justifying everything the other three lessons observed. Underfitting = high bias = low VC, too-rigid; overfitting = high variance = high VC with too little data; regularization = lowering effective capacity; and the \(1/\sqrt{n}\) bound = why more data fixes overfitting. One phenomenon, four complementary descriptions — and a researcher uses all four fluidly.
