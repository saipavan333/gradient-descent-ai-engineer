Regularization (L1/L2), geometrically

If overfitting is a model using too much freedom to chase noise, the cure is to take some freedom away. Regularization does this by penalizing large weights — and the shape of the penalty explains one of ML’s most useful tricks: automatic feature selection.

Track 3 · Lesson 2 Concept + derivation + code ~28 min Builds on: Lessons 3.0–3.1, Tracks 1–2

💡 Concept: penalize complexity to reduce variance

From Lesson 3.1, overfitting is a variance problem: a flexible model uses large, finely-tuned weights to thread the training noise. Regularization fights this by adding a penalty on the size of the weights to the loss, so the optimizer is pulled toward smaller, simpler solutions. Instead of minimizing the loss alone, we minimize:

$$ \min_{\mathbf{w}} \;\; \underbrace{L(\mathbf{w})}_{\text{fit the data}} \;+\; \lambda\,\underbrace{R(\mathbf{w})}_{\text{keep } \mathbf{w} \text{ small}}. $$
(★)

\(L(\mathbf{w})\) is the usual loss (e.g. mean squared error), \(R(\mathbf{w})\) is a penalty that grows with the weights, and \(\lambda \ge 0\) (lambda) is a knob controlling how hard we push for small weights. The two standard penalties are:

$$ \textbf{L2 (ridge):}\;\; R(\mathbf{w}) = \lVert\mathbf{w}\rVert_2^2 = \sum_i w_i^2, \qquad\qquad \textbf{L1 (lasso):}\;\; R(\mathbf{w}) = \lVert\mathbf{w}\rVert_1 = \sum_i |w_i|. $$

\(\lambda\) directly sets the bias–variance balance: \(\lambda=0\) is the unregularized model (low bias, high variance); large \(\lambda\) forces tiny weights (high bias, low variance, in the limit the constant model from last lesson). So \(\lambda\) slides you along the U-curve, and you choose it on a validation set. Because both penalties are convex, adding them to a convex loss keeps the whole problem convex (Track 2) — regularized linear and logistic regression still have a single global optimum.

📐 The geometry: why L1 makes things sparse

The penalized problem (★) has an equivalent constrained form: minimize the loss subject to keeping the weights inside a budget, \(R(\mathbf{w}) \le t\). Now the shapes matter. The L2 budget \(\sum w_i^2 \le t\) is a round ball (a circle in 2D); the L1 budget \(\sum |w_i| \le t\) is a diamond with sharp corners on the axes. 🔑 L1 selects features; L2 shrinks them

L1 (lasso) drives some weights to exactly zero, effectively deleting those features — it performs automatic feature selection, yielding a sparse, interpretable model. L2 (ridge) shrinks all weights smoothly toward zero but rarely to exactly zero — it keeps every feature but tames them, and handles correlated features more gracefully. Choose L1 when you suspect many features are irrelevant; L2 (or both, “elastic net”) otherwise.

🤖 Where regularization comes from: a prior

Regularization isn’t an ad-hoc patch — it falls out of probability, extending Track 1’s maximum likelihood. Recall training = maximizing the likelihood of the data. If we also bring a prior belief that weights should be small (Bayes, Track 1) and maximize the posterior instead (called MAP estimation), the prior contributes exactly a penalty term. A Gaussian prior on the weights yields the L2 penalty; a Laplace (double-exponential) prior yields the L1 penalty. So “prefer small weights” is a prior, and regularized loss is the negative log-posterior. This is the same MLE→loss bridge from Track 1, now upgraded to MAP→regularized-loss — a deeply satisfying unification.

🛠️ From scratch: ridge and lasso

Ridge has a clean closed form using Track 1’s matrix algebra. Lasso has no closed form (the \(|w|\) kink), so we use a short iterative method — soft-thresholding — which makes the sparsity mechanism explicit.

code/track-3/regularization.py
import numpy as np
rng = np.random.default_rng(0)

# Data: y depends on features 0 and 1 only; features 2,3,4 are pure noise.
n, d = 60, 5
X = rng.normal(size=(n, d))
true_w = np.array([3.0, -2.0, 0.0, 0.0, 0.0])
y = X @ true_w + rng.normal(0, 0.5, n)

def ridge(X, y, lam):
    """L2 closed form: w = (X^T X + lam I)^{-1} X^T y  (Track 1 matrix algebra)."""
    d = X.shape[1]
    return np.linalg.solve(X.T @ X + lam*np.eye(d), X.T @ y)

def soft_threshold(z, t):
    return np.sign(z) * np.maximum(np.abs(z) - t, 0.0)

def lasso(X, y, lam, lr=0.01, steps=3000):
    """L1 via proximal gradient: gradient step on MSE, then shrink toward 0."""
    w = np.zeros(X.shape[1])
    for _ in range(steps):
        grad = X.T @ (X @ w - y) / len(y)        # MSE gradient
        w = soft_threshold(w - lr*grad, lr*lam)  # the shrink-toward-zero step
    return w

print("true weights :", true_w)
print("no reg (lam=0):", np.round(ridge(X, y, 0.0), 2))
print("ridge  (L2)  :", np.round(ridge(X, y, 5.0), 2))    # all shrunk, none exactly 0
print("lasso  (L1)  :", np.round(lasso(X, y, 0.3), 2))    # noise features -> exactly 0
# lasso zeros out features 2,3,4 -> it discovered which features matter
⚠️ Don’t penalize the bias, and standardize first

Two practical rules. (1) Don’t regularize the bias/intercept term — it just sets the output’s baseline level and shouldn’t be shrunk toward zero. (2) Standardize features (subtract mean, divide by std — Track 1) before regularizing, so the penalty treats them on equal footing; otherwise a feature measured in millimeters is penalized differently from the same feature in meters.

🔑 Key results to remember

Lock these in

  1. Regularization adds a weight penalty: \(\min_\mathbf{w} L(\mathbf{w}) + \lambda R(\mathbf{w})\). \(\lambda\) slides you along the bias–variance U-curve (tune on validation).
  2. L2 (ridge) \(=\sum w_i^2\): smooth shrinkage, keeps all features. L1 (lasso) \(=\sum|w_i|\): drives weights to exactly zero → feature selection.
  3. Geometry: L1’s diamond has corners on the axes; contours touch there first → sparsity. L2’s circle gives small-but-nonzero weights.
  4. Bayesian view: L2 = Gaussian prior, L1 = Laplace prior; regularized loss = negative log-posterior (MAP).
  5. Standardize features first; don’t penalize the bias term.

🧩 Check your understanding
Answer from memory, then read why.

Q1. You want a model that automatically ignores irrelevant features by setting their weights to exactly zero. Which penalty?

  • L2 (ridge).
    L2 shrinks weights smoothly but essentially never to exactly zero, so it keeps all features. It won’t produce a sparse model.
  • No regularization.
    Without a penalty there’s nothing pushing weights to zero; irrelevant features keep nonzero (often large) weights.
  • L1 (lasso).
    Correct. L1’s diamond geometry drives some weights to exactly zero, performing automatic feature selection — precisely what’s wanted.
  • A larger learning rate.
    The learning rate affects optimization speed, not sparsity. It won’t zero out coefficients.

Q2. As \(\lambda\) increases from 0 toward very large, what happens to bias and variance?

  • Bias increases, variance decreases.
    Correct. Larger \(\lambda\) forces smaller, simpler weights — more rigid (higher bias) but more stable across datasets (lower variance). \(\lambda\) slides you toward the high-bias end of the U-curve.
  • Bias decreases, variance increases.
    Backwards. That’s what reducing \(\lambda\) does. More regularization simplifies the model, raising bias.
  • Both increase.
    Regularization lowers variance (that’s its purpose). Both can’t increase.
  • Neither changes.
    \(\lambda\) directly controls weight size and thus model complexity, so it certainly shifts the bias–variance balance.

Q3. Geometrically, why does L1 produce sparsity but L2 usually doesn’t?

  • L1’s region is larger.
    Size isn’t the point (both are scaled by \(\lambda\)); it’s the shape. A bigger region would just regularize less.
  • L2 is not convex.
    L2 is convex (a smooth bowl). Convexity is shared by both; it doesn’t explain the difference.
  • L1 is non-differentiable everywhere.
    L1 is non-differentiable only at the kinks (where a coordinate is zero) — that’s relevant, but the geometric reason is the corners specifically being on the axes.
  • L1’s diamond has corners on the axes, where loss contours tend to touch first — and a corner has a coordinate equal to zero.
    Correct. The pointy corners stick out onto the axes, so the optimum lands there, zeroing a weight. L2’s round edge has no such preferred points.

Q4. The L2 penalty corresponds to which Bayesian prior on the weights?

  • A uniform prior.
    A uniform prior adds no penalty (MAP reduces to MLE). L2 actively prefers small weights, which a flat prior does not.
  • A Gaussian prior.
    Correct. A zero-mean Gaussian prior makes the negative-log-prior proportional to \(\sum w_i^2\) — exactly the L2 penalty. (L1 corresponds to a Laplace prior.)
  • A Laplace prior.
    That’s the prior for L1, whose log-density is proportional to \(\sum|w_i|\). The Gaussian gives L2.
  • No prior at all.
    “No prior” is MLE (unregularized). L2 specifically encodes a Gaussian prior belief that weights are small.

✍️ Practice

Problem 1 · derivation

Show that the ridge solution is \(\mathbf{w} = (X^\top X + \lambda I)^{-1} X^\top \mathbf{y}\), by minimizing \(\lVert X\mathbf{w}-\mathbf{y}\rVert^2 + \lambda\lVert\mathbf{w}\rVert^2\).

Show solution

Expand the objective: \(J(\mathbf{w}) = (X\mathbf{w}-\mathbf{y})^\top(X\mathbf{w}-\mathbf{y}) + \lambda\,\mathbf{w}^\top\mathbf{w}\). Take the gradient with respect to \(\mathbf{w}\) (matrix calculus, the multivariable version of Track 2’s derivatives):

$$ \nabla J = 2X^\top(X\mathbf{w}-\mathbf{y}) + 2\lambda\mathbf{w} = 0. $$

Rearrange: \(X^\top X\mathbf{w} + \lambda\mathbf{w} = X^\top\mathbf{y}\), i.e. \((X^\top X + \lambda I)\mathbf{w} = X^\top\mathbf{y}\). Solve: \(\mathbf{w} = (X^\top X + \lambda I)^{-1}X^\top\mathbf{y}\). \(\blacksquare\) Bonus: the \(+\lambda I\) makes the matrix invertible even when \(X^\top X\) is singular (Track 1’s determinant!) — ridge fixes the ill-conditioning that breaks plain least squares.

Problem 2 · reasoning

The soft-threshold operator is \(\text{soft}(z,t) = \text{sign}(z)\max(|z|-t,0)\). Explain how it creates sparsity, and what it does to a coefficient \(z=0.2\) with threshold \(t=0.3\).

Show solution

Soft-thresholding shrinks every coefficient toward zero by the amount \(t\), and clips at zero: any coefficient whose magnitude is below \(t\) is set to exactly zero. For \(z=0.2\), \(t=0.3\): \(|z|-t = 0.2-0.3 = -0.1 < 0\), so \(\max(\cdots,0)=0\) and \(\text{soft}=0\). The coefficient is killed. This is the L1 sparsity mechanism in one line — small weights get zeroed each iteration, so irrelevant features fade out, while large weights survive (merely shrunk by \(t\)). It’s the proximal operator of the \(|w|\) penalty.

Problem 3 · code

Run regularization.py. Confirm lasso zeros out the three noise features (2,3,4) while ridge keeps all five small-but-nonzero. What does increasing the lasso \(\lambda\) from 0.3 to 1.0 do?

Show solution

Lasso at \(\lambda=0.3\) returns weights near \((3,-2,0,0,0)\) — it recovered the true sparse structure, zeroing the noise features exactly. Ridge at \(\lambda=5\) returns five nonzero but shrunken weights (e.g. \(\approx(2.6,-1.7,0.1,-0.05,0.08)\)) — it never quite zeros the noise features, just damps them. Raising lasso \(\lambda\) to 1.0 shrinks even the true features toward zero (and may start zeroing them too), trading more bias for less variance — too much regularization now underfits. This is the U-curve: there’s a best \(\lambda\), found on validation data.

Problem 4 · reasoning

“Weight decay” in deep learning adds \(\lambda\lVert\mathbf{w}\rVert^2\) to the loss. Which classical regularizer is it, and what is it doing to a neural network’s millions of weights?

Show solution

Weight decay is exactly L2 regularization applied to the network’s weights. Each gradient step, on top of the loss gradient, it pulls every weight a little toward zero (the gradient of \(\lambda\lVert\mathbf{w}\rVert^2\) is \(2\lambda\mathbf{w}\), so the update gains a \(-2\alpha\lambda\mathbf{w}\) term — the weights “decay”). Functionally it keeps weights small, which limits how sharply the network can fit the training noise, reducing variance and improving generalization — the same bias–variance bargain as ridge regression, now on a model with millions of parameters. It’s one of the most-used regularizers in deep learning (Track 4), and a direct descendant of this lesson.
