Convolutional Neural Networks (CNNs)

⏱ 29 min Lesson 4.6 ⬇ Runnable notebook
Why this existsImages need an architecture that exploits spatial structure. CNNs slide shared filters over the image, detecting patterns anywhere with few parameters — the backbone of computer vision.
By the end, you can
  • Explain convolution and parameter sharing
  • Compute a convolution by hand
  • Describe pooling and the conv-pool-dense architecture
  • Build conv layers in PyTorch

Concept Why images need a different kind of layer

Imagine feeding a modest 200×200 color image to a plain fully-connected layer. The input has 200×200×3 = 120,000 numbers, so a single hidden layer of 1,000 neurons needs 120 million weights — for one layer. Worse, a fully-connected layer treats every pixel as unrelated, ignoring the obvious fact that nearby pixels form edges and shapes, and that a cat is a cat whether it's in the top-left or bottom-right of the frame. The Convolutional Neural Network (CNN) is the architecture built to exploit image structure efficiently.

Its core trick is the convolution: instead of connecting every pixel to every neuron, a small filter (or "kernel") — say 3×3 — slides across the image, computing a weighted sum at each position. The same filter (the same 9 weights) is reused at every location, which is called parameter sharing. That's how a 3×3 filter detects its pattern (an edge, a corner) anywhere in the image with just 9 weights instead of millions.

The output is a feature map — a grid showing where in the image the filter's pattern was found. A convolutional layer learns many filters at once (each detecting a different pattern), producing many feature maps stacked as channels. Early layers learn simple patterns (edges, colors); stacking convolutions lets later layers combine them into textures, then parts, then whole objects.

Concept Pooling: shrink while keeping the signal

Between convolutions, a pooling layer downsamples each feature map — most commonly max pooling, which slides a small window and keeps only the maximum value in each. This halves the spatial size (cutting computation), and makes the network a bit translation-tolerant: small shifts of a feature don't change the pooled output much.

A full CNN stacks these: alternating convolution and pooling layers extract a hierarchy of features and shrink the spatial dimensions, after which the result is flattened and fed to a few fully-connected layers that do the final classification.

Worked example One convolution by hand — an edge detector

Let's apply a single 3×3 filter to a small "image" by hand. Our filter is a classic vertical-edge detector: positive weights on the left column, negative on the right, so it produces a large response where the image goes from bright to dark horizontally.

import numpy as np
image = np.array([[3,1,2,0,1],
                  [0,4,1,2,3],
                  [1,2,3,1,0],
                  [2,0,1,4,2],
                  [1,3,2,0,1]], float)
kernel = np.array([[1,0,-1],
                   [1,0,-1],
                   [1,0,-1]], float)        # vertical-edge detector

kh, kw = kernel.shape
out = np.zeros((image.shape[0]-kh+1, image.shape[1]-kw+1))   # valid positions
for i in range(out.shape[0]):
    for j in range(out.shape[1]):
        patch = image[i:i+kh, j:j+kw]       # the 3x3 region under the filter
        out[i, j] = (patch * kernel).sum()  # weighted sum = one output value
print(out)
Output[[-2. 4. 2.] [-2. -1. 0.] [-2. 0. 3.]]

A 5×5 image convolved with a 3×3 filter gives a 3×3 feature map (the filter fits in 3 horizontal and 3 vertical positions). Each value is the dot product of the filter with the 9 pixels beneath it — and the same 9 weights were reused at all 9 positions. Large-magnitude outputs mark where the vertical-edge pattern is strong; the sign tells the direction of the brightness change. A real conv layer learns these filter weights by backprop rather than hand-setting them, and learns dozens at once. In PyTorch this whole sliding operation is one layer:

import torch.nn as nn
conv = nn.Conv2d(in_channels=3, out_channels=16, kernel_size=3)   # learn 16 filters
# input shape (batch, 3, H, W) -> output (batch, 16, H-2, W-2): 16 feature maps
pool = nn.MaxPool2d(2)                                            # halve H and W

nn.Conv2d(3, 16, 3) learns 16 different 3×3 filters across 3 input channels — and the entire sliding, multiplying, summing you wrote by hand is done efficiently on the GPU. (Keras: Conv2D(16, 3, activation="relu").) That handful of filters, stacked with pooling, is the backbone of image models.

★ Key points to remember
  • CNNs exploit image structure: a small filter slides over the image, reusing the same weights everywhere (parameter sharing).
  • Each filter produces a feature map showing where its pattern occurs; a layer learns many filters (channels).
  • Pooling (usually max) downsamples — less computation, more tolerance to small shifts.
  • Stacking conv+pool builds a hierarchy: edges → textures → parts → objects; then dense layers classify.
  • Convolution shares weights, so a CNN needs far fewer parameters than a fully-connected net on images.

Practice Build the reasoning yourself

1A 32×32 image is convolved with a 5×5 filter (no padding, stride 1). What is the output feature map's height and width? Then a 2×2 max-pool is applied — now what size?
Show solution & reasoning

A filter of size $k$ on an input of size $n$ (no padding, stride 1) gives output size $n - k + 1$. So $32 - 5 + 1 = 28$ → the feature map is 28×28. After 2×2 max pooling (which halves each dimension): 14×14.

This shrinking is why deep CNNs eventually reach a small spatial size (like 7×7 or 4×4) that's then flattened for the dense classifier. If you want the convolution to preserve size, you add "padding" (a border of zeros) of $(k-1)/2$ — here 2 pixels — so the output stays 32×32.

2Compare the parameter count: a dense layer connecting a 100×100 grayscale image (10,000 pixels) to 100 neurons, versus a conv layer with 100 filters of size 3×3. Roughly how many weights each?
Show solution & reasoning

Dense: every pixel to every neuron = $10{,}000 \times 100 = \mathbf{1{,}000{,}000}$ weights. Conv: 100 filters × (3×3) = $100 \times 9 = \mathbf{900}$ weights (plus 100 biases).

That's roughly a 1,000× reduction — and the conv layer can detect its patterns anywhere in the image, while the dense layer would have to learn the same pattern separately for every location. This combination of far fewer parameters and built-in translation handling is exactly why CNNs, not dense nets, power computer vision.

3In the worked example the filter weights were set by hand to detect vertical edges. In a real CNN, where do the filter values come from?
Show solution & reasoning

They're learned by backpropagation, just like any other weights (Lessons 4.3–4.4). The filter entries start random and are updated by gradient descent to minimize the loss, so the network discovers which patterns are useful for the task rather than being told.

Remarkably, when you visualize the filters a trained image CNN learns, the early layers end up resembling edge- and color-detectors very much like our hand-made one — the network rediscovers classic image-processing filters on its own because they're genuinely useful first steps. Later layers learn far more complex, task-specific patterns that would be impossible to design by hand. Hand-setting filters (as in the worked example) is just to show the mechanism; learning them is the power.

★ Deep dive: receptive fields, strides, and the architectures that made history▸

A key concept is the receptive field: the region of the original image that influences a given deep-layer neuron. A single 3×3 filter sees only 3×3 pixels, but stack two and the second layer's neurons effectively see 5×5 of the input; stack many (with pooling), and deep neurons "see" large portions of the image — which is how they can respond to whole objects despite each filter being tiny. This is why depth matters for vision: it grows the receptive field so later layers integrate global context from local operations. Stride (how far the filter jumps each step) is an alternative to pooling for downsampling — a stride-2 convolution both filters and shrinks in one step, common in modern architectures.

The history is worth knowing because the names come up constantly. LeNet (1998) first showed conv+pool for digit recognition. AlexNet (2012) scaled it up with ReLU, dropout, and GPUs and crushed the ImageNet competition, igniting the deep-learning boom. VGG showed that stacking many small 3×3 convolutions works beautifully. ResNet (2015) added "skip connections" that let gradients bypass layers, finally making networks hundreds of layers deep trainable (it directly addresses the vanishing-gradient problem of the next lesson) — and residual connections are now everywhere, including transformers. More recently, Vision Transformers (Track 5) challenged CNNs' dominance by applying attention to image patches, though CNNs remain excellent and efficient, especially with limited data. The throughline: each advance was about getting useful gradients through deeper stacks — exactly the training-dynamics theme we turn to next.

CNNs handle grids like images. But language, audio, and time series are sequences, where order and memory matter. Next, in 4.7 Sequence models: RNNs, LSTMs, GRUs, we build networks that carry information through time.
