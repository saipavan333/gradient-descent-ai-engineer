MLPs, activations & initialization

One neuron is a line. Stack them into layers, separated by nonlinear activations, and you get a multi-layer perceptron — a universal function approximator. Here’s how the pieces assemble, why the nonlinearity is non-negotiable, and why the humble choice of starting weights can make or break training.

Track 4 · Lesson 3 Concept + code ~28 min Builds on: Lessons 4.0–4.2, Track 1

💡 Concept: from neuron to network

A layer is a row of neurons that all see the same input but have their own weights. A multi-layer perceptron (MLP) chains layers: the outputs of one become the inputs of the next. The first layer reads the raw input, the last produces the prediction, and the layers in between are hidden — they learn intermediate representations.

Because every neuron in a layer computes a dot product, a whole layer is a matrix–vector multiply (Track 1): stack the neurons’ weight vectors as the rows of a matrix \(W\), and the layer computes \(\mathbf{y} = f(W\mathbf{x} + \mathbf{b})\). This is why GPUs — built for matrix multiplication — are the engine of deep learning, and why everything you learned about matrices in Track 1 pays off directly here.

📐 Activations: the indispensable bend

We proved it back in Track 1: without a nonlinearity between layers, a deep stack collapses to a single linear map (\(W_2 W_1 \mathbf{x}\)) and can only draw straight boundaries. The activation function is the bend that breaks this collapse and unlocks depth. The common choices:

ActivationFormulaRangeNotes
Sigmoid\(1/(1+e^{-z})\)\((0,1)\)saturates; used for probabilities/gates
Tanh\(\tanh z\)\((-1,1)\)zero-centered, but still saturates
ReLU\(\max(0,z)\)\([0,\infty)\)no positive saturation; the deep-net default
GELU\(\approx z\,\sigma(1.7z)\)\((-,\infty)\)smooth ReLU; standard in transformers

🎯 Initialization: where you start matters

Before training, the weights must be set to something. This seemingly minor choice is surprisingly decisive. Two failure modes from Lesson 1’s vanishing/exploding analysis: if initial weights are too large, pre-activations are huge, activations saturate, gradients vanish, and the network is stuck before it starts. If too small, signals shrink layer by layer toward zero and again nothing learns. And you must break symmetry: if every weight starts identical, every neuron in a layer computes the same thing and receives the same gradient forever, so they never differentiate — you must start with small random weights.

🔑 The initialization rule

Scale the random initial weights so the signal’s variance is preserved as it flows through each layer. The standard recipe is to draw weights with standard deviation \(\propto 1/\sqrt{n_{\text{in}}}\), where \(n_{\text{in}}\) is the number of inputs to the neuron — Xavier/Glorot initialization (for tanh/sigmoid) and He initialization (for ReLU, with a factor of \(\sqrt{2}\)). The intuition: a neuron summing \(n_{\text{in}}\) inputs would otherwise have its variance grow with \(n_{\text{in}}\), so we divide it back out. Get this right and deep networks train; get it wrong and they don’t.

🛠️ From scratch: an MLP on our engine

Built on the Value engine from Lesson 2, an MLP is just a few small classes. Each Neuron holds Value weights; the engine differentiates the whole thing for free.

code/track-4/train_moons.py — the network
import random

class Neuron:
    def __init__(self, nin):
        # small random weights break symmetry (init matters!)
        self.w = [Value(random.uniform(-1, 1)) for _ in range(nin)]
        self.b = Value(0.0)
    def __call__(self, x):
        act = sum((wi*xi for wi, xi in zip(self.w, x)), self.b)  # w·x + b
        return act.tanh()                                         # nonlinearity
    def parameters(self): return self.w + [self.b]

class Layer:
    def __init__(self, nin, nout):
        self.neurons = [Neuron(nin) for _ in range(nout)]
    def __call__(self, x):
        outs = [n(x) for n in self.neurons]
        return outs[0] if len(outs) == 1 else outs
    def parameters(self): return [p for n in self.neurons for p in n.parameters()]

class MLP:
    def __init__(self, nin, nouts):
        sz = [nin] + nouts
        self.layers = [Layer(sz[i], sz[i+1]) for i in range(len(nouts))]
    def __call__(self, x):
        for layer in self.layers: x = layer(x)
        return x
    def parameters(self): return [p for layer in self.layers for p in layer.parameters()]

model = MLP(2, [8, 8, 1])             # 2 inputs -> 8 -> 8 -> 1 output
print("parameters:", len(model.parameters()))   # 105 Values, all differentiable

🔑 Key results to remember

Lock these in

  1. A layer = many neurons sharing an input = a matrix multiply \(\mathbf{y}=f(W\mathbf{x}+\mathbf{b})\). An MLP chains layers; hidden layers learn representations.
  2. Activations are essential: without them, depth collapses to one linear layer (Track 1). ReLU avoids positive-side saturation → mitigates vanishing gradients.
  3. Sigmoid/tanh saturate — flat slope at the extremes kills gradients in deep stacks.
  4. Initialization matters: small random weights (break symmetry), scaled \(\propto 1/\sqrt{n_\text{in}}\) (Xavier/He) to keep signal variance stable.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. What goes wrong if you initialize all weights in a layer to the same value (say 0.1)?

  • The weights become too large.
    Magnitude isn’t the issue here; \(0.1\) is small. The problem is that the weights are identical.
  • Every neuron computes the same thing and gets the same gradient, so they never differentiate.
    Correct. Identical weights ⇒ identical outputs ⇒ identical gradients ⇒ they update in lockstep forever. This “symmetry” must be broken with random initialization.
  • Training is faster.
    It’s effectively broken: the layer behaves like a single neuron no matter how many it has. That’s not faster, it’s crippled.
  • Nothing — it’s fine.
    It’s a classic failure. Symmetric initialization wastes the entire layer’s capacity.

Q2. Why did ReLU largely replace sigmoid/tanh as the default hidden activation?

  • It’s smoother.
    ReLU is actually not smooth (it has a kink at 0); sigmoid/tanh are smoother. Smoothness isn’t the advantage.
  • It outputs probabilities.
    That’s sigmoid’s niche (range \((0,1)\)). ReLU outputs \([0,\infty)\), not probabilities.
  • It’s bounded.
    ReLU is unbounded above. Boundedness is a property of sigmoid/tanh, not a ReLU advantage.
  • Its derivative is 1 for positive inputs, so it doesn’t shrink gradients in deep stacks.
    Correct. No positive-side saturation means gradients pass through deep networks without vanishing — the key reason ReLU enabled training very deep models.

Q3. A fully-connected layer computing \(\mathbf{y}=f(W\mathbf{x}+\mathbf{b})\) relies on which Track-1 operation for \(W\mathbf{x}\)?

  • Eigen-decomposition.
    No eigen-decomposition in a forward pass; that’s for analyzing matrices (PCA), not applying a layer.
  • The determinant.
    Determinants measure volume scaling; they aren’t used to compute a layer’s output.
  • Matrix–vector multiplication.
    Correct. Stacking the neurons’ weight vectors as rows of \(W\), the layer’s pre-activations are \(W\mathbf{x}\) — a matrix–vector product (Track 1, Lesson 5).
  • Vector addition only.
    Addition handles the bias \(+\mathbf{b}\), but the core \(W\mathbf{x}\) is a matrix–vector multiply, not just addition.

Q4. The recommended weight-init standard deviation scales like \(1/\sqrt{n_\text{in}}\). Why divide by \(\sqrt{n_\text{in}}\)?

  • To keep the variance of a neuron’s pre-activation stable as the number of inputs grows.
    Correct. Summing \(n_\text{in}\) independent terms multiplies variance by \(n_\text{in}\); scaling weights by \(1/\sqrt{n_\text{in}}\) divides it back out, keeping signals well-scaled across layers.
  • To make weights larger for bigger layers.
    It makes them smaller for bigger fan-in (dividing by a larger number), the opposite.
  • To guarantee convergence.
    Good init helps a lot but guarantees nothing on a non-convex loss; it’s about signal scaling, not a convergence proof.
  • It’s an arbitrary convention.
    It’s principled, derived from preserving variance through a sum of \(n_\text{in}\) terms — not arbitrary.

✍️ Practice

Problem 1 · count

How many learnable parameters does an MLP with layer sizes \([2, 8, 8, 1]\) have? (Each neuron has one weight per input plus one bias.)

Show solution

Count per layer (a layer from \(n_\text{in}\) to \(n_\text{out}\) has \(n_\text{out}\times(n_\text{in}+1)\) params — weights plus one bias each). Layer 1 (\(2\to8\)): \(8\times(2+1)=24\). Layer 2 (\(8\to8\)): \(8\times(8+1)=72\). Layer 3 (\(8\to1\)): \(1\times(8+1)=9\). Total \(24+72+9 = 105\) — exactly what len(model.parameters()) prints. Tiny by modern standards (GPT-scale models have billions), but the counting rule is identical at every scale.

Problem 2 · reasoning

A 20-layer network with sigmoid activations trains painfully slowly, with the early layers barely changing. Diagnose the problem and suggest two fixes.

Show solution

Diagnosis: vanishing gradients. Sigmoid’s derivative is at most \(0.25\) and near zero when saturated; the gradient reaching layer 1 is a product of ~20 such factors, e.g. \(0.25^{20}\approx 10^{-12}\) — effectively zero, so early layers don’t learn (Lesson 1’s long-product problem). Fixes: (1) switch hidden activations to ReLU (derivative 1 on the positive side, no saturation); (2) use proper initialization (He) to keep signal variance stable; and beyond those, normalization layers (batch/layer norm) and residual connections (Track 5) — all of which exist largely to keep gradients flowing through deep stacks. The fact that a one-line activation change can rescue a dead network shows how much these “details” matter.

Problem 3 · code

Run activations.py and read off sigmoid′ and tanh′ at \(x=10\). What do the near-zero values imply for a deep network using these activations?

Show solution

You’ll see sigmoid′(10) ≈ \(0.00005\) and tanh′(10) ≈ \(10^{-8}\) — essentially zero. In a deep network, any neuron whose input lands in this saturated region passes almost no gradient backward, and since backprop multiplies these along the depth, the upstream layers get vanishingly small gradients and stop learning. This is the quantitative face of the vanishing-gradient problem and the concrete reason saturating activations fell out of favor for deep hidden layers (they survive in specific roles: sigmoid for output probabilities and gates, tanh in some recurrent cells).

Problem 4 · reasoning

The “universal approximation theorem” says an MLP with one hidden layer can approximate any continuous function. Why, then, do we use deep networks instead of one very wide hidden layer?

Show solution

Universal approximation is an existence result: a single wide layer can represent the function, but it may need an astronomically large number of neurons to do so, and finding those weights by gradient descent may be hopeless. Depth is about efficiency. Deep networks build functions compositionally — each layer reuses and combines features from the previous one — so they can represent many natural functions with exponentially fewer parameters than a shallow network would need. Depth also matches the hierarchical structure of real data (edges → shapes → objects; letters → words → meaning). So the theorem tells us shallow nets are sufficient in principle, while practice shows deep nets are vastly more efficient — a recurring theme: representability and learnability are different questions (echoing Track 3’s VC story).
