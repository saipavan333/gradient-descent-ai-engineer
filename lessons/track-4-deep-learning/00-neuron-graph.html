The neuron & the computational graph

Everything you’ve built — vectors, dot products, gradients, loss — now snaps together into the atom of deep learning: the neuron. And the way we’ll string neurons into trainable networks is an idea of startling generality: the computational graph.

Track 4 · Lesson 0 Concept + code ~24 min Builds on: Tracks 1–3

Track 4 is where it all becomes real. Over the next five lessons you will build a working automatic-differentiation engine from scratch, derive backpropagation, and train a neural network that learns a curved decision boundary — using only the math from Tracks 1–3. We start with the single neuron and the data structure that makes training possible.

💡 Concept: a neuron is everything you already know

A neuron is almost insultingly simple — and you already have every piece. It takes an input vector \(\mathbf{x}\), forms a weighted sum with a weight vector \(\mathbf{w}\) (a dot product, Track 1), adds a bias \(b\) to shift the result, and passes that through a nonlinear activation function \(f\):

$$ z = \mathbf{w}\cdot\mathbf{x} + b = \sum_i w_i x_i + b, \qquad y = f(z). $$
(★)

That’s it. The weights decide how much each input matters; the bias sets the neuron’s baseline tendency to fire; the activation bends the output (we’ll see in Lesson 3 why that bend is essential). A neuron with no activation is just linear regression (Track 2). A network is many neurons wired together — and the wiring is best understood as a graph.

📐 The computational graph: every computation is a graph

Here is the idea that makes modern deep learning possible. Any numerical computation — no matter how complex — can be broken into a sequence of elementary operations (add, multiply, tanh, …), and drawn as a computational graph: a directed graph whose nodes are intermediate values and whose edges are the operations that produced them. Take a tiny example, \(e = a\times b\) then \(d = e + c\):

Why bother drawing arithmetic as a graph? Because the graph records exactly how every value was computed — and that record is precisely what we need to run the chain rule backward and get the gradient of the output with respect to every input automatically. That backward pass is backpropagation (Lesson 1), and the engine that builds the graph and runs it is autograd (Lesson 2). A neural network — even a billion-parameter transformer — is just an enormous computational graph, and training it is forward pass, then backward pass, then a gradient step, repeated. Master this small graph and you’ve grasped the skeleton of all of it.

💡 Two passes, one graph

The forward pass walks the graph from inputs to output, computing values — this is the network making a prediction. The backward pass walks the same graph from output to inputs, computing gradients — this is the network figuring out how to improve. Same structure, two directions. Everything in Track 4 is built on this duality.

🛠️ From scratch: a neuron and a graph

code/track-4/neuron_forward.py
import numpy as np

def neuron(x, w, b, activation=np.tanh):
    z = np.dot(w, x) + b          # weighted sum + bias (a dot product, Track 1)
    return activation(z)

x = np.array([2.0, 0.0, -1.0])    # inputs
w = np.array([-3.0, 1.0, 0.5])    # weights
b = 6.0                           # bias
z = np.dot(w, x) + b              # -3*2 + 1*0 + 0.5*-1 + 6 = -0.5
print("z =", z, "  y = tanh(z) =", round(np.tanh(z), 4))   # y ~ -0.4621

# A tiny computational graph, built by hand: e = a*b, then d = e + c
a, b2, c = 2.0, -3.0, 10.0
e = a * b2          # multiply node
d = e + c           # add node
print("e =", e, " d =", d)        # -6, 4

Run it and change a weight: predict whether \(y\) rises or falls before you press enter. Right now we build the graph implicitly (Python computes \(e\) then \(d\) and forgets the structure). In Lesson 2 we’ll make the graph an explicit object that remembers — so it can differentiate itself.

🔑 Key results to remember

Lock these in

  1. A neuron computes \(y = f(\mathbf{w}\cdot\mathbf{x} + b)\): a dot product (Track 1), a bias shift, and a nonlinear activation.
  2. Weights \(\mathbf{w}\) and bias \(b\) are the learnable parameters; the activation \(f\) supplies the nonlinearity.
  3. Any computation is a computational graph — nodes are values, edges are operations.
  4. Forward pass: inputs → output (prediction). Backward pass: output → inputs (gradients). Same graph, two directions.
  5. A neural network is a big computational graph; training = forward, backward, gradient step, repeat.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. A neuron computes \(y=f(\mathbf{w}\cdot\mathbf{x}+b)\). Which Track-1 operation is the \(\mathbf{w}\cdot\mathbf{x}\) part?

  • The cross product.
    There’s no cross product here (and cross products return vectors). The weighted sum of inputs is a single number.
  • The dot product.
    Correct. \(\mathbf{w}\cdot\mathbf{x}=\sum_i w_i x_i\) is exactly the dot product from Track 1 — the weighted sum at the heart of every neuron.
  • The determinant.
    Determinants act on square matrices and measure volume scaling. A neuron’s pre-activation is a dot product, not a determinant.
  • Matrix inversion.
    No inverse is involved in a forward pass; it’s a sum of products plus a bias.

Q2. In the graph for \(d = a\times b + c\) with \(a=2, b=-3, c=10\), what is the forward value of \(d\)?

  • \(9\)
    That would be \(a+b+c\)? No — \(2+(-3)+10=9\), but the graph multiplies \(a\) and \(b\) first, it doesn’t add them.
  • \(-60\)
    That multiplies everything: \(2\times-3\times10\). But the graph adds \(c\) after the product, not multiplies by it.
  • \(4\)
    Correct. \(e = a\times b = -6\), then \(d = e + c = -6 + 10 = 4\). Follow the graph’s operations in order.
  • \(16\)
    Recheck the product’s sign: \(2\times(-3) = -6\), not \(+6\). With \(-6\), \(d = 4\).

Q3. What does the forward pass through a computational graph correspond to, in ML terms?

  • The network making a prediction.
    Correct. Forward = compute values from inputs to output = run the model to produce its output/prediction.
  • The network updating its weights.
    Weight updates happen after the backward pass, in the optimizer step — not during the forward pass.
  • Computing the gradients.
    Gradients come from the backward pass. The forward pass only computes values.
  • Loading the data.
    Data loading is a separate step; the forward pass is the computation that turns inputs into an output.

Q4. Why represent a computation as a graph rather than just evaluating it?

  • Graphs run faster.
    A graph isn’t inherently faster to evaluate forward; the benefit is what it enables afterward.
  • To avoid using a dot product.
    The graph still uses dot products and other ops; it organizes them, it doesn’t replace them.
  • Graphs use less memory.
    Storing the graph (for the backward pass) actually uses more memory than a bare evaluation. Memory isn’t the motivation.
  • The recorded structure lets us run the chain rule backward to get gradients automatically.
    Correct. The graph remembers how each value was produced, which is exactly what backpropagation needs to compute every gradient — the foundation of training.

✍️ Practice

Problem 1 · compute

A neuron has weights \(\mathbf{w}=(1, -2)\), bias \(b=0.5\), and ReLU activation. What is its output for input \(\mathbf{x}=(3, 1)\)? For \(\mathbf{x}=(0, 2)\)?

Show solution

For \(\mathbf{x}=(3,1)\): \(z = 1\cdot3 + (-2)\cdot1 + 0.5 = 3 - 2 + 0.5 = 1.5\); ReLU\((1.5)=\max(0,1.5)=1.5\). For \(\mathbf{x}=(0,2)\): \(z = 1\cdot0 + (-2)\cdot2 + 0.5 = -3.5\); ReLU\((-3.5)=\max(0,-3.5)=0\) — the neuron is “off.” ReLU lets a neuron stay silent (output 0) for inputs that don’t excite it, which is a big part of why deep ReLU networks are sparse and trainable.

Problem 2 · draw

Draw the computational graph for \(L = (a + b)\times(b + c)\). Then compute the forward value at \(a=1, b=2, c=3\).

Show solution

The graph: \(a,b\) feed an add node → \(u = a+b\); \(b,c\) feed another add node → \(v = b+c\); then \(u,v\) feed a multiply node → \(L = u\times v\). Note \(b\) is used twice (it has two outgoing edges) — that matters for backprop later (its gradient will accumulate from both paths). Forward at \(a=1,b=2,c=3\): \(u = 3\), \(v = 5\), \(L = 15\).

Problem 3 · code

Modify neuron_forward.py: keep the same inputs/weights but change the bias from 6.0 to 0.0. Predict the new \(z\) and \(y\), then verify.

Show solution

With \(b=0\): \(z = -3\cdot2 + 1\cdot0 + 0.5\cdot(-1) + 0 = -6.5\), and \(y = \tanh(-6.5) \approx -1.0\) (tanh saturates near \(-1\) for large negative inputs). Dropping the bias from 6 to 0 pushed \(z\) from \(-0.5\) to \(-6.5\), flipping the neuron from “barely negative” to “fully off.” The bias is a powerful knob — it sets where the activation’s transition happens.

Problem 4 · reasoning

Why is a neuron with no activation function (just \(y = \mathbf{w}\cdot\mathbf{x}+b\)) unable to benefit from being stacked into deep layers?

Show solution

Without an activation, a neuron is a linear (affine) function. Stacking linear layers composes to a single linear function — exactly the argument from Track 1, Lesson 6: \(W_2(W_1\mathbf{x}) = (W_2 W_1)\mathbf{x}\). So a deep stack of activation-free neurons collapses to one linear layer and can only draw straight decision boundaries, no matter how deep. The nonlinear activation \(f\) is what breaks this collapse and lets depth add real expressive power. This is why activations (Lesson 3) are non-negotiable — and it’s a direct payoff of the linear-algebra you built in Track 1.
