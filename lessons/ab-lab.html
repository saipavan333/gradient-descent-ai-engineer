A/B Test Lab

Offline metrics are a proxy; the ground truth is a live experiment. Ship variant B to half your users, keep A for the rest, and ask the hard question: is B really better, or did you just get lucky? This lab runs that experiment and does the statistics for you.

Set a real improvement (or none at all), choose how many users you test, and press Run. You'll get observed conversion rates, a confidence interval, and a p-value. Run it a few times with a tiny sample and watch the verdict flip-flop — that randomness is exactly why you need statistics, not eyeballs.

▶ Run the experiment

Bars are observed conversion; whiskers are 95% confidence intervals. If the intervals barely overlap and p < 0.05, the difference is unlikely to be chance.

i
What the p-value means. It's the probability of seeing a gap this big if B were actually no better than A. Small p (< 0.05) means "chance alone rarely produces this" → we call it significant. But note the trap: with a huge sample, even a meaningless 0.1% lift becomes "significant"; with a tiny sample, a real lift can hide in the noise. Significance is not the same as importance.

Try these

1. The A/A test. Set the true lift to 0 — B is identical to A. Run it 100×. About 5% of the time it still reports "significant". That false-positive rate is exactly the 0.05 threshold you chose, and it's why you don't peek and stop early.

2. Underpowered. Keep a real +2% lift but set users to 100. Run repeatedly — sometimes significant, mostly not. The effect is real but your sample is too small to see it reliably. That's low statistical power.

3. Power up. Now slide users to 20,000. The confidence intervals shrink, the same +2% lift becomes reliably significant. More data doesn't change the truth — it sharpens your ability to detect it.

What this lab proves
  • Observed ≠ true. Each experiment is a noisy sample; the confidence interval shows how noisy.
  • p < 0.05 = unlikely by chance. But ~5% of null experiments cross it anyway — real false positives.
  • Sample size is power. More users → tighter intervals → smaller real effects become detectable.
  • Significant ≠ important. A tiny lift can be significant at scale; always check the effect size too.
