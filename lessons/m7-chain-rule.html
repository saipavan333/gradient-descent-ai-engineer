The chain rule — backprop's engine

⏱ 24 min Lesson 1.7 ▶ Open in Colab
Why this existsA neural network is functions inside functions. To train it you need the derivative of the whole stack, and the chain rule gets it by multiplying simple local slopes — the literal engine of backpropagation.
By the end, you can
  • Apply the chain rule to a composed function f(g(x))
  • Extend it to multi-step chains as 'multiply the local slopes'
  • Trace derivatives through a computational graph
  • See why this makes training deep networks tractable

Concept Why one little rule rules all of deep learning

A neural network is a tall stack of steps: the input goes into step 1, its output feeds step 2, that feeds step 3, and so on, until the last step produces a prediction and an error. In math, a stack like this is called a composition — functions inside functions, written $f(g(x))$ (read "f of g of x": first do $g$ to $x$, then do $f$ to the result). To train the network we need the derivative of the whole stack — how the final error changes when we nudge a number buried deep inside. The tool that gets it is the chain rule, and essentially all of deep learning's training rests on it.

Concept The chain rule: multiply the local slopes

Suppose $y$ depends on $u$, and $u$ in turn depends on $x$. How does $y$ change when $x$ changes? The chain rule says you multiply the rate of change at each link in the chain:

$$ \frac{dy}{dx} \;=\; \frac{dy}{du}\,\cdot\,\frac{du}{dx}. $$

Read it as: "how $y$ changes per change in $x$" equals "how $y$ changes per change in $u$" times "how $u$ changes per change in $x$." Each piece, like $\frac{dy}{du}$, is just a derivative (a slope) — we call them local slopes because each measures one link of the chain on its own.

The everyday intuition is gears: if gear $x$ turns gear $u$ at 3× speed, and gear $u$ turns gear $y$ at 2× speed, then $x$ turns $y$ at $3 \times 2 = 6$× speed. Rates of change compose by multiplication. Here's the smallest real example. Let $y = (3x + 1)^2$. Break it into an inner and an outer piece: let $u = 3x + 1$, so $y = u^2$. Then $\dfrac{dy}{du} = 2u$ (power rule) and $\dfrac{du}{dx} = 3$ (the derivative of $3x+1$). Multiply them: $\dfrac{dy}{dx} = 2u \cdot 3 = 6u = 6(3x+1)$. At $x = 1$ that's $6(4) = 24$. We differentiated a composed function without ever expanding the square — by splitting it into links and multiplying their slopes.

Concept Computational graphs and backpropagation

Real models have many links and many parameters, so we organize the computation as a computational graph: a diagram of the operations, with values flowing forward (left to right) to produce the output, and derivatives flowing backward (right to left) to produce the gradient. The forward trip is the forward pass; the backward trip is the backward pass. Applying the chain rule across this graph — from the output back to every input — is exactly what backpropagation ("backprop") is.

The crucial efficiency: the backward pass reuses the values already computed in the forward pass and multiplies local derivatives as it goes, so it gets the gradient for every parameter in roughly the same time as one forward pass. That is why training networks with billions of parameters is even possible.

i
Backprop is not a new kind of math

It often sounds mysterious, but backpropagation is just the chain rule applied systematically across a computational graph, remembering intermediate results so nothing is computed twice. If you understand "multiply the local slopes along the path," you understand backprop. The framework handles the bookkeeping for you.

Worked example Backprop by hand, then checked by autograd

Take a tiny "one neuron + squared error" computation with weight $w=2$, input $x=3$, bias $b=1$. We go forward to get the numbers, then backward to get a gradient:

$$ z = w\,x = 6, \qquad a = z + b = 7, \qquad L = a^2 = 49. $$

We want $\dfrac{\partial L}{\partial w}$ — how the loss $L$ changes if we nudge the weight $w$. Walk the chain backward from $L$ to $w$, multiplying the local derivative at each link:

$$ \frac{\partial L}{\partial w} = \underbrace{\frac{\partial L}{\partial a}}_{2a\,=\,14}\cdot \underbrace{\frac{\partial a}{\partial z}}_{1}\cdot \underbrace{\frac{\partial z}{\partial w}}_{x\,=\,3} = 14 \cdot 1 \cdot 3 = 42. $$

(Each link: $L = a^2$ so $\partial L/\partial a = 2a = 14$; $a = z+b$ so $\partial a/\partial z = 1$; $z = wx$ so $\partial z/\partial w = x = 3$. Multiply: $14 \times 1 \times 3 = 42$.)

import torch

w = torch.tensor(2.0, requires_grad=True)
x = torch.tensor(3.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)

z = w * x
a = z + b
L = a**2
L.backward()                 # backprop: apply the chain rule across the graph

print("dL/dw:", w.grad.item())   # 42.0
print("dL/db:", b.grad.item())   # 14.0
print("dL/dx:", x.grad.item())   # 28.0
OutputdL/dw: 42.0 dL/db: 14.0 dL/dx: 28.0

Autograd reproduces our hand calculation exactly: $\partial L/\partial w = 42$. Check the other two with the chain rule yourself — $\partial L/\partial b = 2a \cdot 1 = 14$, and $\partial L/\partial x = 2a \cdot w = 14 \cdot 2 = 28$ (here $x$'s path to $L$ runs through the extra factor $w$, which is why its gradient is bigger). Every gradient is just a product of local slopes along the path from $L$ back to that input. Scale this graph up to millions of nodes and you have the training of a modern neural network.

★ Key points to remember
  • A network is a composition of functions; training needs the derivative of the whole stack.
  • The chain rule multiplies the local rates: $\dfrac{dy}{dx} = \dfrac{dy}{du}\cdot\dfrac{du}{dx}$.
  • Organize the computation as a computational graph: values flow forward, gradients flow backward.
  • Backpropagation = the chain rule applied across that graph, reusing forward values for speed.
  • A gradient with respect to any parameter is a product of local derivatives along the path from the loss to that parameter.

Practice Build the reasoning yourself

1Use the chain rule to differentiate $y = e^{5x}$. (Recall the derivative of $e^u$ is $e^u$.)
Show solution & reasoning

Let $u = 5x$, so $y = e^u$. Then $\frac{dy}{du} = e^u$ and $\frac{du}{dx} = 5$. Multiply: $\frac{dy}{dx} = e^u \cdot 5 = 5e^{5x}$.

The pattern to internalize: for $e^{(\text{something})}$, the derivative is $e^{(\text{something})}$ times the derivative of that something. The "5" came from the inner link and would have been silently dropped without the chain rule.

2A 3-link chain has local slopes $\frac{dy}{du}=2$, $\frac{du}{dv}=3$, $\frac{dv}{dx}=4$. Find $\frac{dy}{dx}$. Then: if instead every link were $0.5$, what would a 10-link chain's overall slope be — and what does that hint about very deep networks?
Show solution & reasoning

Three links: $\frac{dy}{dx} = 2 \times 3 \times 4 = 24$ (multiply all the local slopes along the path).

Ten links of $0.5$: $0.5^{10} = 1/1024 \approx 0.00098$ — almost zero. This is the vanishing-gradient problem: when many small local slopes multiply, the gradient that reaches the earliest layers becomes vanishingly tiny, so those layers barely learn. (If the links were each $>1$, the product would explode instead.) Deep networks need careful activation functions and tricks like normalization and residual connections — Track 4 — precisely to keep these products from collapsing or blowing up. You just predicted a famous real-world failure mode from the chain rule alone.

3Explain, in plain language, why the backward pass can compute the gradient for thousands of parameters in about the same time as a single forward pass.
Show solution & reasoning

Because backprop reuses work. The forward pass computes and stores every intermediate value once. The backward pass then sweeps from the loss back to the inputs a single time, and at each operation it just multiplies the incoming gradient by that operation's local slope — cheap arithmetic on values it already has. Crucially, the gradient arriving at a shared node is computed once and handed to all the parameters feeding it, so nothing is recomputed.

The reasoning: the cost is one forward sweep plus one backward sweep, regardless of how many parameters there are — it scales with the size of the graph, not with the number of parameters separately. Computing each parameter's gradient independently (re-running the network per parameter) would be millions of times slower. This reuse is the entire reason large-scale training is feasible.

★ Deep dive: forward-mode vs reverse-mode autodiff (why backprop goes backward)▸

There are two ways to apply the chain rule mechanically through a graph, and the choice is about efficiency, not correctness.

Forward mode propagates derivatives forward, alongside the values, answering "how does each intermediate change as one chosen input changes?" It's efficient when you have few inputs and many outputs.

Reverse mode (backpropagation) does one forward pass to compute values, then propagates derivatives backward, answering "how does the one chosen output (the loss) change as each intermediate changes?" It's efficient when you have many inputs and one output.

Machine learning is the second situation exactly: a model has millions of parameters (inputs) but a single number for the loss (output). Reverse mode gets the gradient for all parameters in one backward sweep, whereas forward mode would need a separate pass per parameter. That asymmetry — many knobs, one error — is why every deep-learning framework uses reverse-mode autodiff. The price is memory: you must store the forward-pass intermediate values to use them on the way back, which is why training a big model needs far more memory than just running it. That trade — compute the gradient cheaply by remembering the forward pass — is a theme you'll meet again in Tracks 4 and 7.
