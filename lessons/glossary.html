Gradient Descent

Glossary — every key term in one place

How to use thisEvery term below is defined in full in the lesson where it first appears — this page is a quick reference to look one up later, grouped by area. If a definition here feels thin, the lesson gives the intuition, a worked example, and a diagram.

Foundations Math & Python

Vector — An ordered list of numbers; the basic way data and parameters are represented.

Matrix — A grid of numbers; applying one is a linear transformation, and a matrix multiply is the core operation of ML.

Gradient — The vector of partial derivatives — which way, and how steeply, a function increases; points the way to adjust parameters.

Gradient descent — The learning algorithm: repeatedly step parameters opposite the gradient to reduce error.

Learning rate — The size of each gradient-descent step; too big diverges, too small crawls.

Probability distribution — A description of how likely each outcome is; models output and sample from these.

Bayes' rule — How to update a belief given new evidence — the backbone of many ML methods.

Maximum likelihood — Choosing parameters that make the observed data most probable; justifies common loss functions.

Vectorization — Replacing Python loops with array operations (NumPy) so numerical code runs fast.

Data leakage — When information from the test set (or the future) sneaks into training, inflating scores and ruining real performance.

Classical ML Core ideas & metrics

Supervised learning — Learning a mapping X → y from labeled examples.

Overfitting — Fitting the training data so closely (including its noise) that the model fails on new data.

Bias–variance tradeoff — Error splits into bias (too-simple/underfit) and variance (too-sensitive/overfit); complexity trades one for the other.

Regularization — Penalizing complexity (L1/L2, dropout, early stopping) to reduce overfitting.

Cross-validation — Rotating which fold is held out to estimate performance and tune without touching the test set.

Precision — Of the items predicted positive, the fraction that truly were: TP/(TP+FP).

Recall — Of the actual positives, the fraction caught: TP/(TP+FN).

F1 score — The harmonic mean of precision and recall — a single balanced metric.

Confusion matrix — The table of true/false positives and negatives from which precision and recall are computed.

Ensemble — Combining many models (e.g., a random forest) so their independent errors cancel — lowering variance.

Deep Learning Neural networks

Neuron — A weighted sum of inputs plus a bias, passed through a nonlinear activation.

Activation function — The nonlinearity (ReLU, sigmoid, tanh, GELU) that lets stacked layers model complex patterns.

Backpropagation — Computing every weight's gradient efficiently via the chain rule, in one backward pass.

Vanishing gradients — When gradients shrink toward zero through many layers, so early layers barely learn; fixed by ReLU, good init, normalization, residual connections.

CNN — Convolutional neural network — uses shared local filters, ideal for images (locality + translation invariance).

RNN — Recurrent neural network — processes sequences with a hidden state; largely superseded by transformers.

Dropout — Randomly zeroing units during training to prevent co-adaptation — a regularizer.

Transfer learning — Reusing a pretrained model and adapting only a small part to a new task — strong results from little data.

NLP & Transformers Language models' architecture

Token — The unit of text a model processes (a word or word-piece); text is split into tokens before modeling.

Embedding — A dense vector representing a token or item, where similar things are close together.

Self-attention — Letting every position attend to every other via query-key-value, capturing long-range relationships directly.

Transformer — The architecture built on stacked self-attention; the foundation of modern LLMs, prized for long-range modeling and parallelism.

Positional encoding — Information added to inputs so the order-agnostic transformer knows token order.

Context window — The maximum number of tokens a model can attend to at once — a hard limit on input and history.

LLMs & GenAI Large language models

Pretraining — Learning language by predicting the next token on massive text — where an LLM gains its capability.

RLHF — Reinforcement learning from human feedback — aligning a model with human preferences for helpfulness and safety.

Fine-tuning — Further training a model on task-specific data to shape its behavior or style.

LoRA — Low-Rank Adaptation — fine-tuning by training a tiny low-rank adapter while freezing the model; cheap and swappable.

Prompting — Steering a fixed model at inference time via the input (few-shot examples, chain-of-thought).

Chain-of-thought — Prompting the model to reason step by step, which improves performance on hard problems.

Temperature — A decoding knob for randomness — low for factual/precise output, high for creative.

Hallucination — When a model confidently generates false information, because it optimizes plausibility, not truth.

Quantization — Running a model at lower numerical precision to cut memory and speed up inference, with little accuracy loss.

RAG Retrieval-augmented generation

RAG — Retrieving relevant documents at query time and grounding the LLM's answer in them — for current, private, citable knowledge.

Vector database — A store of embeddings with fast approximate nearest-neighbor search, powering retrieval.

Chunking — Splitting documents into passages before embedding; you can only retrieve what your chunks contain.

Recall@k — Whether the relevant chunk is among the top-k retrieved — the key retrieval metric for RAG.

Reranking — A second, precise pass (often a cross-encoder) that reorders retrieved candidates for quality.

Hybrid search — Combining semantic (embedding) and keyword (BM25) retrieval so exact terms and meaning are both covered.

Faithfulness — Whether a generated answer is actually grounded in the retrieved context (vs hallucinated).

Train/serve skew — Features computed differently at training vs serving time — a silent accuracy killer.

Agents Tool-using systems

Agent — An LLM in a loop that reasons, calls tools, observes results, and repeats to accomplish multi-step tasks.

Tool use / function calling — Letting a model emit a structured request that your code executes — search, code, an API.

ReAct — Interleaving Thought → Action → Observation so tool use is deliberate and the trace is debuggable.

Guardrails — Independent checks around a model (permission allowlists, budgets, human approval) that enforce safety regardless of the model.

Prompt injection — Malicious instructions hidden in content a model reads, attempting to hijack it — an unsolved security problem.

Least privilege — Giving a system only the minimum access it needs, so a compromised agent can't cause serious harm.

MLOps & System Design Running AI in production

MLOps — The engineering of deploying, serving, monitoring, and maintaining ML in production — the lifecycle loop.

Feature store — A system that computes and serves features consistently for both training and serving, preventing skew.

Model registry — A versioned catalog of trained models with staged promotion and instant rollback.

Serving — Running a packaged model so it answers prediction requests — online (real-time), batch, or streaming.

Latency vs throughput — Time for one request vs total requests handled — online serving optimizes the first, batch the second.

Drift — When live data moves away from the training distribution, silently degrading a deployed model; fixed by retraining.

Autoscaling — Automatically adjusting the number of serving replicas with traffic — enough at peak, no waste off-peak.

Two-stage funnel — Cheap wide retrieval then expensive narrow ranking — how recommenders and search scale.

QPS — Queries per second — the request rate a system must handle, sized for the peak, not the average.

Caching — Reusing stored results for repeated requests; savings scale with the hit rate.

Evals & Safety Measuring and governing

Goodhart's law — When a measure becomes the target, it stops being a good measure — the trap of optimizing a single proxy.

Offline vs online evaluation — Testing on a fixed dataset (fast proxy) vs A/B testing on real users (ground truth).

A/B test — A randomized experiment comparing a new system against the current one on live traffic.

LLM-as-judge — Using a capable model to grade outputs at scale — validated against human labels first.

Disparate impact — When a model's outcomes disproportionately harm a group; the four-fifths rule is a common red-flag check.

Red-teaming — Deliberately attacking your own system to find failures and jailbreaks before real adversaries do.

PII — Personally Identifiable Information — data that must be minimized, protected, and often redacted.

Reproducibility — Getting the same result again by pinning code, data, environment, and the random seed.

Computer Vision Seeing, detecting, generating

Image tensor — An image as a height x width x channels grid of pixel numbers — all a model ever sees.

Convolution — Sliding a small learned filter over an image to detect a pattern anywhere it appears (weight sharing).

Feature map — The output of a convolution — where in the image a given filter's pattern was found.

Data augmentation — Random flips, crops, rotations, and color shifts that turn one image into many, improving generalization.

Object detection — Localizing and classifying each object — a bounding box plus a class label per object.

IoU — Intersection over Union — overlap area / union area of two boxes; a detection usually counts as correct at IoU >= 0.5.

Segmentation — Assigning a class to every pixel — the finest-grained vision task, giving exact shapes (semantic vs instance).

U-Net — An encoder-decoder with skip connections: compress to understand, expand to localize; used for masks and diffusion denoising.

Vision Transformer (ViT) — A transformer that treats image patches as tokens — 'an image is worth 16x16 words'.

Inductive bias — Assumptions built into an architecture; CNNs bake in locality/translation-invariance, ViTs assume less and need more data.

Diffusion model — A generator that learns to reverse noising: denoise from pure noise into a new image (Stable Diffusion, DALL-E).

Latent diffusion — Running diffusion in a compressed latent space (not raw pixels) to make text-to-image fast.

CLIP — A model that aligns images and text in one shared embedding space via contrastive learning — enabling zero-shot classification.

Multimodal / VLM — A model working across data types (images + text); modern vision-language models feed image tokens into an LLM.
