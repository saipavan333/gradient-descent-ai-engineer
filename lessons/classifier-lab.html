Four classic algorithms, one dataset, side by side. Drop your own points and watch how logistic regression, k-nearest-neighbours, a decision tree, and a small neural network each carve up the very same plane — completely differently.

There's no single "best" classifier — each makes different assumptions about what a boundary should look like. A straight line? A jagged staircase? A halo around each point? Draw a shape that fools one of them and see which others handle it. That intuition is what "choosing a model" actually means.

Same data, different beliefs. Logistic regression believes the boundary is a straight line. k-NN believes nearby points share a label, so its boundary hugs the data. A decision tree believes in axis-aligned rectangles, so it draws staircases. A neural net believes in smooth curves. None is "right" — the best choice depends on what shape your real data actually has, and how much of it you have.

Try these

1. The circle test. Load Circle and select Logistic Regression. It fails — no straight line separates an inner disk from an outer ring, so accuracy stalls. Switch to k-NN or Neural Net and the boundary wraps the circle. This is the linear-vs-nonlinear divide in one click.

2. Overfit a tree. On any dataset, crank the tree's depth up. The staircase gets more and more detailed until it traces individual points — perfect training accuracy, but it's memorising noise. Now do the same by dropping k-NN's k to 1.

3. Make one point matter. Add a single stray point of the wrong class in the middle of the other cluster. Watch k-NN grow a little island around it (sensitive), while logistic regression barely flinches (stubborn). That's the bias–variance trade-off, live.

What this lab proves
  • Every model has an inductive bias — an assumption about what boundaries look like (lines, staircases, halos, curves).
  • Linear models can't bend — logistic regression needs linearly separable data (or engineered features).
  • Flexibility cuts both ways — k-NN with k=1 and deep trees fit anything, including noise (overfitting).
  • Model choice is problem-dependent — match the model's bias to your data's real shape; there's no universal best.
