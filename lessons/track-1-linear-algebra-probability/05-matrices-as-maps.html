Matrices as linear transformations

Stop thinking of a matrix as a grid of numbers. It’s a function that moves space — and it’s completely described by where it sends the basis vectors. This single mental model makes matrices, neural-network layers, and everything after click into place.

Track 1 · Lesson 5 Concept + code ~30 min Builds on: Lessons 1–4

💡 Concept: a matrix is a function on vectors

A matrix is a rectangular array of numbers, like \(A=\begin{bmatrix}2 & 1\\ 0 & 1.5\end{bmatrix}\). We write \(a_{ij}\) for the entry in row \(i\), column \(j\). But its purpose is to act on vectors: feed a matrix a vector and it returns a new vector. In other words, a matrix is a function \(A:\mathbb{R}^n\to\mathbb{R}^m\). The whole lesson is about understanding what kind of function it is — and the answer is the nicest kind: a linear one that moves all of space in a rigid, grid-preserving way.

How does a matrix act on a vector? The operation is the matrix–vector product \(A\mathbf{x}\), and there are two equivalent ways to see it. The first is the one that builds real intuition.

The columns view (the one to internalize)

Read the matrix–vector product as a linear combination of the matrix’s columns, with the vector’s components as the weights. If \(A\) has columns \(\mathbf{a}_1,\mathbf{a}_2\) and \(\mathbf{x}=(x_1,x_2)\), then

$$ A\mathbf{x} \;=\; x_1\,\mathbf{a}_1 + x_2\,\mathbf{a}_2 \;=\; x_1\!\begin{bmatrix}a_{11}\\ a_{21}\end{bmatrix} + x_2\!\begin{bmatrix}a_{12}\\ a_{22}\end{bmatrix}. $$

Watch what this says about the basis vectors. Feeding in \(\mathbf{e}_1=(1,0)\) gives \(A\mathbf{e}_1 = 1\cdot\mathbf{a}_1 + 0\cdot\mathbf{a}_2 = \mathbf{a}_1\), the first column. Likewise \(A\mathbf{e}_2=\mathbf{a}_2\), the second column. So:

🔑 The one idea that unlocks matrices

The columns of \(A\) are exactly where the basis vectors land after the transformation. To build the matrix for any linear map, just answer: “where does \(\mathbf{e}_1\) go? where does \(\mathbf{e}_2\) go?” Stack those answers as columns — that’s your matrix.

The rows view (the one you compute with)

The same product can be read row-by-row: entry \(i\) of \(A\mathbf{x}\) is the dot product of row \(i\) of \(A\) with \(\mathbf{x}\):

$$ (A\mathbf{x})_i \;=\; \sum_{j=1}^{n} a_{ij}\,x_j \qquad\text{(row } i \text{ of } A,\ \text{dotted with } \mathbf{x}). $$

For our example, \(A\mathbf{x}=\begin{bmatrix}2&1\\0&1.5\end{bmatrix}\begin{bmatrix}x_1\\x_2\end{bmatrix}=\begin{bmatrix}2x_1+1x_2\\ 0x_1+1.5x_2\end{bmatrix}\). The two views give identical numbers — the columns view tells you what the matrix means, the rows view is how your hand (and the CPU) actually grinds it out. Use both.

📐 What makes it “linear”

A transformation \(T\) is linear if it respects our two operations from Lesson 2 — adding then transforming equals transforming then adding, and scaling then transforming equals transforming then scaling:

$$ T(\mathbf{u}+\mathbf{v}) = T(\mathbf{u}) + T(\mathbf{v}), \qquad T(c\,\mathbf{u}) = c\,T(\mathbf{u}). $$

Every matrix product satisfies these (you’ll prove it in practice), and — strikingly — every linear transformation can be written as a matrix. Matrices and linear maps are two names for one thing. Visually, linearity has a crisp signature: straight grid lines stay straight, parallel lines stay parallel and evenly spaced, and the origin doesn’t move. The grid may be stretched and sheared into a parallelogram lattice, but it never curves:

📐 Non-square matrices change the dimension

An \(m\times n\) matrix (\(m\) rows, \(n\) columns) maps \(\mathbb{R}^n\to\mathbb{R}^m\): it eats an \(n\)-vector and outputs an \(m\)-vector. The columns view makes the bookkeeping obvious — there are \(n\) columns (one weight per input component), and each column lives in \(\mathbb{R}^m\) (the output space). A \(3\times 2\) matrix takes plane vectors to space; a \(2\times 3\) matrix flattens space vectors onto the plane. The shapes must line up: for \(A\mathbf{x}\) to make sense, the number of columns of \(A\) must equal the dimension of \(\mathbf{x}\).

⚠️ The shape rule, once and for all

\((m\times n)\) times \((n\times 1)\) gives \((m\times 1)\). The inner numbers (\(n\) and \(n\)) must match and cancel; the outer numbers (\(m\) and \(1\)) survive as the output shape. Ninety percent of “why won’t my tensors multiply?” errors in deep-learning code are this rule.

🤖 Why this is literally a neural network layer

A fully-connected neural-network layer computes \(\mathbf{y} = W\mathbf{x} + \mathbf{b}\): multiply the input \(\mathbf{x}\) by a weight matrix \(W\) (a linear transformation), then add a bias vector \(\mathbf{b}\) (a shift). That’s this lesson plus an addition. The matrix \(W\) reshapes the input space — rotating, stretching, and projecting it — to make the next step’s job easier. A deep network stacks many such transformations (with a nonlinearity between them, which is what lets it bend the grid; you’ll see why that matters in Track 4). When you read “a 768→3072 linear layer” in a transformer, you now know exactly what it is: a \(3072\times 768\) matrix mapping \(\mathbb{R}^{768}\to\mathbb{R}^{3072}\).

🛠️ From scratch: matrix–vector products both ways

We implement both the rows view and the columns view, confirm they agree with each other and with NumPy, then watch a matrix move the basis vectors.

code/track-1/matrix_vector.py
import numpy as np

def matvec_rows(A, x):
    """Rows view: output[i] = dot(row i of A, x)."""
    m, n = A.shape
    out = np.zeros(m)
    for i in range(m):
        s = 0.0
        for j in range(n):
            s += A[i, j] * x[j]      # row i dotted with x
        out[i] = s
    return out

def matvec_cols(A, x):
    """Columns view: output = sum_j x[j] * (column j of A)."""
    m, n = A.shape
    out = np.zeros(m)
    for j in range(n):
        out += x[j] * A[:, j]        # weighted sum of columns
    return out

A = np.array([[2.0, 1.0],
              [0.0, 1.5]])
x = np.array([3.0, 2.0])

print(matvec_rows(A, x))   # [8. 3.]
print(matvec_cols(A, x))   # [8. 3.]   same answer, different reasoning
print(A @ x)               # [8. 3.]   NumPy's built-in (use this in practice)

# "Columns are where the basis lands":
print(A @ np.array([1.0, 0.0]))   # [2. 0.]  == first column of A
print(A @ np.array([0.0, 1.0]))   # [1. 1.5] == second column of A

# A rotation by 90 degrees, applied to e1
R = np.array([[0.0, -1.0],
              [1.0,  0.0]])
print(R @ np.array([1.0, 0.0]))   # [0. 1.]  -> e1 rotates onto e2

🔑 Key results to remember

Lock these in

  1. A matrix is a linear transformation; \(A\mathbf{x}\) applies it to \(\mathbf{x}\).
  2. Columns view: \(A\mathbf{x}=\sum_j x_j(\text{column }j)\). The columns are where the basis vectors land.
  3. Rows view: \((A\mathbf{x})_i = \) row \(i\) of \(A\) dotted with \(\mathbf{x}\). Same result; this is how you compute.
  4. Linear means \(T(\mathbf{u}+\mathbf{v})=T\mathbf{u}+T\mathbf{v}\) and \(T(c\mathbf{u})=cT\mathbf{u}\): grid lines stay straight & even, origin fixed.
  5. An \(m\times n\) matrix maps \(\mathbb{R}^n\to\mathbb{R}^m\); for \(A\mathbf{x}\), columns of \(A\) must match \(\dim\mathbf{x}\).
  6. A neural-network layer is \(\mathbf{y}=W\mathbf{x}+\mathbf{b}\) — exactly a matrix transformation plus a shift.

🧩 Check your understanding
Answer from memory, then read why.

Q1. For \(A=\begin{bmatrix}2&1\\0&1.5\end{bmatrix}\), what is \(A\,\mathbf{e}_2\) where \(\mathbf{e}_2=(0,1)\)?

  • \((2,0)\)
    That’s \(A\mathbf{e}_1\), the first column. \(\mathbf{e}_2\) picks out the second column.
  • \((1,1.5)\)
    Correct. \(A\mathbf{e}_2\) is the second column of \(A\). Feeding a basis vector returns the matching column — that’s the whole columns-view idea.
  • \((1,0)\)
    That’s \(\mathbf{e}_1\) itself, not its image. Apply \(A\): the second column is \((1,1.5)\).
  • \((3,1.5)\)
    This looks like \(A\) applied to \((1,1)\), summing both columns. But \(\mathbf{e}_2=(0,1)\) selects only the second column.

Q2. A linear transformation must always…

  • keep every vector the same length.
    Only rotations/reflections preserve length; scaling and shearing change it, yet they’re still linear. Length preservation isn’t required.
  • be a rotation.
    Rotations are linear, but so are scalings, shears, projections, and reflections. Linearity is far broader than rotation.
  • leave the origin fixed.
    Correct. \(T(\mathbf{0})=T(0\cdot\mathbf{v})=0\cdot T(\mathbf{v})=\mathbf{0}\). Any map that moves the origin (like adding a bias) is not linear — it’s “affine.”
  • increase the dimension.
    Linear maps can lower the dimension (a projection), keep it, or raise it. There’s no required direction.

Q3. A \(4\times 3\) matrix \(A\) acts on a vector \(\mathbf{x}\). For \(A\mathbf{x}\) to be defined, \(\mathbf{x}\) must live in which space, and the output lands in which?

  • \(\mathbf{x}\in\mathbb{R}^3\), output in \(\mathbb{R}^4\).
    Correct. Columns count (3) must equal \(\dim\mathbf{x}\), so \(\mathbf{x}\in\mathbb{R}^3\); rows count (4) is the output dimension, so \(A\mathbf{x}\in\mathbb{R}^4\).
  • \(\mathbf{x}\in\mathbb{R}^4\), output in \(\mathbb{R}^3\).
    Reversed. An \(m\times n\) matrix maps \(\mathbb{R}^n\to\mathbb{R}^m\); here \(n=3\) (columns) is the input, \(m=4\) (rows) the output.
  • \(\mathbf{x}\in\mathbb{R}^3\), output in \(\mathbb{R}^3\).
    The input dimension is right, but a \(4\times 3\) matrix outputs a 4-vector, not a 3-vector. The row count sets the output size.
  • Any dimension works.
    No — the inner dimensions must match. \(\mathbf{x}\) must have exactly 3 components (one per column).

Q4. You want a matrix that sends \(\mathbf{e}_1\to(0,1)\) and \(\mathbf{e}_2\to(-1,0)\) (a \(90^\circ\) rotation). What is it?

  • \(\begin{bmatrix}0&1\\-1&0\end{bmatrix}\)
    These are the right vectors but placed as rows. The images of the basis go in as columns, so this matrix is the transpose of what you want (it rotates the other way).
  • \(\begin{bmatrix}1&0\\0&1\end{bmatrix}\)
    That’s the identity — it leaves \(\mathbf{e}_1,\mathbf{e}_2\) where they are, no rotation at all.
  • \(\begin{bmatrix}0&-1\\0&1\end{bmatrix}\)
    Its columns are \((0,0)\) and \((-1,1)\), which aren’t the targets. Build it by placing each image as a column.
  • \(\begin{bmatrix}0&-1\\1&0\end{bmatrix}\)
    Correct. First column \((0,1)=A\mathbf{e}_1\), second column \((-1,0)=A\mathbf{e}_2\). Stacking the images as columns gives the matrix directly.

✍️ Practice

Problem 1 · compute

Let \(A=\begin{bmatrix}1&2\\3&4\end{bmatrix}\) and \(\mathbf{x}=(5,6)\). Compute \(A\mathbf{x}\) using the rows view, then confirm it with the columns view.

Show solution

Rows: row 1 \((1,2)\cdot(5,6)=5+12=17\); row 2 \((3,4)\cdot(5,6)=15+24=39\). So \(A\mathbf{x}=(17,39)\).

Columns: \(5\begin{bmatrix}1\\3\end{bmatrix}+6\begin{bmatrix}2\\4\end{bmatrix}=\begin{bmatrix}5\\15\end{bmatrix}+\begin{bmatrix}12\\24\end{bmatrix}=\begin{bmatrix}17\\39\end{bmatrix}\). ✓ Same vector, reached two ways.

Problem 2 · derivation

Prove that the matrix–vector product is linear: show \(A(\mathbf{u}+\mathbf{v})=A\mathbf{u}+A\mathbf{v}\) using the rows-view formula \((A\mathbf{x})_i=\sum_j a_{ij}x_j\).

Show solution

Compare component \(i\) of each side. Left side:

$$ \big(A(\mathbf{u}+\mathbf{v})\big)_i = \sum_j a_{ij}(u_j+v_j) = \sum_j \big(a_{ij}u_j + a_{ij}v_j\big) = \sum_j a_{ij}u_j + \sum_j a_{ij}v_j. $$

The two sums are exactly \((A\mathbf{u})_i\) and \((A\mathbf{v})_i\), so the left side equals \((A\mathbf{u}+A\mathbf{v})_i\). True for every \(i\), so the vectors are equal. \(\blacksquare\) The same one-line move (distribute, then split the sum) proves \(A(c\mathbf{u})=cA\mathbf{u}\).

Problem 3 · design

Write down the \(2\times 2\) matrix that reflects vectors across the \(x\)-axis (flips the sign of the \(y\)-coordinate). Verify it sends \((3,2)\to(3,-2)\).

Show solution

Ask where the basis goes: \(\mathbf{e}_1=(1,0)\) stays \((1,0)\) (it’s on the axis), and \(\mathbf{e}_2=(0,1)\) flips to \((0,-1)\). Stack as columns:

$$ A=\begin{bmatrix}1 & 0\\ 0 & -1\end{bmatrix}. $$

Check: \(A(3,2) = 3(1,0)+2(0,-1) = (3,-2)\). ✓

Problem 4 · code

In NumPy, build the \(90^\circ\) counter-clockwise rotation matrix and apply it three times to \((1,0)\). Predict the result before running, then verify.

Show solution 
import numpy as np
R = np.array([[0.0, -1.0],
              [1.0,  0.0]])
v = np.array([1.0, 0.0])
print(R @ v)            # [0. 1.]   90 deg
print(R @ R @ v)        # [-1. 0.]  180 deg
print(R @ R @ R @ v)    # [0. -1.]  270 deg

Each application turns the vector another \(90^\circ\) counter-clockwise: \((1,0)\to(0,1)\to(-1,0)\to(0,-1)\). Four applications would return it to \((1,0)\) — a full turn. (Chaining matrices like R @ R is the subject of the next lesson: composition.)
