Vectors: the atoms of ML

A single object that is, at once, a list of numbers, an arrow in space, and a point. Get comfortable with all three pictures and you’ve grasped the most-used idea in machine learning.

Track 1 · Lesson 1 Concept + code ~20 min Builds on: nothing

💡 Concept: three ways to see one thing

Suppose I hand you two numbers, 3 and 2, and tell you they belong together in that order. That ordered pair, written \((3, 2)\), is a vector. That’s it — to begin with, a vector is just an ordered list of numbers. The order matters: \((3,2)\) is not the same vector as \((2,3)\).

The reason vectors are powerful is that the same object can be read in three different ways, and you can switch between them whenever one is more convenient. Let’s meet all three.

View 1 — a list of numbers

The literal reading. \((3, 2)\) is two numbers in a fixed order. Each number is called a component (or coordinate, or entry). We write the vector in bold, \(\mathbf{v}\), and name its components with subscripts:

$$ \mathbf{v} = (v_1, v_2) = (3, 2), \qquad v_1 = 3,\quad v_2 = 2. $$

Here \(\mathbf{v}\) (bold) is the whole vector; \(v_1\) and \(v_2\) (not bold, with a subscript) are its individual numbers. The subscript is just an address: \(v_1\) means “the first component of \(\mathbf{v}\).” We often stack the components vertically instead — it means exactly the same thing:

$$ \mathbf{v} = \begin{bmatrix} 3 \\ 2 \end{bmatrix}. $$

View 2 — an arrow

Now picture a flat plane with a horizontal axis and a vertical axis crossing at a point we call the origin, written \(O\). Read \(\mathbf{v} = (3,2)\) as a set of instructions: from the origin, go 3 to the right, then 2 up. Draw an arrow from \(O\) to where you land. That arrow is the vector.

An arrow carries two pieces of information: a direction (which way it points) and a length (how far it reaches, sometimes called its magnitude). We’ll make “length” precise in Lesson 3; for now the picture is enough.

View 3 — a point

If you only care where the arrow lands and not the journey, you can forget the arrow and keep the destination: the point \((3,2)\). This is the view we use when a vector represents a thing (a data sample) rather than a movement. All three views describe the same \((3,2)\):

💡 The mental switch to practice

“List” is best for computing (a computer stores the numbers). “Arrow” is best for operations like adding or scaling (you’ll see why next lesson). “Point” is best for data (one sample = one location in space). Same object, chosen lens.

📐 Beyond two dimensions

Nothing forced us to stop at two numbers. A vector can have any number of components. Three numbers \((2,3,2)\) describe an arrow in 3-D space — go 2 along, 3 across, 2 up:

The number of components is the vector’s dimension. A vector with \(n\) components is called an \(n\)-dimensional vector, and the collection of all possible \(n\)-dimensional vectors of real numbers gets a name:

$$ \mathbb{R}^n \;=\; \{\, (x_1, x_2, \dots, x_n) \;:\; \text{each } x_i \text{ is a real number} \,\}. $$

Let’s unpack every symbol, because we’ll use this notation constantly. \(\mathbb{R}\) is the set of real numbers (all the numbers on the number line: \(0,\ -1,\ 2.5,\ \pi,\dots\)). The superscript \(n\) in \(\mathbb{R}^n\) says “ordered lists of \(n\) of them.” The curly braces \(\{\,\cdots:\cdots\,\}\) read as “the set of all things of this form, such that this condition holds.” So \(\mathbb{R}^2\) is the plane, \(\mathbb{R}^3\) is space, and \(\mathbb{R}^{784}\) is where a \(28\times 28\) pixel image lives (more on that in a moment). When we write \(\mathbf{v} \in \mathbb{R}^n\), the symbol \(\in\) means “is an element of” — i.e. “\(\mathbf{v}\) is an \(n\)-dimensional real vector.”

📖 Definition — vector (for now)

An element \(\mathbf{v} \in \mathbb{R}^n\) is an ordered list of \(n\) real numbers \(\mathbf{v}=(v_1,\dots,v_n)\). Two vectors are equal exactly when they have the same dimension and every matching component agrees: \(\mathbf{u}=\mathbf{v}\) iff \(u_i = v_i\) for all \(i\). The special zero vector \(\mathbf{0}=(0,\dots,0)\) has every component zero.

(“iff” is mathematician shorthand for “if and only if” — it means the statement holds in both directions.) Later, in Track 1’s wrap-up, we’ll meet the fully general definition of a vector space; everything there will specialize back to these lists of numbers.

🤖 Why this is the foundation of ML

Machine learning runs on data, and a vector is how we feed a single piece of data to a model: each measurement becomes one component. Describe a house by its size, number of bedrooms, age, and price, and the house becomes a point in \(\mathbb{R}^4\):

A grayscale image is the same story with more components: a \(28\times 28\) image is just \(784\) brightness values, so it’s a single point in \(\mathbb{R}^{784}\). “Learning” will turn out to mean finding good geometric structure among these points — which directions matter, which points are close, how to draw boundaries between them. That is why we spend this whole track building rock-solid intuition for vectors and the space they live in.

⚠️ One number, one axis

A 784-dimensional vector is not “mysterious” — it’s the exact same concept as \((3,2)\), just with 784 components instead of 2. You can’t draw it, but you can compute with it identically. Whenever high dimensions feel scary, fall back to the 2-D picture; the algebra is the same.

🛠️ From scratch: vectors in code

In Python the natural container for a vector is a NumPy array. NumPy is the standard numerical library; an array is an efficient, fixed-size list of numbers we can do math on directly. Here is the full mapping from the math to the code.

code/track-1/vectors.py
import numpy as np

# A vector is just an ordered list of numbers.
v = np.array([3, 2])          # the vector v = (3, 2)
print(v)                      # [3 2]
print(v.shape)                # (2,)   -> "2 components, i.e. v is in R^2"

# Components. NOTE: math counts from 1 (v_1, v_2); Python counts from 0.
print(v[0], v[1])             # 3 2    -> v[0] is v_1, v[1] is v_2

# Dimension = number of components
print("dimension:", v.shape[0])   # dimension: 2

# A 3-D vector and the zero vector in R^3
w     = np.array([2, 3, 2])
zero3 = np.zeros(3)
print(w, zero3)               # [2 3 2] [0. 0. 0.]

# Vector EQUALITY: same dimension AND every component matches
a = np.array([3, 2])
b = np.array([3, 2])
c = np.array([2, 3])
print(np.array_equal(a, b))   # True
print(np.array_equal(a, c))   # False -> order matters!
⚠️ The 1 vs 0 trap

Mathematicians index from 1 (\(v_1\) is the first component). Python and NumPy index from 0 (v[0] is the first component). This off-by-one mismatch is the single most common beginner bug. Whenever you translate a formula to code, pause and shift every index down by one.

🔑 Key results to remember

Lock these in

  1. A vector is an ordered list of \(n\) real numbers, \(\mathbf{v}\in\mathbb{R}^n\). Order matters.
  2. The same vector has three readings: a list, an arrow (direction + length), and a point.
  3. \(\mathbb{R}^n\) is the set of all \(n\)-component real vectors; \(n\) is the dimension.
  4. Two vectors are equal iff they match component-by-component (same dimension required).
  5. In ML, every data sample is a vector — each feature is one coordinate.
  6. Code indexes from 0; math indexes from 1. Mind the gap.

🧩 Quiz

🧩 Check your understanding
Wrong answers are deliberately tempting. Answer from memory, then read why.

Q1. Which statement about \((4, 1)\) and \((1, 4)\) is correct?

  • They’re the same vector, just written differently.
    A vector is an ordered list. Re-ordering the components generally gives a different vector pointing in a different direction.
  • They’re equal because they contain the same numbers.
    Containing the same set of numbers isn’t enough — equality requires each position to match. \(v_1\) differs (4 vs 1).
  • They’re different vectors; component order matters.
    Correct. \((4,1)\) reaches far right and slightly up; \((1,4)\) reaches slightly right and far up. Different arrows, different points.
  • They can’t be compared without knowing the dimension.
    Both are clearly in \(\mathbb{R}^2\) (two components each), so they’re directly comparable.

Q2. A \(28\times 28\) grayscale image is treated as a vector. Which space does it live in?

  • \(\mathbb{R}^{2}\), because the image is 2-dimensional.
    The image is laid out in 2-D, but as a vector its dimension is the number of values, not the shape of the grid.
  • \(\mathbb{R}^{28}\), one component per row.
    Each row has 28 pixels, but there are 28 rows; you need a component for every pixel.
  • \(\mathbb{R}^{784}\), one component per pixel.
    Correct. \(28\times 28 = 784\) brightness values, so the image is one point in \(\mathbb{R}^{784}\).
  • \(\mathbb{R}^{56}\), because \(28+28=56\).
    Adding the side lengths counts a border, not the pixels. The grid holds \(28\times 28\) values, a product not a sum.

Q3. In code, v = np.array([7, 8, 9]). What is v[1], and which math component is it?

  • 7, which is \(v_1\).
    Python starts at 0, so v[1] is the second stored number, not the first.
  • 8, which is \(v_2\).
    Correct. v[1] is the second entry, 8, and the second math component is \(v_2\). The index shifts by one between conventions.
  • 9, which is \(v_3\).
    That’s v[2], the last entry. v[1] sits one earlier.
  • 8, which is \(v_1\).
    The value 8 is right, but its math name is \(v_2\) (second component), not \(v_1\).

Q4. Which reading of a vector is most natural when it represents a single customer described by 5 measurements?

  • A point in \(\mathbb{R}^5\).
    Correct. A data sample is a thing, not a movement — its most natural reading is a single location (point) in feature space.
  • An arrow showing a movement.
    Arrows shine for operations like “add this displacement.” A customer isn’t a movement, so the point view fits better.
  • It must be a \(5\times 5\) grid.
    Five measurements give five components — a vector in \(\mathbb{R}^5\), not a 25-entry grid.
  • Only as a raw list; the geometry doesn’t apply.
    The geometry very much applies — “nearby points = similar customers” is exactly how many ML methods reason.

✍️ Practice

Do these with pen and paper (and Python where noted) before opening the solutions.

Problem 1 · concept

Write the vector \(\mathbf{v} = (5, 0, -2)\) in stacked (column) form, state its dimension, and give \(v_2\) and \(v_3\).

Show solution

Stacked form just turns the row into a column:

$$ \mathbf{v} = \begin{bmatrix} 5 \\ 0 \\ -2 \end{bmatrix}. $$

It has three components, so its dimension is \(3\) and \(\mathbf{v}\in\mathbb{R}^3\). Reading off by position: \(v_2 = 0\) (the second component) and \(v_3 = -2\) (the third). A component can be zero or negative — that’s perfectly normal.

Problem 2 · reasoning

Are \((1, 2, 3)\) and \((1, 2)\) equal? Are they even comparable? Explain using the definition of equality.

Show solution

No. Equality requires the same dimension first, and then every matching component to agree. Here one vector lives in \(\mathbb{R}^3\) and the other in \(\mathbb{R}^2\), so they aren’t even the same kind of object — there’s no third component of \((1,2)\) to compare against \(3\). They are not equal, and strictly speaking the equality test doesn’t apply across different dimensions.

Problem 3 · code

In NumPy, create the vector \(\mathbf{u}=(10, 20, 30, 40)\). Print its dimension, print \(u_3\) (the math third component), and verify it is not equal to \((40,30,20,10)\).

Show solution 
import numpy as np
u = np.array([10, 20, 30, 40])
print(u.shape[0])               # 4  -> dimension
print(u[2])                     # 30 -> u_3 is the 3rd component, index 2
print(np.array_equal(u, np.array([40, 30, 20, 10])))  # False

The key trap: \(u_3\) (math, third) is u[2] (code, index 2) \(= 30\). The reversed vector shares the same numbers but in different positions, so it is a different vector and equality is False.

Problem 4 · stretch

A model represents each word in a vocabulary as a vector in \(\mathbb{R}^{300}\) (a “word embedding”). Without computing anything, what would it mean for two such vectors to be “close,” and why is the point-view the right lens here?

Show solution

Each word is a fixed thing, so it’s naturally a point in \(\mathbb{R}^{300}\), not a movement. Two points being “close” would mean the 300 numbers describing them are similar in all coordinates at once — and the design goal of embeddings is exactly that words used in similar contexts (e.g. cat and kitten) land near each other. We can’t picture 300 dimensions, but “nearby points = similar meaning” is the same intuition as two close dots in the plane. We’ll make “close” precise in Lesson 3 with the dot product and distance.
