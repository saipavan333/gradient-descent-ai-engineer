Linear combinations, span & basis

Given a handful of vectors, which points in space can you reach by scaling and adding them? The answer — and the idea of a “minimal set that reaches everything” — is the backbone of dimensionality, coordinates, and later PCA.

Track 1 · Lesson 4 Concept + code ~26 min Builds on: Lessons 1–3

💡 Concept: what can you build from a few vectors?

Recall from Lesson 2 that a linear combination of vectors \(\mathbf{u}\) and \(\mathbf{w}\) is any vector you can make by scaling each and adding: \(c\,\mathbf{u} + d\,\mathbf{w}\), for scalars \(c, d\). Picking different scalars gives different results. The natural question — and it’s a deep one — is: as \(c\) and \(d\) range over all real numbers, which points can you reach?

Span — everything you can reach

The span of a set of vectors is the collection of all their linear combinations — the complete set of reachable points. For vectors \(\mathbf{v}_1,\dots,\mathbf{v}_k\):

$$ \operatorname{span}\{\mathbf{v}_1,\dots,\mathbf{v}_k\} \;=\; \big\{\, c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \cdots + c_k\mathbf{v}_k \;:\; c_1,\dots,c_k \in \mathbb{R} \,\big\}. $$

Read it as “the set of all \(c_1\mathbf{v}_1+\cdots+c_k\mathbf{v}_k\), such that the scalars range over all reals.” The span is not one vector; it’s an entire region of space. Two small cases reveal everything:

The catch: vectors can be redundant

“Two vectors span the plane” isn’t always true. If the second vector lies along the same line as the first — say \(\mathbf{w}=-0.6\,\mathbf{u}\) — then every combination \(c\mathbf{u}+d\mathbf{w}\) is still just a multiple of \(\mathbf{u}\), and the span collapses back to a single line. The second vector added no new direction.

📐 Linear independence: no wasted vectors

We need a precise word for “each vector pulls its weight.” A set of vectors is linearly independent if none of them is a linear combination of the others — equivalently, if the only way to combine them into the zero vector is the trivial way (all scalars zero):

$$ c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \cdots + c_k\mathbf{v}_k = \mathbf{0} \quad\Longrightarrow\quad c_1 = c_2 = \cdots = c_k = 0. $$

If instead some nonzero choice of scalars produces \(\mathbf{0}\), the vectors are linearly dependent — at least one is redundant and can be rebuilt from the rest. Let’s see why these two phrasings (“none is a combination of the others” and “only the trivial combination gives \(\mathbf{0}\)”) are the same statement.

🧮 The two definitions are equivalent

Suppose some nonzero combination vanishes: \(c_1\mathbf{v}_1+\cdots+c_k\mathbf{v}_k=\mathbf{0}\) with, say, \(c_1\neq0\). Solve for \(\mathbf{v}_1\): divide by \(-c_1\) to get \(\mathbf{v}_1 = -\tfrac{c_2}{c_1}\mathbf{v}_2 - \cdots - \tfrac{c_k}{c_1}\mathbf{v}_k\). So \(\mathbf{v}_1\) is a combination of the others — dependence. Run the argument backwards and the converse holds too. Hence “a nontrivial combination hits zero” ⇔ “some vector is redundant.”

💡 The 2-D shortcut

In the plane, two vectors \(\mathbf{a}=(a_1,a_2)\) and \(\mathbf{b}=(b_1,b_2)\) are dependent exactly when one is proportional to the other, which happens iff \(a_1 b_2 - a_2 b_1 = 0\). If that quantity is nonzero, they point in genuinely different directions and span the whole plane. (You’ll recognize \(a_1b_2-a_2b_1\) as the determinant in Lesson 7 — here it’s just a proportionality test.)

🎯 Basis: a minimal complete set of directions

Put the two ideas together. A basis for a space is a set of vectors that is both spanning (their combinations reach every point) and linearly independent (no redundancy). It is the “just right” set: enough vectors to build everything, but not one more than necessary. The number of vectors in a basis is the dimension of the space — and remarkably, every basis of a given space has the same size.

The most familiar basis of the plane is the standard basis \(\mathbf{e}_1=(1,0)\) and \(\mathbf{e}_2=(0,1)\). Saying a point has “coordinates \((3,2)\)” really means \(\mathbf{p}=3\mathbf{e}_1+2\mathbf{e}_2\) — coordinates are just the scalars in the linear combination that rebuilds the point from the basis. 🔑 The big idea

A basis is a coordinate system. Choosing a good basis can make data simple — directions that matter line up with the axes. This is the entire premise of Principal Component Analysis (PCA): find the basis in which your data’s most important variation lies along the first few axes, then keep only those. We’ll build PCA later, but its foundation is exactly this lesson.

🤖 Why this matters for ML

Three direct payoffs. (1) Dimensionality. The “dimension” of your data is the size of a basis for the space its features actually occupy — often far smaller than the number of raw features, because features are correlated (linearly dependent-ish). Finding that true dimension is dimensionality reduction. (2) Redundant features. If feature C is always \(2\times\)feature A minus feature B, it’s linearly dependent and adds no information — independence is how we detect that. (3) Coordinates as representations. A neural network’s hidden layer is, in effect, re-expressing your data in a new, learned basis where the task becomes easy. “Representation learning” is “learning a good basis.”

🛠️ From scratch: span & independence in code

We’ll (a) form linear combinations, (b) test 2-D independence with the proportionality rule, and (c) solve for a point’s coordinates in a chosen basis — i.e. find the scalars \(c,d\) with \(c\mathbf{f}_1+d\mathbf{f}_2=\mathbf{p}\).

code/track-1/span_basis.py
import numpy as np

def linear_combination(coeffs, vectors):
    """c1*v1 + c2*v2 + ... given lists of scalars and vectors."""
    out = np.zeros(len(vectors[0]), dtype=float)
    for c, v in zip(coeffs, vectors):
        out += c * np.asarray(v, float)
    return out

print(linear_combination([2, 1], [(2.0, 0.5), (0.6, 1.8)]))   # [4.6 2.8]

def independent_2d(a, b):
    """Two 2-D vectors are independent iff a1*b2 - a2*b1 != 0."""
    a, b = np.asarray(a, float), np.asarray(b, float)
    cross = a[0]*b[1] - a[1]*b[0]
    return not np.isclose(cross, 0.0)

print(independent_2d((1, 0), (0, 1)))     # True  -> standard basis
print(independent_2d((2, 1), (4, 2)))     # False -> (4,2) = 2*(2,1)

def coordinates_in_basis(p, f1, f2):
    """Solve c*f1 + d*f2 = p for the coordinates (c, d)."""
    F = np.column_stack([f1, f2])         # 2x2 matrix whose columns are the basis
    return np.linalg.solve(F, p)          # the unique solution (basis => invertible)

c, d = coordinates_in_basis(np.array([3.0, 2.0]),
                            np.array([1.0, 0.4]),
                            np.array([0.4, 1.0]))
print(round(c, 2), round(d, 2))           # 2.62 0.95  -> matches Figure 4.4

We sneak-previewed np.column_stack and np.linalg.solve; you don’t need to understand the matrix machinery yet (that’s the next two lessons). For now, read it as “ask the computer for the scalars that rebuild \(\mathbf{p}\) from \(\mathbf{f}_1,\mathbf{f}_2\)” — which is exactly “find \(\mathbf{p}\)’s coordinates in that basis.”

🔑 Key results to remember

Lock these in

  1. Span = the set of all linear combinations of some vectors = every point they can reach.
  2. One nonzero vector spans a line; two independent vectors span a plane; \(k\) independent vectors span a \(k\)-dimensional space.
  3. Linearly independent ⇔ the only combination equal to \(\mathbf{0}\) is the all-zero one ⇔ no vector is redundant.
  4. A basis = spanning + independent (a minimal complete set). Its size is the space’s dimension.
  5. Coordinates are the scalars in the linear combination that rebuilds a point from the basis — and they change if the basis changes.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. What is the span of the single vector \((2,2)\)?

  • The whole plane \(\mathbb{R}^2\).
    One vector can only be scaled, never tilted, so it can’t reach off its own line. You need a second, independent vector to fill the plane.
  • The line \(y=x\) through the origin.
    Correct. Every multiple \(c(2,2)=(2c,2c)\) has equal coordinates, tracing the line \(y=x\) through the origin.
  • Just the point \((2,2)\).
    Span includes all scalar multiples, not only \(c=1\). It’s an infinite line, not a single point.
  • The two points \((2,2)\) and \((0,0)\).
    Those are two members of the span (for \(c=1\) and \(c=0\)), but the span is the entire line they sit on.

Q2. Which pair of vectors is linearly dependent?

  • \((1,0)\) and \((0,1)\)
    Independent — these are the standard basis; neither is a multiple of the other (\(1\cdot1-0\cdot0=1\neq0\)).
  • \((1,2)\) and \((2,1)\)
    Independent: \(1\cdot1 - 2\cdot2 = -3 \neq 0\). They point in different directions.
  • \((3,1)\) and \((-1,2)\)
    Independent: \(3\cdot2 - 1\cdot(-1) = 7 \neq 0\).
  • \((2,3)\) and \((4,6)\)
    Correct. \((4,6)=2\,(2,3)\), so one is a scalar multiple of the other and \(2\cdot6-3\cdot4=0\). They share a line.

Q3. A basis for \(\mathbb{R}^3\) must contain exactly how many vectors, and satisfy what?

  • Any number, as long as they span \(\mathbb{R}^3\).
    Spanning alone isn’t enough — four spanning vectors in \(\mathbb{R}^3\) include a redundant one, so they’re not a basis. A basis forbids redundancy.
  • Two independent vectors.
    Two independent vectors only span a plane inside \(\mathbb{R}^3\); they can’t reach every point of 3-D space.
  • Three vectors that are independent and span \(\mathbb{R}^3\).
    Correct. The dimension of \(\mathbb{R}^3\) is 3, so a basis has exactly 3 vectors, and they must be both independent and spanning. (For 3 vectors in \(\mathbb{R}^3\), either property actually implies the other.)
  • Three vectors, any three will do.
    Not any three — if they’re dependent (e.g. all in one plane) they fail to span \(\mathbb{R}^3\). Independence is required.

Q4. The point \(\mathbf{p}\) has coordinates \((3,2)\) in the standard basis. In a different basis, its coordinates are \((2.6, 1.0)\). What changed?

  • Only the description; the point itself is unchanged.
    Correct. A point is a fixed location; coordinates are just the recipe to rebuild it from a chosen basis. Change the basis and the recipe changes, but the point stays put.
  • The point physically moved to a new location.
    Nothing moved. We re-expressed the same point in different reference directions.
  • The point now lives in a different space.
    It’s still the same point in the same plane \(\mathbb{R}^2\); only the coordinate system differs.
  • One of the coordinate systems must be wrong.
    Both are valid. Many bases describe the same space; each gives correct — but different — coordinates.

✍️ Practice

Problem 1 · reasoning

Do \((1,1)\) and \((2,2)\) span the plane? What exactly is their span?

Show solution

No. Since \((2,2)=2(1,1)\), the two are linearly dependent — the proportionality test gives \(1\cdot2-1\cdot2=0\). Every combination \(c(1,1)+d(2,2)=(c+2d)(1,1)\) is still a multiple of \((1,1)\). So their span is just the line \(y=x\) through the origin, not the plane. You’d need a vector off that line (e.g. \((1,0)\)) to span all of \(\mathbb{R}^2\).

Problem 2 · compute

Express \(\mathbf{p}=(7,4)\) as a linear combination of \(\mathbf{e}_1=(1,0)\) and \(\mathbf{e}_2=(0,1)\), and then of \(\mathbf{f}_1=(1,1)\) and \(\mathbf{f}_2=(1,-1)\).

Show solution

Standard basis is immediate: \(\mathbf{p}=7\mathbf{e}_1+4\mathbf{e}_2\).

For \(\mathbf{f}_1,\mathbf{f}_2\), solve \(c(1,1)+d(1,-1)=(7,4)\), i.e. \(c+d=7\) and \(c-d=4\). Adding the equations: \(2c=11\Rightarrow c=5.5\); then \(d=7-5.5=1.5\). Check: \(5.5(1,1)+1.5(1,-1)=(7,\,4)\) ✓. So in this basis \(\mathbf{p}\) has coordinates \((5.5,\,1.5)\) — same point, different numbers.

Problem 3 · derivation

Prove that any set of vectors that includes the zero vector is linearly dependent.

Show solution

Say the set is \(\{\mathbf{0}, \mathbf{v}_2, \dots, \mathbf{v}_k\}\). Choose the scalars \(c_1=1\) and \(c_2=\cdots=c_k=0\). Then \(c_1\mathbf{0}+c_2\mathbf{v}_2+\cdots+c_k\mathbf{v}_k = 1\cdot\mathbf{0} = \mathbf{0}\). We produced the zero vector with a nonzero scalar (\(c_1=1\)), which is exactly the definition of linear dependence. \(\blacksquare\) Intuitively: the zero vector points nowhere, so it can never add a new direction.

Problem 4 · code

Use the coordinates_in_basis function to find the coordinates of \((7,4)\) in the basis \(\mathbf{f}_1=(1,1)\), \(\mathbf{f}_2=(1,-1)\). Confirm they match your hand calculation in Problem 2.

Show solution 
import numpy as np
def coordinates_in_basis(p, f1, f2):
    F = np.column_stack([f1, f2])
    return np.linalg.solve(F, p)

print(coordinates_in_basis(np.array([7.0, 4.0]),
                           np.array([1.0, 1.0]),
                           np.array([1.0, -1.0])))    # [5.5 1.5]

The output [5.5 1.5] matches the hand result \((5.5, 1.5)\). The solver is doing the same elimination you did by hand — adding and subtracting the two equations — just mechanically.
