Random variables & distributions

To do math with chance, we attach numbers to outcomes — that’s a random variable — and describe how those numbers are spread out — that’s a distribution. These two ideas turn “uncertainty” into something you can compute, sample, and learn from.

Track 1 · Lesson 10 Concept + code ~26 min Builds on: Lesson 9

💡 Concept: numbers attached to chance

Last lesson, outcomes could be anything — heads, tails, sick, healthy. To compute with them, we need numbers. A random variable is simply a rule that assigns a number to each outcome — formally, a function \(X:\Omega\to\mathbb{R}\) from the sample space to the real line. We write random variables as capital letters (\(X\), \(Y\)) and the specific values they take as lowercase (\(x\)).

Example: flip two coins. Let \(X\) = the number of heads. Then \(X\) maps the outcome \(HH\) to \(2\), both \(HT\) and \(TH\) to \(1\), and \(TT\) to \(0\):

Two flavors: discrete and continuous

Random variables come in two kinds. A discrete one takes separate, countable values (the count of heads: \(0,1,2\); a die: \(1\)–\(6\)). A continuous one can take any value in a range (a person’s height; tomorrow’s temperature). They’re described by slightly different objects, so we take them in turn.

📐 Discrete: the probability mass function

For a discrete \(X\), the probability mass function (PMF) gives the probability of each value: \(p(x) = P(X = x)\). It’s an honest probability — each \(p(x)\) is between \(0\) and \(1\), and because some value must occur, they sum to one:

$$ 0 \le p(x) \le 1, \qquad \sum_{\text{all } x} p(x) = 1. $$

For two coin flips, \(p(0)=\tfrac14,\ p(1)=\tfrac12,\ p(2)=\tfrac14\) (note \(HT\) and \(TH\) both give one head, so \(p(1)\) is doubled). They sum to \(1\). ✓

📐 Continuous: the probability density function

Continuous variables need a twist. The probability of any exact value is zero — there are infinitely many possible heights, so \(P(\text{height} = 1.8000\ldots\text{m})=0\). Instead we use a probability density function (PDF) \(f(x)\), where probability is the area under the curve over a range:

$$ P(a \le X \le b) = \int_a^b f(x)\,dx, \qquad \int_{-\infty}^{\infty} f(x)\,dx = 1. $$

The tall “\(\int\)” is an integral — for our purposes just “the area under \(f\) between \(a\) and \(b\).” The total area is \(1\), mirroring “the PMF sums to one.” One subtlety that trips everyone up at first: a density \(f(x)\) is not a probability and can exceed \(1\) (a sharp, narrow peak is tall); only areas are probabilities, and those never exceed \(1\).

📐 One object for both: the CDF

The cumulative distribution function (CDF) works for discrete and continuous alike: \(F(x) = P(X \le x)\), the probability of landing at or below \(x\). It always starts at \(0\) (far left), climbs to \(1\) (far right), and never decreases. For discrete variables it jumps at each value; for continuous ones it rises smoothly.

⚠️ The mistake everyone makes once

For a continuous variable, the value of the PDF \(f(x)\) is not the probability of \(x\). \(P(X=x)=0\) for every single \(x\). Probability only exists over an interval, as area. If you ever read a density of \(2.5\) and panic that “probability is \(2.5\),” remember: only the area is a probability.

🤖 Why models are distributions

Modern models don’t just output a guess — they output a distribution. A classifier’s final softmax layer produces a Categorical PMF over classes (probabilities that sum to one); you read off the most likely class, but the whole distribution carries the model’s confidence. A language model outputs a PMF over the next token at every step, and “sampling” from it is how it writes text. A regression model can output the \(\mu\) and \(\sigma\) of a Gaussian to express uncertainty in its prediction. Training these models means adjusting parameters so the distribution they output puts high probability on the data actually observed — which is the likelihood, the bridge to Lesson 12.

🛠️ From scratch: sampling, PMFs, and densities

code/track-1/random_variables.py
import numpy as np
rng = np.random.default_rng(0)

# Simulate X = number of heads in 2 coin flips, then build the empirical PMF
flips = rng.integers(0, 2, size=(100_000, 2))   # 0=tails, 1=heads
X = flips.sum(axis=1)                            # number of heads per trial: 0,1,2
values, counts = np.unique(X, return_counts=True)
pmf = counts / counts.sum()
print(dict(zip(values.tolist(), np.round(pmf, 3).tolist())))
# ~ {0: 0.25, 1: 0.50, 2: 0.25}  -> matches the theory

# A continuous variable: samples form a histogram that approximates the PDF
heights = rng.normal(170, 7, size=100_000)       # Gaussian-ish heights (cm)
# P(165 <= height <= 175) by counting (area under the PDF):
frac = np.mean((heights >= 165) & (heights <= 175))
print("P(165<=h<=175) ~", round(frac, 3))

# The CDF: P(X <= x) is just the fraction of samples at or below x
def empirical_cdf(samples, x):
    return np.mean(samples <= x)
print("F(170) ~", round(empirical_cdf(heights, 170), 3))   # ~0.5 (170 is the mean)

# A density can exceed 1: a narrow uniform on [0, 0.5] has height 2
u = rng.uniform(0, 0.5, size=100_000)
hist, edges = np.histogram(u, bins=10, density=True)
print("max density ~", round(hist.max(), 2))     # ~2.0, yet it's a valid PDF

🔑 Key results to remember

Lock these in

  1. A random variable \(X:\Omega\to\mathbb{R}\) attaches a number to each outcome (capital \(X\), values \(x\)).
  2. Discrete → PMF \(p(x)=P(X=x)\), with \(\sum p(x)=1\). Continuous → PDF \(f(x)\), with probability = area, \(\int f = 1\).
  3. A density can exceed 1; only areas are probabilities. For continuous \(X\), \(P(X=x)=0\).
  4. The CDF \(F(x)=P(X\le x)\) runs \(0\to1\), never decreasing — works for both kinds.
  5. ML models output distributions: softmax = Categorical PMF; sampling = drawing from that distribution.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why.

Q1. A continuous PDF satisfies \(f(0.3) = 2.0\). What does this tell you?

  • \(P(X = 0.3) = 2.0\).
    Probabilities never exceed \(1\), and for a continuous variable \(P(X=0.3)=0\) anyway. \(f\) is a density, not a probability.
  • The PDF is invalid because it exceeds 1.
    A density may exceed \(1\) (it just has to integrate to \(1\)). A tall, narrow peak is perfectly legal.
  • The density is high near \(0.3\); probability comes from area, not this height.
    Correct. \(f(0.3)=2\) means values near \(0.3\) are relatively likely, but actual probability is the area under \(f\) over an interval.
  • About 200% of outcomes equal \(0.3\).
    Nonsensical — you can’t have 200% of outcomes. The number \(2.0\) is a density value, not a fraction of outcomes.

Q2. For a discrete random variable, which must be true of its PMF \(p(x)\)?

  • \(\int p(x)\,dx = 1\).
    Integration is for continuous densities. Discrete PMFs are summed, not integrated.
  • \(\sum_x p(x) = 1\) and each \(0\le p(x)\le 1\).
    Correct. The probabilities of all values must add to \(1\), and each is a genuine probability in \([0,1]\).
  • \(p(x)\) can be any positive number.
    Each \(p(x)\) is a probability, so it cannot exceed \(1\) (unlike a continuous density).
  • \(p(x) = 0\) for all \(x\).
    Then nothing could happen and the total would be \(0\), not \(1\). Some values must carry positive mass.

Q3. The softmax output of an image classifier over 1000 classes is best described as…

  • a continuous PDF.
    Classes are discrete categories, not a continuum, so a PDF/area model doesn’t fit.
  • a single number.
    Softmax produces a whole vector of probabilities (one per class), not one number — though you may later pick the top one.
  • a CDF.
    A CDF accumulates probability up to a threshold; softmax gives the probability of each class, which is a PMF, not a running total.
  • a Categorical PMF over the 1000 classes.
    Correct. Softmax yields 1000 non-negative numbers summing to \(1\) — exactly a discrete probability mass function over the classes.

Q4. Which property does every CDF \(F(x)\) have?

  • It is non-decreasing, with \(F(-\infty)=0\) and \(F(\infty)=1\).
    Correct. As the threshold rises you can only accumulate more probability, from \(0\) up to a total of \(1\).
  • It sums to 1 over all \(x\).
    That’s the PMF. The CDF is a running total that approaches \(1\); its values aren’t summed.
  • It can decrease where the variable is unlikely.
    Never. Adding more outcomes (raising \(x\)) can’t reduce accumulated probability, so \(F\) never decreases.
  • It always exceeds 1 in the middle.
    A CDF is a probability \(P(X\le x)\), so it stays within \([0,1]\) everywhere.

✍️ Practice

Problem 1 · compute

A fair die is rolled; \(X\) is the result. Write its PMF, then compute \(P(X \le 2)\) from the CDF.

Show solution

Each face is equally likely: \(p(x) = 1/6\) for \(x=1,\dots,6\), and they sum to \(1\) ✓. The CDF at \(2\) accumulates the first two values: \(F(2) = P(X\le 2) = p(1)+p(2) = 1/6 + 1/6 = 1/3\).

Problem 2 · reasoning

A PDF is \(f(x) = 2x\) for \(0 \le x \le 1\) and \(0\) elsewhere. Verify it’s a valid density, and find \(P(X \le 0.5)\).

Show solution

Valid if it’s non-negative (it is, for \(x\ge0\)) and integrates to \(1\). The area under \(2x\) from \(0\) to \(1\) is \(\int_0^1 2x\,dx = [x^2]_0^1 = 1\). ✓ (Note \(f(1)=2>1\), a legal density value.) For the probability:

$$ P(X\le 0.5) = \int_0^{0.5} 2x\,dx = [x^2]_0^{0.5} = 0.25. $$

So \(F(0.5)=0.25\) — values bunch toward the right since the density grows with \(x\).

Problem 3 · code

Simulate the die from Problem 1, build the empirical PMF from 200,000 rolls, and check \(P(X\le2)\approx 1/3\).

Show solution 
import numpy as np
rng = np.random.default_rng(2)
X = rng.integers(1, 7, size=200_000)            # 1..6 inclusive
vals, counts = np.unique(X, return_counts=True)
print(np.round(counts / counts.sum(), 3))       # ~[0.167]*6
print("P(X<=2) ~", round(np.mean(X <= 2), 3))   # ~0.333

Each value sits near \(1/6\approx0.167\) and \(P(X\le2)\approx0.333\), matching the theory. The match improves as the sample size grows (law of large numbers).

Problem 4 · stretch

Why does a language model’s “temperature” setting (which sharpens or flattens its next-token PMF) change how creative its text is? Frame your answer in terms of the distribution.

Show solution

At each step the model outputs a Categorical PMF over the vocabulary. Low temperature sharpens that PMF — mass concentrates on the few most-likely tokens, so sampling almost always picks them: safe, repetitive, “predictable” text. High temperature flattens the PMF toward uniform — unlikely tokens get a real chance, so the output is more varied and surprising (and more error-prone). “Creativity vs. reliability” is literally a knob on the entropy of the output distribution. This is the practical face of everything in this lesson: the model is a distribution, and how you sample from it shapes the result.
