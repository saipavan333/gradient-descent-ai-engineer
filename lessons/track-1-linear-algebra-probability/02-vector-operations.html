Vector operations & geometry

There are only two things you can do to vectors: add them, and scale them. Astonishingly, all of linear algebra — and the entire training loop of a neural network — is built from just these two moves.

Track 1 · Lesson 2 Concept + code ~22 min Builds on: Lesson 1

💡 Concept: two operations, that’s all

In Lesson 1 a vector sat still. Now we make it move. There are exactly two fundamental operations on vectors: vector addition (combine two vectors into one) and scalar multiplication (stretch or shrink a vector). The word scalar just means an ordinary single number (an element of \(\mathbb{R}\)) — it’s called a scalar because multiplying by it scales a vector. We’ll write scalars as plain letters like \(c\) or \(\alpha\), and keep vectors bold like \(\mathbf{u}, \mathbf{v}\).

Addition: add matching components

To add two vectors of the same dimension, add their matching components. For \(\mathbf{u},\mathbf{v}\in\mathbb{R}^n\):

$$ \mathbf{u}+\mathbf{v} = (u_1+v_1,\; u_2+v_2,\; \dots,\; u_n+v_n). $$

For example \((3,1)+(1,2) = (4,3)\). The rule is purely mechanical — but it has a beautiful picture. Place \(\mathbf{v}\)’s tail at \(\mathbf{u}\)’s tip and walk along it. Where you end up is \(\mathbf{u}+\mathbf{v}\). This is the tip-to-tail rule: addition is “do one displacement, then the other.”

Because we add numbers component-by-component, and ordinary number addition doesn’t care about order, vector addition inherits two tidy properties. Addition is commutative, \(\mathbf{u}+\mathbf{v}=\mathbf{v}+\mathbf{u}\) (the parallelogram is the same whichever side you walk first), and associative, \((\mathbf{u}+\mathbf{v})+\mathbf{w}=\mathbf{u}+(\mathbf{v}+\mathbf{w})\) (chaining displacements, grouping doesn’t matter). Adding the zero vector changes nothing: \(\mathbf{v}+\mathbf{0}=\mathbf{v}\).

Scalar multiplication: stretch, shrink, flip

To multiply a vector by a scalar \(c\), multiply every component by \(c\):

$$ c\,\mathbf{v} = (c\,v_1,\; c\,v_2,\; \dots,\; c\,v_n). $$

Geometrically, \(c\mathbf{v}\) keeps \(\mathbf{v}\)’s line through the origin but changes how far along it you go. If \(c>1\) the arrow stretches; if \(0shrinks; if \(c<0\) it flips to point the opposite way; and \(c=0\) collapses it to the zero vector. Crucially, the direction only ever stays the same or reverses — scaling never tilts a vector off its line.

Subtraction: the difference is a displacement

We don’t need a new rule for subtraction. Define it through the two operations we already have: scale \(\mathbf{v}\) by \(-1\), then add.

$$ \mathbf{u}-\mathbf{v} \;\equiv\; \mathbf{u} + (-1)\mathbf{v} \;=\; (u_1-v_1,\dots,u_n-v_n). $$

(The symbol \(\equiv\) means “is defined to be.”) The geometry is the single most useful picture in this lesson: \(\mathbf{u}-\mathbf{v}\) is the arrow that points from the tip of \(\mathbf{v}\) to the tip of \(\mathbf{u}\). Reason: starting at \(\mathbf{v}\) and adding \(\mathbf{u}-\mathbf{v}\) must land you at \(\mathbf{u}\), since \(\mathbf{v}+(\mathbf{u}-\mathbf{v})=\mathbf{u}\). So the difference is exactly the displacement from \(\mathbf{v}\) to \(\mathbf{u}\).

⚠️ Which way does the difference point?

\(\mathbf{u}-\mathbf{v}\) points toward \(\mathbf{u}\) (the vector that’s written first / not subtracted). \(\mathbf{v}-\mathbf{u}\) is the same arrow reversed. Getting this backwards silently breaks distance and similarity calculations later, so anchor it now.

📐 The rules these obey

Adding and scaling interact through distributive laws, which let us expand and factor expressions just like with ordinary numbers. For scalars \(c,d\) and vectors \(\mathbf{u},\mathbf{v}\):

$$ c(\mathbf{u}+\mathbf{v}) = c\mathbf{u}+c\mathbf{v}, \qquad (c+d)\mathbf{v} = c\mathbf{v}+d\mathbf{v}, \qquad c(d\mathbf{v}) = (cd)\mathbf{v}. $$

You don’t need to memorize these as rules to recite — they’re what you’d guess from the component formulas, because each one is just the ordinary distributive law applied in every component at once. Verify the first one by hand in the practice section; it’s a one-line proof and a good habit.

An expression like \(c\mathbf{u}+d\mathbf{v}\) — scale some vectors and add the results — is called a linear combination of \(\mathbf{u}\) and \(\mathbf{v}\). It is the central object of the next two lessons, and ultimately of every linear model in ML. For now just notice: a linear combination uses only our two operations, nothing else.

🤖 Why ML is mostly adding and scaling vectors

Two quick previews to show these aren’t toy operations.

Averaging data. The “center of mass” of a dataset \(\mathbf{x}_1,\dots,\mathbf{x}_m\) is the mean vector \( \bar{\mathbf{x}} = \tfrac{1}{m}(\mathbf{x}_1+\mathbf{x}_2+\cdots+\mathbf{x}_m)\) — add all the data vectors, then scale by \(1/m\). That single point summarizes a whole cloud of data, and it’s nothing but addition and scaling.

🤖 The training loop is vector subtraction

When a neural network learns, it stores all its parameters in one big vector \(\mathbf{w}\) and repeatedly updates it by \(\mathbf{w} \leftarrow \mathbf{w} - \alpha\, \mathbf{g}\): take a step of size \(\alpha\) (a scalar) in the direction \(-\mathbf{g}\). That’s scale a vector, then subtract — the exact two operations from this lesson, run millions of times. You’ll build this in Track 2.

🛠️ From scratch: operations in code

NumPy lets you write vector math almost exactly as it reads on paper. The operations are vectorized: +, -, and *-by-a-number act component-wise automatically — no loops needed.

code/track-1/vector_ops.py
import numpy as np

u = np.array([3, 1])
v = np.array([1, 2])

print(u + v)          # [4 3]   addition: component-wise
print(u - v)          # [2 -1]  subtraction: points from v to u
print(2 * v)          # [2 4]   scalar multiplication: stretch
print(-1 * v)         # [-1 -2] flip direction
print(0.5 * u)        # [1.5 0.5] shrink

# A linear combination  c*u + d*v
c, d = 2.0, -1.0
print(c * u + d * v)  # [5.0 0.0]

# The mean (center of mass) of several vectors
X = np.array([[3, 1],
              [1, 2],
              [2, 6]])       # three vectors, one per row
mean = X.mean(axis=0)        # average down the columns
print(mean)                  # [2. 3.]  == (1/3)*( (3,1)+(1,2)+(2,6) )

# Verify a distributive law numerically: c(u+v) == c*u + c*v
print(np.array_equal(c*(u+v), c*u + c*v))   # True

Heads-up for later: u * v in NumPy multiplies component-wise (\((3,1)*(1,2)=(3,2)\)), which is not the dot product. We meet the real vector product — a single number — in the next lesson.

🔑 Key results to remember

Lock these in

  1. Addition is component-wise; geometrically it’s tip-to-tail (a parallelogram diagonal).
  2. Scalar multiplication scales every component; it stretches/shrinks/flips along the same line.
  3. Subtraction is \(\mathbf{u}+(-1)\mathbf{v}\); the arrow \(\mathbf{u}-\mathbf{v}\) points from \(\mathbf{v}\) to \(\mathbf{u}\).
  4. Addition is commutative & associative; the distributive laws \(c(\mathbf{u}+\mathbf{v})=c\mathbf{u}+c\mathbf{v}\) hold.
  5. A linear combination \(c\mathbf{u}+d\mathbf{v}\) uses only these two operations — and it’s everywhere in ML.

🧩 Quiz

🧩 Check your understanding
Answer from memory, then read why each option is right or wrong.

Q1. Compute \((5, -2) + (-1, 6)\).

  • \((4, 4)\)
    Correct. Add matching components keeping every sign: \(5+(-1)=4\) and \(-2+6=4\).
  • \((6, 8)\)
    This ignores the signs and adds magnitudes: \(|5|+|{-1}|=6\), \(|{-2}|+|6|=8\). But \(-1\) and \(-2\) are negative; signs must be kept.
  • \((4, -8)\)
    The first component is right, but the second did \(-2-6=-8\) instead of \(-2+6=4\). This is an addition, so add the \(+6\).
  • \((-5, -12)\)
    This multiplies the components \((5\cdot-1,\,-2\cdot6)\) instead of adding them. Addition is not multiplication.

Q2. The arrow \(\mathbf{a}-\mathbf{b}\) points…

  • from the tip of \(\mathbf{a}\) to the tip of \(\mathbf{b}\).
    That’s \(\mathbf{b}-\mathbf{a}\), the reverse. Remember: the difference points toward the vector written first.
  • from the tip of \(\mathbf{b}\) to the tip of \(\mathbf{a}\).
    Correct. Since \(\mathbf{b}+(\mathbf{a}-\mathbf{b})=\mathbf{a}\), starting at \(\mathbf{b}\) and adding the difference lands you at \(\mathbf{a}\) — so it points from \(\mathbf{b}\) to \(\mathbf{a}\).
  • always in the same direction as \(\mathbf{a}\).
    Only if \(\mathbf{b}=\mathbf{0}\). In general the difference has its own direction set by both vectors.
  • toward the origin.
    No fixed relation to the origin; the difference connects the two tips, wherever they are.

Q3. For a nonzero \(\mathbf{v}\), which scalar multiple does not lie on the same line through the origin as \(\mathbf{v}\)?

  • \(3\mathbf{v}\)
    On the line — stretched in the same direction.
  • \(-2\mathbf{v}\)
    On the line — pointing the opposite way, but still along the same line.
  • \(0\,\mathbf{v}=\mathbf{0}\)
    The origin lies on every line through the origin, including \(\mathbf{v}\)’s. So this is on the line too.
  • None — every scalar multiple stays on that line.
    Correct. This is the defining feature of scaling: \(c\mathbf{v}\) never leaves \(\mathbf{v}\)’s line through the origin, for any \(c\). To leave the line you must add a different vector.

Q4. The mean of \((0,0)\), \((4,0)\), and \((2,6)\) is which point?

  • \((2, 2)\)
    Correct. Sum \(=(6,6)\); divide by \(3\) to get \((2,2)\). The mean is “add them all, then scale by \(1/m\).”
  • \((6, 6)\)
    That’s the sum before dividing by the count \(m=3\). The mean must scale the sum down.
  • \((3, 3)\)
    This divides by 2, not 3. There are three vectors, so divide by 3.
  • \((2, 0)\)
    This averages only the first two and forgets \((2,6)\). The mean uses every sample.

✍️ Practice

Problem 1 · compute

Let \(\mathbf{u}=(2,5)\) and \(\mathbf{v}=(6,1)\). Compute \(\mathbf{u}+\mathbf{v}\), \(\mathbf{u}-\mathbf{v}\), and \(3\mathbf{u}-2\mathbf{v}\).

Show solution

\(\mathbf{u}+\mathbf{v}=(2+6,\;5+1)=(8,6)\).

\(\mathbf{u}-\mathbf{v}=(2-6,\;5-1)=(-4,4)\).

\(3\mathbf{u}=(6,15)\) and \(2\mathbf{v}=(12,2)\), so \(3\mathbf{u}-2\mathbf{v}=(6-12,\;15-2)=(-6,13)\). Scale first, then subtract component-wise.

Problem 2 · derivation

Prove the distributive law \(c(\mathbf{u}+\mathbf{v}) = c\mathbf{u}+c\mathbf{v}\) for \(\mathbf{u},\mathbf{v}\in\mathbb{R}^n\), using only the component definitions.

Show solution

Look at the \(i\)-th component of each side. On the left, the \(i\)-th component of \(\mathbf{u}+\mathbf{v}\) is \(u_i+v_i\), so the \(i\)-th component of \(c(\mathbf{u}+\mathbf{v})\) is

$$ c\,(u_i+v_i) = c\,u_i + c\,v_i, $$

where the equality is just the distributive law for ordinary real numbers. On the right, the \(i\)-th component of \(c\mathbf{u}+c\mathbf{v}\) is \(c\,u_i + c\,v_i\) by definition. The two sides agree in every component \(i=1,\dots,n\), and two vectors that agree component-by-component are equal. \(\blacksquare\)

Notice the pattern: a vector identity is proved by reducing it to a scalar identity in each coordinate. You’ll reuse this move constantly.

Problem 3 · code

Using NumPy, confirm that vector addition is commutative for \(\mathbf{u}=(1,2,3)\) and \(\mathbf{v}=(4,5,6)\), and that \(2(\mathbf{u}-\mathbf{v})\) equals \(2\mathbf{u}-2\mathbf{v}\).

Show solution 
import numpy as np
u = np.array([1, 2, 3]); v = np.array([4, 5, 6])
print(np.array_equal(u + v, v + u))            # True  (commutative)
print(np.array_equal(2*(u - v), 2*u - 2*v))    # True  (distributive)

Both print True. Numerical checks like these won’t prove a law (they test specific inputs), but they’re a fast sanity check and catch typos in your formulas.

Problem 4 · reasoning

A robot is at position \(\mathbf{p}=(5,5)\) and wants to reach target \(\mathbf{t}=(8,1)\). What single vector should it add to its position to get there, and what does that vector represent geometrically?

Show solution

It needs the displacement that takes \(\mathbf{p}\) to \(\mathbf{t}\), which is \(\mathbf{t}-\mathbf{p}=(8-5,\;1-5)=(3,-4)\). Adding it checks out: \(\mathbf{p}+(\mathbf{t}-\mathbf{p})=\mathbf{t}\). Geometrically, \(\mathbf{t}-\mathbf{p}\) is the arrow pointing from the robot to the target — “go 3 right and 4 down.” (This is exactly the “points to the first-written vector” rule: \(\mathbf{t}-\mathbf{p}\) points to \(\mathbf{t}\).)
