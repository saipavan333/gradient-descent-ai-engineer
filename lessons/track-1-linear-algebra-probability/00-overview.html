Welcome — and how to learn this

A short orientation before Track 1. What we’re doing, why this order, and how to study so the ideas actually stick.

Track 1 · Lesson 0 Orientation ~12 min read No prerequisites

You don’t need any background to start. If you can do middle-school arithmetic and you’re willing to think slowly, you have everything required. We will build every other idea — vectors, matrices, probability, calculus, and eventually neural networks and transformers — from the ground up, in an order where nothing ever depends on something you haven’t learned yet.

① The goal, stated plainly

By the end of this curriculum you will be able to do three things that define a researcher: derive the core ideas of modern machine learning from first principles, implement models from scratch and run them, and read, reproduce, and extend research papers. We get there one small, solid step at a time.

Track 1 is the language everything else is written in: linear algebra (how to describe data and transformations) and probability (how to describe uncertainty). Almost every equation you will ever meet in ML is built from these two. So we start here.

② How every lesson is taught

Each lesson follows the same loop, deliberately. We build a mental picture first, then make it precise, then turn it into code you run, then poke at it with an experiment. Intuition tells you what should be true; math tells you what is true and why; code proves you actually understand it; experiment keeps you honest.

📦 What every lesson contains

A Concept (intuition before formulas, every symbol introduced before it’s used); a worked Derivation or from-scratch Implementation that runs; Key results worth memorizing; a Quiz whose wrong answers are tempting (each explained); Practice problems with full solutions; and a Deep dive with primary sources. The test for a lesson is always the same: could you now derive it, implement it, and explain it to a peer?

③ How to actually study this

The difference between reading about math and knowing it is doing it with your own hand. A few habits that matter more than they sound:

  • Predict before you reveal. When you hit a boxed result or a solution, cover it and try to produce it yourself first. Being wrong and then seeing why is how the idea sticks.
  • Re-derive, don’t re-read. If a step isn’t obvious, write it out on paper until it is. Re-reading feels productive but mostly isn’t.
  • Run the code, then break it. Change a number, predict what happens, run it, and see if you were right. Every lesson’s code is built to be poked at.
  • Use the quizzes as retrieval. Answering from memory is a far stronger signal than recognizing the right answer when you see it.

⑤ The road through Track 1

Here’s the path. Each builds directly on the one before it.

Track 1 · Linear Algebra & Probability

  1. Vectors — the single object that represents a piece of data.
  2. Vector operations — adding and scaling, the only two moves there are.
  3. The dot product — length, angle, and similarity in one number.
  4. Span & basis — what a set of vectors can and cannot reach.
  5. Matrices as transformations — functions that move space itself.
  6. Matrix multiplication — composing transformations.
  7. …then determinants, eigenvectors, and the whole of probability (coming next).

🧩 Check your understanding

🧩 Quick check
These are about how to use the course. Answer from memory.

Q1. You reach a boxed “Key result.” What’s the most effective thing to do?

  • Highlight it and move on, so you can find it later.
    Highlighting feels like studying but creates almost no memory. Recognition isn’t recall.
  • Cover it, try to state or derive it yourself, then check.
    Correct. Retrieving an answer from memory — even unsuccessfully — is what builds durable understanding.
  • Copy it word-for-word into notes.
    Transcription is passive. You can copy a sentence perfectly and still not understand it.
  • Read it three times in a row.
    Re-reading produces a feeling of fluency without the underlying knowledge. One active recall beats many re-reads.

Q2. A derivation step isn’t obvious to you. What does this course expect you to do?

  • Trust it — the author surely checked it.
    Trust is how misunderstandings hide. The goal is for you to be able to reproduce every step.
  • Skip it; the conclusion is what matters.
    In research the steps are the skill. A conclusion you can’t derive is one you can’t extend or debug.
  • Work it out on paper until it’s obvious, demanding the missing step.
    Correct. “No hand-waving” cuts both ways: the lessons won’t skip steps, and neither should you.
  • Wait until a later lesson explains it.
    The path is ordered so nothing depends on later material. If a step is unclear now, the pieces to understand it are already in front of you.

🔬 Deep dive & sources

The learning techniques above aren’t folklore — they’re among the most replicated findings in cognitive science.

  • paper Roediger & Karpicke (2006), “Test-Enhanced Learning,” Psychological Science — retrieval practice beats re-study. doi:10.1111/j.1467-9280.2006.01693.x
  • review Dunlosky et al. (2013), “Improving Students’ Learning…,” Psych. Science in the Public Interest — ranks practice testing & spacing highest. doi:10.1177/1529100612453266
  • book Brown, Roediger & McDaniel, Make It Stick (2014) — a readable summary of the science of durable learning.
