Matrix multiplication = composition

The “row-times-column” rule looks arbitrary when you first meet it. It isn’t. It’s forced on us by one demand: multiplying matrices should mean doing one transformation after another. Once you see that, the rule — and why order matters — becomes inevitable.

Track 1 · Lesson 6 Concept + derivation + code ~28 min Builds on: Lessons 1–5

💡 Concept: stacking transformations

In Lesson 5 a matrix became a transformation of space. The obvious next question: what if you apply two in a row — first \(B\), then \(A\)? Doing one function after another is called composition, and the entire point of matrix multiplication is to package that composition into a single new matrix, written \(AB\). The defining property we insist on is:

$$ (AB)\,\mathbf{x} \;=\; A\,(B\,\mathbf{x}) \qquad\text{for every vector } \mathbf{x}. $$
(†)

In words: applying the product \(AB\) to \(\mathbf{x}\) gives the same result as first applying \(B\), then applying \(A\) to whatever \(B\) produced. Read \(AB\) right-to-left — \(B\) acts first. (That feels backwards at first, but it matches function notation \(f(g(x))\): the inner function \(g\) runs first.) Everything about how to multiply matrices is squeezed out of this one requirement.

🧮 Deriving the rule (it’s forced, not arbitrary)

Let’s discover the formula for \(AB\) rather than memorize it. From Lesson 5 we know a transformation’s matrix is built column by column: column \(j\) is the image of the basis vector \(\mathbf{e}_j\). So to find \(AB\), we just need to know where the composition sends each \(\mathbf{e}_j\). Follow \(\mathbf{e}_j\) through both steps:

$$ \mathbf{e}_j \;\xrightarrow{\;B\;}\; B\mathbf{e}_j \;=\; (\text{column } j \text{ of } B) \;\xrightarrow{\;A\;}\; A\,(\text{column } j \text{ of } B). $$

Therefore column \(j\) of \(AB\) is \(A\) applied to column \(j\) of \(B\). That’s already a complete, meaningful description. To get the entry-level formula, recall the rows view: the \(i\)-th entry of “\(A\) times a vector” is row \(i\) of \(A\) dotted with that vector. Taking that vector to be column \(j\) of \(B\):

$$ (AB)_{ij} \;=\; (\text{row } i \text{ of } A)\cdot(\text{column } j \text{ of } B) \;=\; \sum_{k} a_{ik}\,b_{kj}. $$
(‡)

There it is — the famous rule, derived from “do \(B\) then \(A\).” Each output entry is a dot product of a row of \(A\) with a column of \(B\):

⚠️ The shape rule (and why it must hold)

For row \(i\) of \(A\) and column \(j\) of \(B\) to dot together, they need the same length — so the number of columns of \(A\) must equal the number of rows of \(B\). Schematically, \((m\times k)(k\times n)=(m\times n)\): the inner \(k\)’s must match and cancel, the outer dimensions give the result’s shape. This is the same “inner dimensions match” rule from Lesson 5, now between two matrices.

🎯 Order matters: \(AB \neq BA\)

Multiplying ordinary numbers, \(3\times 5 = 5\times 3\). Matrices are not like that, and the composition picture tells you instantly why: rotating-then-shearing is a different physical operation from shearing-then-rotating, so they land in different places. Watch the same shape go through both orders:

It is, however, associative: \(A(BC)=(AB)C\). This is free from the composition view — both sides mean “apply \(C\), then \(B\), then \(A\),” and grouping which pair you combine first can’t change the end-to-end effect. Associativity is what lets a deep network fold a long chain of layer-matrices together, and it’s why we can write \(ABC\) without parentheses. There is also an identity matrix \(I\) (ones on the diagonal, zeros elsewhere) that does nothing: \(AI=IA=A\), the matrix version of multiplying by \(1\).

🤖 Why ML is a tower of matrix multiplications

A deep neural network is a composition of layers, so a forward pass is a sequence of matrix multiplications — on modern hardware, that’s essentially all a GPU is doing. Composition also delivers a profound lesson about why networks need nonlinearities: if you stack two linear layers with no nonlinearity between them, \(W_2(W_1\mathbf{x}) = (W_2 W_1)\mathbf{x}\), which is just one linear layer with matrix \(W_2 W_1\). No matter how many you stack, the composition collapses to a single matrix — a network that can only ever draw straight boundaries. The squiggle of activation functions between layers is precisely what stops this collapse. You couldn’t have understood that argument before this lesson; now it’s obvious.

🤖 One line, remembered

“Stacked linear maps multiply into one matrix” \(\;\Rightarrow\;\) depth without nonlinearity buys you nothing. This is the reason ReLU/GELU exist, and you’ll exploit it directly when you build MLPs in Track 4.

🛠️ From scratch: matrix multiplication

The definition \((‡)\) is a triple loop. We’ll write it, check it against NumPy, then verify both the composition law \((†)\) and non-commutativity in code.

code/track-1/matrix_multiply.py
import numpy as np

def matmul_from_scratch(A, B):
    """C[i,j] = sum_k A[i,k] * B[k,j]  -- the definition, by hand."""
    m, k = A.shape
    k2, n = B.shape
    assert k == k2, f"shape mismatch: {A.shape} x {B.shape} (inner dims must match)"
    C = np.zeros((m, n))
    for i in range(m):
        for j in range(n):
            s = 0.0
            for kk in range(k):
                s += A[i, kk] * B[kk, j]     # row i of A, column j of B
            C[i, j] = s
    return C

A = np.array([[1.0, 2.0],
              [3.0, 4.0]])
B = np.array([[5.0, 6.0],
              [7.0, 8.0]])

print(matmul_from_scratch(A, B))   # [[19. 22.] [43. 50.]]
print(A @ B)                        # [[19. 22.] [43. 50.]]  -> matches

# Composition law (dagger):  (AB)x == A(Bx)
x = np.array([1.0, -1.0])
print(np.allclose((A @ B) @ x, A @ (B @ x)))   # True

# Non-commutativity: AB != BA in general
print(np.allclose(A @ B, B @ A))               # False

# Stacking two LINEAR layers collapses to one matrix (no nonlinearity!)
W1 = np.random.randn(4, 3)
W2 = np.random.randn(2, 4)
x  = np.random.randn(3)
print(np.allclose(W2 @ (W1 @ x), (W2 @ W1) @ x))   # True -> one effective matrix

The triple loop is \(O(mnk)\) and deliberately slow — it’s for understanding. NumPy’s @ calls decades-optimized linear-algebra libraries (BLAS); on a GPU the same operation is what your hardware is built to do thousands of times per layer. Same math, wildly different speed.

🔑 Key results to remember

Lock these in

  1. Meaning: \(AB\) is the single matrix for “apply \(B\), then \(A\)” — composition. Read right-to-left.
  2. Rule: \((AB)_{ij}=\) row \(i\) of \(A\) dotted with column \(j\) of \(B\) \(=\sum_k a_{ik}b_{kj}\). It’s derived from \((†)\), not arbitrary.
  3. Shapes: \((m\times k)(k\times n)=(m\times n)\); inner dimensions must match.
  4. Not commutative: \(AB\neq BA\) in general. But associative: \(A(BC)=(AB)C\); and \(AI=IA=A\).
  5. Stacked linear layers multiply into one matrix \(W_2W_1\) — which is exactly why deep nets need nonlinearities.

🧩 Check your understanding
Answer from memory, then read why.

Q1. Compute the top-left entry of \(AB\) where \(A=\begin{bmatrix}1&2\\3&4\end{bmatrix}\), \(B=\begin{bmatrix}0&5\\6&7\end{bmatrix}\).

  • \(0\)
    This multiplies only the top-left entries \(1\cdot0\). But each entry is a full dot product of a row with a column, not a single product.
  • \(5\)
    Looks like \(1\cdot5\) using the wrong column. Entry \((1,1)\) uses row 1 of \(A\) and column 1 of \(B\), which is \((0,6)\).
  • \(12\)
    Correct. Row 1 of \(A\) is \((1,2)\); column 1 of \(B\) is \((0,6)\); their dot product is \(1\cdot0+2\cdot6=12\).
  • \(17\)
    That’s row 1 of \(A\) dotted with column 2 of \(B\): \(1\cdot5+2\cdot6=17\) — that’s entry \((1,2)\), not \((1,1)\).

Q2. In the product \(AB\) applied to \(\mathbf{x}\), which transformation acts on \(\mathbf{x}\) first?

  • \(A\), because it’s written first.
    Reading order is left-to-right, but application order is right-to-left, exactly like \(f(g(x))\). \(A\) acts last.
  • \(B\), because \((AB)\mathbf{x}=A(B\mathbf{x})\).
    Correct. The parentheses show \(B\) touches \(\mathbf{x}\) first, then \(A\) acts on the result. Read products right-to-left.
  • They act simultaneously.
    Composition is sequential, not simultaneous — one transformation feeds into the next.
  • It depends on the matrices.
    It never depends on the matrices: by definition \((AB)\mathbf{x}=A(B\mathbf{x})\), so \(B\) is always first.

Q3. \(A\) is \(2\times 3\) and \(B\) is \(3\times 4\). Which products are defined, and what shape is \(AB\)?

  • Both \(AB\) and \(BA\); \(AB\) is \(2\times 4\).
    \(BA\) would be \((3\times4)(2\times3)\) — inner dims \(4\) and \(2\) don’t match, so \(BA\) is undefined.
  • Only \(BA\) is defined.
    Backwards. \(BA\)’s inner dimensions (4 then 2) clash; it’s \(AB\) whose inner dims (3 and 3) match.
  • Neither is defined.
    \(AB\) is defined: \((2\times3)(3\times4)\) has matching inner dimension 3.
  • Only \(AB\) is defined; it is \(2\times 4\).
    Correct. \((2\times3)(3\times4)\): inner 3’s match and cancel, leaving the outer dimensions \(2\times4\). \(BA\) is undefined.

Q4. Why must a deep network put a nonlinearity between linear layers?

  • Without it, stacked linear layers collapse to a single matrix \(W_2W_1\), so depth adds no power.
    Correct. \(W_2(W_1\mathbf{x})=(W_2W_1)\mathbf{x}\) by associativity — many linear layers equal one. Nonlinearities break this collapse, letting the network bend space.
  • Because matrix multiplication is not commutative.
    Non-commutativity is true but unrelated here; the issue is that composing linear maps yields another linear map, regardless of order.
  • To make the matrices square.
    Layers are routinely non-square; squareness isn’t required and isn’t the reason for nonlinearities.
  • To speed up training.
    Nonlinearities are about expressive power (escaping pure linearity), not speed.

✍️ Practice

Problem 1 · compute

With \(A=\begin{bmatrix}1&2\\3&4\end{bmatrix}\) and \(B=\begin{bmatrix}5&6\\7&8\end{bmatrix}\), compute both \(AB\) and \(BA\) and confirm they differ.

Show solution

\(AB\): entries are row\(_i(A)\cdot\)col\(_j(B)\):

$$ AB=\begin{bmatrix}1\cdot5+2\cdot7 & 1\cdot6+2\cdot8\\ 3\cdot5+4\cdot7 & 3\cdot6+4\cdot8\end{bmatrix}=\begin{bmatrix}19&22\\43&50\end{bmatrix}. $$

\(BA\): now rows of \(B\) dot columns of \(A\):

$$ BA=\begin{bmatrix}5\cdot1+6\cdot3 & 5\cdot2+6\cdot4\\ 7\cdot1+8\cdot3 & 7\cdot2+8\cdot4\end{bmatrix}=\begin{bmatrix}23&34\\31&46\end{bmatrix}. $$

\(AB\neq BA\) — concrete proof that order matters.

Problem 2 · derivation

Prove that matrix multiplication is associative, \(A(BC)=(AB)C\), without touching the entry formula — using only the composition property \((†)\).

Show solution

Two matrices are equal iff they send every vector to the same place (same action ⇒ same columns ⇒ same matrix). So take any \(\mathbf{x}\) and apply each side, repeatedly using \((†)\):

$$ \big(A(BC)\big)\mathbf{x} = A\big((BC)\mathbf{x}\big) = A\big(B(C\mathbf{x})\big). $$

$$ \big((AB)C\big)\mathbf{x} = (AB)\big(C\mathbf{x}\big) = A\big(B(C\mathbf{x})\big). $$

Both equal \(A(B(C\mathbf{x}))\) for every \(\mathbf{x}\), so the two product matrices act identically and are therefore equal. \(\blacksquare\) This is the power of the transformation view: a fact that’s tedious with sums of indices is a two-line truth about composing functions.

Problem 3 · code

Empirically confirm associativity for three random \(3\times 3\) matrices, and confirm the identity matrix \(I\) satisfies \(AI=A\).

Show solution 
import numpy as np
A, B, C = (np.random.randn(3, 3) for _ in range(3))
print(np.allclose(A @ (B @ C), (A @ B) @ C))   # True  (associative)
I = np.eye(3)                                   # identity matrix
print(np.allclose(A @ I, A))                    # True
print(np.allclose(I @ A, A))                    # True

np.allclose (not array_equal) is the right check here: floating-point arithmetic makes the two groupings differ in the last decimal places, so we test “equal up to tiny rounding,” which is the honest standard for numerical work.

Problem 4 · reasoning

A rotation by \(30^\circ\) is \(R\); a rotation by \(60^\circ\) is \(S\). What single matrix is \(SR\), and does \(RS\) equal it here? Why is this case special?

Show solution

\(SR\) means “rotate \(30^\circ\), then \(60^\circ\)” = rotate \(90^\circ\). \(RS\) means “rotate \(60^\circ\), then \(30^\circ\)” = also \(90^\circ\). So here \(SR=RS\). Rotations about the same point commute because angles simply add, and addition is commutative. This does not contradict “matrices generally don’t commute” — it’s a special family (2-D rotations) that happens to. In general (e.g. a rotation and a shear, Figure 6.2) order matters. Spotting which special structures commute is a genuinely useful research instinct.
