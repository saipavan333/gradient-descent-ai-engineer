Language modeling basics

⏱ 23 min Lesson 5.3 ▶ Open in Colab
Why this existsA language model predicts the next token — one simple, self-supervised objective that, at scale, yields grammar, facts, and reasoning. It is the objective behind every model from autocomplete to GPT.
By the end, you can
  • Define next-token prediction and self-supervision
  • Contrast n-gram and neural language models
  • Interpret perplexity
  • Explain autoregressive generation

Concept One objective to rule them all: predict the next token

A language model does one deceptively simple thing: given some text so far (the context), it predicts what comes next. More precisely, it outputs a probability distribution over the entire vocabulary — how likely each possible next token is. Given "the cat sat on the ___", a good model puts high probability on "mat" and "floor," low on "democracy."

That's the whole training objective, and it's astonishingly powerful. To predict the next token well across all of human text, a model is implicitly forced to learn grammar, facts, reasoning patterns, and style — because all of those help it guess better. This is self-supervised learning: the "labels" are free, since the next token in any text is the answer, so you can train on essentially the entire internet without anyone labeling anything (Lesson 3.1's supervised setup, but with the data labeling itself).

Concept From counting to neural networks

The oldest language models are n-grams: just count. A bigram model estimates "what follows 'the'?" by counting how often each word followed "the" in a corpus. Simple and fast, but it has fatal flaws: it can only look back a word or two (longer contexts are seen too rarely to count), and it can't generalize — if it never saw "the purple elephant," it assigns it zero probability, even though it's perfectly sensible.

Neural language models fix both. By representing context with embeddings (Lesson 5.2) and processing them with a network, they generalize — "purple elephant" gets reasonable probability because "purple" and "elephant" have sensible vectors even in a new combination — and, with transformers (the rest of this track), they condition on thousands of previous tokens, not two. We measure a language model's quality with perplexity: roughly, how "surprised" it is by real text (lower is better — a perplexity of 10 means it's about as uncertain as choosing uniformly among 10 options at each step).

Concept Generation = predict, append, repeat

Once a model can predict the next token, it can generate text by doing so repeatedly: predict a token, append it to the context, predict the next, and so on. This is called autoregressive generation — each new token becomes part of the input for the next step.

So a chatbot writing a paragraph is literally running next-token prediction hundreds of times, each time conditioned on everything written so far (your prompt plus its own output). The simple objective and this simple loop are the entire engine behind modern text generation.

Worked example A bigram language model by counting

Let's build the simplest possible language model — a bigram counter — and read off its next-token distribution. It estimates $P(\text{next} \mid \text{previous word})$ purely from how often pairs co-occur in a tiny corpus.

In this corpus, "the" was followed by "cat" twice and "mat" once, so the model predicts $P(\text{cat} \mid \text{the}) = 0.67$ and $P(\text{mat} \mid \text{the}) = 0.33$. To generate, you'd sample from this distribution, append the result, and repeat. The limitations are immediately visible: it only looks back one word (it has no idea "the" came after "sat on"), and any word-pair it never saw gets probability 0. A neural model replaces this lookup table with a network over embeddings that conditions on the whole context and generalizes to unseen combinations — but the objective is identical to this toy: put probability mass on the token that actually comes next. Everything in the rest of this track is about doing this one thing dramatically better.

★ Key points to remember
  • A language model predicts the next token as a probability distribution over the vocabulary.
  • This objective is self-supervised — the next token is its own label — so you can train on unlabeled text at scale.
  • n-gram models just count (short context, no generalization); neural models use embeddings to generalize and see long context.
  • Perplexity measures how well a model predicts text (lower = better).
  • Generation is autoregressive: predict a token, append it, repeat.

Practice Build the reasoning yourself

1Using the worked example's corpus, what tokens can follow "cat," and with what probabilities? ("the cat sat on the mat the cat ran")
Show solution & reasoning

Find every place "cat" appears and see what comes next: "the cat sat" and "the cat ran". So "cat" is followed by "sat" once and "ran" once. With a total of 2: $P(\text{sat} \mid \text{cat}) = 0.5$ and $P(\text{ran} \mid \text{cat}) = 0.5$.

Notice the model has no way to prefer one — both are equally likely given only "cat." A real model conditioned on more context ("the cat sat on the mat the cat ___") might lean toward "ran" because it already "used" sat, but a bigram is blind to that. This is exactly the short-context limitation that motivates everything ahead.

2Explain how a single objective — "predict the next token" — could force a model to learn arithmetic, facts, and even translation, without ever being explicitly taught them.
Show solution & reasoning

Because predicting the next token well on diverse text requires those abilities as a byproduct. To complete "2 + 2 =" the model must learn arithmetic; to complete "The capital of France is" it must learn facts; to complete "The French word for cat is" it must learn translation; to continue a story coherently it must track characters and reasoning. The training data contains all of these patterns, and the only way to minimize next-token error across all of it is to internalize the underlying skills.

This is the profound insight behind large language models: a simple, scalable objective on enough diverse data yields broad capabilities for free, because so many tasks can be framed as "predict what text comes next." It's why the same model can write code, answer questions, and translate — none were trained as separate tasks; they all fell out of next-token prediction at scale (the heart of Track 6).

3Model A has perplexity 12 on a test set; Model B has perplexity 30. Which is better, and what does perplexity intuitively measure?
Show solution & reasoning

Model A is better — lower perplexity is better. Perplexity intuitively measures how "surprised" or "confused" the model is by real text: a perplexity of 12 means that, on average, the model is about as uncertain as if it were choosing uniformly among 12 equally-likely tokens at each step, whereas Model B is as uncertain as among 30. Less surprise = better predictions = a better language model.

It's essentially the exponential of the average cross-entropy loss (Lesson 3.3) on held-out text, so minimizing next-token loss is minimizing perplexity. It's the standard intrinsic metric for language models, though for real applications you also evaluate downstream task performance (Track 11), since low perplexity doesn't guarantee a model is helpful or safe.

★ Deep dive: why next-token prediction scales into intelligence▸

It's worth sitting with how unreasonable it is that "predict the next token" produces systems that write essays and code. The bet — borne out empirically — is that compression is comprehension: to compress text into a model that predicts it well, you must discover the regularities that generate it, which are grammar, facts, and reasoning. A model that truly minimized next-token loss on all human text would have to model the processes that produced that text, including the thoughts behind it. We're not there, but the trajectory — bigger models, more data, lower loss, broader capabilities — has held remarkably steadily, which is the scaling story of Track 6.

Two technical points make this concrete. First, the loss being minimized is exactly the cross-entropy from Lesson 3.3, averaged over every position in the training text — so a language model is "just" a giant classifier predicting one of ~50,000 classes (the vocabulary) at each step, trained with the same loss as logistic regression. The architecture (transformers) is what makes it good; the objective is humble. Second, the autoregressive setup has a subtle consequence called exposure bias: during training the model always sees correct previous tokens (from real text), but during generation it sees its own previous tokens, so a single early mistake can compound ("hallucination" and topic drift partly stem from this). Techniques in Track 6 (better decoding, RLHF) and Track 11 (evaluation) address these. But the core loop you learned here — predict the next token, append, repeat — is, scaled to billions of parameters and trillions of tokens, the entire foundation of generative AI. The next three lessons build the architecture (attention, then the transformer) that turned this simple objective into something that works astonishingly well.

To predict the next token well, a model must figure out which earlier words matter for the current one. The mechanism that does this — and powers every modern model — is next. In 5.4 Attention, intuitively, we build the intuition before the math.
