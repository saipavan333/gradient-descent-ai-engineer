NumPy arrays & vectorization

⏱ 26 min Lesson 2.2 ▶ Open in Colab
Why this existsNumPy's array is the structure under everything in ML, and vectorization is what makes Python fast enough for real numerical work. Master arrays and you've mastered the substrate the field runs on.
By the end, you can
  • Create arrays and read shape, dtype, and axes
  • Replace loops with vectorized operations
  • Index and slice arrays, including boolean masks
  • Know when you have a view versus a copy

Concept The array: one data structure under all of ML

Every dataset, image, sound clip, and neural-network tensor is, underneath, a NumPy array (its full name is ndarray, for "n-dimensional array"). An array is just a grid of numbers, all of the same type. Three properties describe any array, and you'll reach for them constantly:

  • shape — the size along each dimension, written as a tuple (a fixed, comma-separated list in parentheses). A table with 3 rows and 4 columns has shape (3, 4); a batch of 64 color 32×32 images has shape (64, 3, 32, 32).
  • dtype — short for "data type," the kind of number every element is (int64 for whole numbers, float32 for decimals, bool for true/false). Having one dtype for the whole array is what makes it fast.
  • axes — the numbered directions of the grid. axis=0 runs down the rows; axis=1 runs across the columns. Getting axes right is half of NumPy.

You create arrays in a handful of ways: np.array([1,2,3]) from a Python list, np.zeros((3,4)) / np.ones(...) for grids pre-filled with 0s or 1s, np.arange(12) for the counting sequence 0…11, np.linspace(0,1,5) for 5 evenly-spaced points between 0 and 1, and rng.normal(...) for random numbers. .reshape() rearranges the same numbers into a new shape (e.g. turn a flat list of 12 into a 3×4 grid).

Concept Vectorization: operate on whole arrays, not one element at a time

The single most important habit in numerical Python is vectorization: write an operation on the entire array at once instead of writing a loop (a "do this for each item" instruction) that visits elements one by one. The expressions a + b, a * 2, and np.sqrt(a) all apply element-wise (to every entry, position by position) — and the actual looping happens in fast compiled C, not in slow Python.

★
The mantra

If you find yourself writing a for loop over the elements of an array, pause — there's almost always a vectorized one-liner that's both shorter and dramatically faster. "Remove the loop" is the recurring move of the whole field; it's why GPUs (which do thousands of operations at once) power deep learning.

Concept Indexing, slicing, and boolean masks

You pull parts of an array out by position or by condition. An index is a position number (counting starts at 0, so a[0] is the first element). Slicing grabs a range with start:stop for each axis: a[1:3, 0:2] takes rows 1–2 and columns 0–1 (important quirk: the stop index is excluded, so 1:3 gives positions 1 and 2, not 3). A boolean mask selects by a true/false test: a[a > 5] keeps exactly the elements where the test is true — no loop needed. ("Boolean" just means true/false.)

Worked example Vectorization, timed — and masking in action

Let's normalize a million numbers to "z-scores" (subtract the mean, divide by the standard deviation — i.e. re-express each value as "how many spreads above or below average it is") in two ways — a hand-written loop and a vectorized expression — and time both. Then filter with a boolean mask.

Same answer, ~34× faster, and the vectorized version is one line. That gap is why you avoid element loops. Now boolean masking — find and summarize the unusually large values:

big = x[x > 70]                      # keep only elements above 70
print("count above 70:", big.size)
print("their mean:", big.mean().round(2))

# Masks can also WRITE: lift everything below 30 up to 30, in place, no loop
x[x < 30] = 30
Outputcount above 70: 23017 their mean: 73.72

The expression x > 70 produces an array of true/false values (the mask), and indexing with that mask keeps the elements where it's true. Reading and writing through masks — instead of looping with if statements — is the NumPy way, and it's exactly how you'll filter rows, apply thresholds, and clean data throughout this track.

★ Key points to remember
  • An array has a shape (sizes per dimension), one dtype, and numbered axes (0 = down rows, 1 = across columns).
  • Vectorize: operate on whole arrays (a + b, np.sqrt(a)) — element-wise, in fast C, 10–100× faster than a Python loop.
  • If you're looping over an array's elements in NumPy, there's almost certainly a vectorized one-liner instead.
  • Slice by position (a[1:3, 0:2], stop excluded, counting from 0); mask by condition (a[a > 5]).
  • Masks can read and write (a[a < 0] = 0) — no if loop needed.

Practice Build the reasoning yourself

1Without a loop, write NumPy to compute the element-wise square of a = np.array([1, 2, 3, 4]), then the sum of those squares.
Show solution & reasoning
sq = a ** 2          # array([1, 4, 9, 16])  -- element-wise square
total = sq.sum()     # 30
# or in one line: (a ** 2).sum()

The reasoning: ** 2 applies the square to every element (vectorized), and .sum() adds the results into one number. No index, no loop. As a bonus, (a ** 2).sum() is exactly the squared L2 norm from Lesson 1.1 — Track 1's math, now as one fast NumPy expression.

2Given a temperature array temps, write one line that replaces every reading below 0 with 0 (clip cold values to freezing), in place.
Show solution & reasoning
temps[temps < 0] = 0

The reasoning: temps < 0 builds a true/false mask of the cold positions, and assigning to that masked selection writes 0 into exactly those slots — a vectorized conditional replacement that would otherwise be a for loop with an if. (NumPy also offers np.clip(temps, 0, None) for the same effect without changing the original.)

3You expect a $10 \times 10$ array, but a.shape prints (100,). Your later code a[2, 3] then errors. What happened, and how do you fix it?
Show solution & reasoning

The array is 1-D (100 elements in a single row), not 2-D, so two-position access a[2, 3] is invalid — NumPy expects one position for a 1-D array. Fix it with a = a.reshape(10, 10) (same numbers, rearranged into 10 rows × 10 columns); then a[2, 3] works.

The reasoning: shape bugs are the #1 source of NumPy errors. The habit to build is print the shape whenever something behaves oddly — the shape almost always explains the error message. You'll lean on this constantly once broadcasting (next lesson) enters the picture.

★ Deep dive: views vs. copies, and the dtype traps that bite beginners▸

Views vs. copies. A NumPy slice usually returns a view — a window onto the same underlying memory, not a fresh array. So b = a[0:3]; b[0] = 999 will change a[0] too, which surprises everyone the first time. This is a feature (slicing is free — no copying gigabytes), but it means you should call .copy() when you want an independent array. Boolean indexing and fancy indexing (selecting with a list of positions), by contrast, return copies. When in doubt, check with np.shares_memory(a, b).

dtype traps. Because an array has one fixed type, a few sharp edges appear — and none of them throw an error, they just produce wrong numbers, which is exactly why they're worth knowing:

  • Integer overflow. np.array([200], dtype=np.uint8) + 100 silently wraps around to 44 (8-bit math caps at 255). Use a wider dtype (int64) or floats when values may grow.
  • Accidental truncation. Storing a decimal into an integer array drops the fraction: a = np.array([1,2,3]); a[0] = 2.9 keeps 2, not 2.9.
  • float32 vs float64. Deep learning uses float32 (half the memory, fast on GPUs) and sometimes float16; scientific code often defaults to float64. Mixing them can quietly cost precision or speed, so be deliberate.

A five-second print(a.dtype, a.shape) when results look off will save you hours, and it's the single most useful debugging reflex in numerical Python.
