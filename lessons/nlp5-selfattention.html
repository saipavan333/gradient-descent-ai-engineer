Self-attention & multi-head attention

⏱ 28 min Lesson 5.5 ▶ Open in Colab
Why this existsSelf-attention — softmax(QKᵀ/√d)V — is the one equation behind transformers. Understand the attention matrix, the √d scaling, and multi-head attention and you understand the computational core of modern AI.
By the end, you can
  • Compute self-attention from scratch
  • Explain the √dₖ scaling
  • Read an attention matrix
  • Describe multi-head attention

Concept Self-attention: every token attends to every token

Self-attention applies the attention from Lesson 5.4 within a single sequence: every token is a Query and a Key and a Value, so each token attends to every token (itself included) and rebuilds its representation as a blend of the whole sentence, weighted by relevance. "Self" because the queries, keys, and values all come from the same sequence. Done for all tokens at once, it's just a few matrix multiplies — the entire computation fits in one line:

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V$$

Read the equation right to left through the diagram. $QK^\top$ produces an $n \times n$ matrix of scores — every token's Query dotted with every token's Key. We divide by $\sqrt{d_k}$ (the square root of the key dimension) for scaling: without it, dot products in high dimensions grow large, pushing softmax into a near-one-hot, low-gradient regime; the $\sqrt{d_k}$ keeps scores moderate so training stays stable. Softmax (applied to each row) gives the attention weights, and multiplying by $V$ produces each token's output as the weighted blend of all Values.

Concept The attention matrix

That $n \times n$ score matrix, after softmax, is the attention matrix: row $i$ tells you how much token $i$ attends to every other token (and the row sums to 1). It's the most interpretable artifact in a transformer — you can literally read off which words each word is looking at.

Concept Multi-head attention: several at once

One attention operation captures one kind of relationship. Multi-head attention runs several in parallel — each "head" has its own learned Q/K/V projections, so each can specialize. One head might track subject–verb agreement, another coreference ("it" → "cat"), another nearby word order. Their outputs are concatenated and linearly combined, giving the model a rich, multi-faceted view in a single layer.

Worked example Self-attention on a sequence, from scratch

We'll run the full equation on a tiny 3-token sequence with dimension 4. For clarity we use the embeddings directly as Q, K, and V (a real layer multiplies by learned $W_Q, W_K, W_V$ first, but the attention math is identical).

Read the attention matrix. Token 0 attends most to itself (0.51) — its Query matches its own Key best — and gives some attention to token 2 (0.31), which shares a feature with it. Token 2, which shares features with both others, spreads its attention more evenly (0.27, 0.27, 0.45). Every row sums to 1 (it's a softmax), and the output is each token's new representation: a weighted blend of all the values. Token 2's output [0.73, 0.73, …] is a genuine mixture of tokens 0 and 1 — it has absorbed context from both. That's self-attention in full: a few matrix multiplies turn a bag of token vectors into context-aware representations. Stack this with an MLP and you have a transformer block (next lesson).

★ Key points to remember
  • Self-attention: every token is Query, Key, and Value, so each attends to all tokens and is rebuilt as a context-weighted blend.
  • The whole thing is one equation: $\text{softmax}(QK^\top/\sqrt{d_k})\,V$ — a couple of matrix multiplies.
  • The $\sqrt{d_k}$ scaling keeps scores moderate so softmax gradients stay healthy.
  • The attention matrix (rows = queries, sum to 1) shows who attends to whom — the model's most readable internal signal.
  • Multi-head attention runs several attentions in parallel, each learning a different relationship, then concatenates them.

Practice Build the reasoning yourself

1A sequence has 3 tokens. What is the shape of the attention matrix $QK^\top$, and what does entry $(2, 0)$ mean?
Show solution & reasoning

With 3 tokens, $Q$ and $K$ are each $3 \times d$, so $QK^\top$ is $3 \times 3$ — one score for every (query, key) pair. Entry $(2, 0)$ is the dot product of token 2's Query with token 0's Key: how much token 2 attends to token 0 (before softmax).

In general, $n$ tokens give an $n \times n$ attention matrix. That quadratic size is the key cost of attention: for a 1,000-token context it's a million entries; for 100,000 tokens, ten billion. This $O(n^2)$ scaling is the bottleneck for long-context transformers and the target of much research (efficient/sparse attention) — a direct consequence of every token attending to every token.

2In the worked example, token 2 ([1,1,0,0]) spread its attention fairly evenly (0.27, 0.27, 0.45) while token 0 concentrated on itself (0.51). Explain why, from the embeddings.
Show solution & reasoning

Attention scores are dot products of the (here identical) Q and K vectors. Token 0 is [1,0,1,0]; its dot product with itself is 2 (high), with token 2 [1,1,0,0] is 1 (they share the first feature), and with token 1 [0,1,0,1] is 0 (no shared features). So token 0's scores are [2, 0, 1] → after scaling and softmax, it attends most to itself.

Token 2 [1,1,0,0] shares one feature with token 0 (the first) and one with token 1 (the second), so its dot products are [1, 1, 2] — more balanced — giving the evener spread (0.27, 0.27, 0.45). The lesson: a token attends to others in proportion to how aligned their key vectors are with its query, and a token that overlaps with everyone distributes its attention broadly. Real models learn the Q/K projections so these overlaps encode useful linguistic relationships rather than raw feature sharing.

3Self-attention treats its input as a set — if you shuffle the token order, the computation is unchanged except for the reordering. Why is that a problem for language, and what fixes it?
Show solution & reasoning

It's a problem because word order carries meaning: "dog bites man" and "man bites dog" have identical token sets but opposite meanings. Pure self-attention is permutation-equivariant — it computes the same relationships regardless of order — so on its own it can't tell those two sentences apart. It sees a bag of words, not a sequence.

The fix is positional encoding (Lesson 5.7): add a unique, position-dependent vector to each token's embedding before attention, so "dog" in position 1 differs from "dog" in position 3. Now the attention scores depend on where each token is, not just what it is, and the model can use order. This is why every transformer injects positional information — attention provides the "what relates to what," and positional encoding provides the "in what order."

★ Deep dive: the learned projections, multi-head mechanics, and the cost of attention▸

The worked example set $Q = K = V = X$ for clarity, but a real attention layer first projects the input through three learned matrices: $Q = XW_Q$, $K = XW_K$, $V = XW_V$. These projections are where the learning lives — $W_Q$ and $W_K$ are trained so that tokens which should attend to each other end up with aligned query/key vectors (large dot product), and $W_V$ shapes what information actually flows. Separating Key from Value lets a token be found by one criterion while contributing different content, which adds expressiveness. In multi-head attention with $h$ heads, the model splits the dimension into $h$ pieces, runs $h$ separate scaled-dot-product attentions (each with its own smaller $W_Q, W_K, W_V$), concatenates the $h$ outputs, and applies a final output projection $W_O$. Total compute is similar to one big attention, but the model gets $h$ different "views" — and interpretability studies find heads that genuinely specialize (a "previous-token head," a "coreference head," and so on).

The defining cost of self-attention is its quadratic complexity: with $n$ tokens, computing all pairwise scores is $O(n^2)$ in both time and memory. This is fine for a few thousand tokens but becomes prohibitive for very long documents — doubling the context quadruples the cost. It's why context windows were historically limited and why a whole research area chases cheaper attention: sparse attention (each token attends to only some others), linear attention approximations, FlashAttention (a memory-efficient exact implementation that made long contexts practical), and architectural tricks like sliding windows. Understanding that the power of attention (every token sees every token) is exactly what makes it expensive (every-to-every is $n^2$) is the key trade-off of the whole architecture — and the reason "long context" is both a major selling point and a hard engineering problem in modern LLMs (Track 6). For now, the elegant core stands: a couple of matrix multiplies, a softmax, and the model can relate any word to any other.

Self-attention mixes information across tokens. Wrap it with a feed-forward network, residual connections, and normalization, and you get the unit that, stacked, is the model. Next: 5.6 The Transformer architecture.
