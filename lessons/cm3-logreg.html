Logistic regression & classification

⏱ 27 min Lesson 3.3 ▶ Open in Colab
Why this existsMost real ML questions are yes/no, not how-much. Logistic regression bends the line into a probability, giving calibrated confidence and the threshold dial that powers every classifier.
By the end, you can
  • Map a linear score to a probability with the sigmoid
  • Read the linear decision boundary
  • Train with cross-entropy and know why not MSE
  • Extend to many classes with softmax

Concept From predicting numbers to predicting categories

Linear regression predicts a number. But a huge fraction of real problems ask a yes/no question instead: is this email spam? is this tumor malignant? will this customer churn? That's classification — predicting which class (category) an example belongs to. We'll start with two classes (binary classification), which we label 0 and 1.

The natural first idea — just use linear regression and round — fails, and it's worth seeing why. The line $z = \mathbf{w}\cdot\mathbf{x} + b$ can output any number from $-\infty$ to $+\infty$, but a class answer should be a probability between 0 and 1. We need to squash that unbounded score into the $[0,1]$ range. The function that does this is the sigmoid (also called the logistic function), written $\sigma$:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

▶
Take it to the lab

Open the Classifier Playground — drop your own points and watch logistic regression, k-NN, a decision tree and a small neural net each draw a completely different boundary on the same data.

So logistic regression is just: compute the linear score $z = \mathbf{w}\cdot\mathbf{x} + b$ exactly as before, then pass it through the sigmoid to get $\hat{y} = \sigma(z)$, the predicted probability that the example is class 1. The name is a historical quirk — it has "regression" in it but it's a classification model. It remains one of the most used models in the world: fast, interpretable, and a rock-solid baseline.

Concept The decision boundary is a straight line

To turn the probability into an actual decision, pick a threshold — usually 0.5 — and predict class 1 when $\hat{y} > 0.5$. Since $\sigma(z) > 0.5$ exactly when $z > 0$, the dividing line between the two predicted classes is where $\mathbf{w}\cdot\mathbf{x} + b = 0$. That's a straight line (in 2-D) or a flat hyperplane (in higher dimensions) — the decision boundary.

This is the model's key strength and its key limitation in one sentence: the boundary is always linear. That's wonderfully stable and interpretable, but it means logistic regression can't carve out a curved or circular region on its own — for that you'd engineer nonlinear features (as in Lesson 3.2) or reach for a tree or an SVM (Lessons 3.7, 3.9).

Concept The loss: cross-entropy, not squared error

How do we train it? We need a loss that's small when the predicted probability is confident and correct, and large when it's confident and wrong. That loss is cross-entropy (also called log loss). For binary labels it is:

$$L = -\frac{1}{n}\sum_{i=1}^{n}\Big[\,y_i \log \hat{y}_i + (1 - y_i)\log(1 - \hat{y}_i)\,\Big]$$

Read it intuitively: if the true label is $y_i = 1$, only the first term survives, $-\log\hat{y}_i$ — which is near 0 when $\hat{y}_i$ is close to 1 (good) and shoots up toward infinity as $\hat{y}_i\to 0$ (a confident, wrong prediction is punished brutally). The $y_i = 0$ case is the mirror image. We use cross-entropy rather than MSE because it pairs with the sigmoid to give a clean, convex bowl that gradient descent minimizes smoothly; MSE-plus-sigmoid is bumpy and trains poorly. There's no closed form, so we always optimize with gradient descent.

Concept More than two classes: softmax

When there are $K$ classes (digit 0–9, say), we use one linear score per class and convert the whole vector of scores into probabilities with the softmax function: exponentiate each score and divide by the total, so the outputs are all positive and sum to exactly 1. The class with the highest probability is the prediction. Softmax is the multi-class generalization of the sigmoid, and it's the exact output layer of nearly every classification neural network you'll build in Track 4.

Worked example Classifying tumors with logistic regression

Back to the 569 breast-tumor biopsies from Lesson 3.1. We standardize the features (logistic regression trains best when features are on a common scale — Lesson 2.5), fit the model on the training half, and score the untouched test half. Note the discipline: the scaler is fit on train only, then applied to test.

98.2% of the held-out tumors classified correctly with the simplest possible model. Now the part that makes logistic regression special — it doesn't just say "class 1," it tells you how sure it is. Let's look at its probability for the first test example:

probs = clf.predict_proba(X_test_s[:1])[0]   # [P(malignant), P(benign)]
print("P(malignant), P(benign):", probs.round(3))
print("true label (0=malignant, 1=benign):", int(y_test[0]))
OutputP(malignant), P(benign): [0.989 0.011] true label (0=malignant, 1=benign): 0

The model assigned 98.9% probability to "malignant," and the true label was indeed malignant. That calibrated confidence is the payoff over a model that only blurts a hard label: a doctor can treat a 0.98 very differently from a 0.55. In Lesson 3.4 we'll see that this probability, plus a tunable threshold, is the lever for trading off the two kinds of mistakes.

★ Key points to remember
  • Logistic regression computes a linear score, then squashes it with the sigmoid into a probability in $(0,1)$.
  • Predict class 1 when the probability exceeds a threshold (default 0.5); the decision boundary is linear.
  • It's trained by minimizing cross-entropy (log loss), which punishes confident wrong answers hard.
  • Softmax extends it to $K$ classes: scores → positive numbers that sum to 1.
  • Its output is a real probability, not just a label — that confidence is the whole point.

Practice Build the reasoning yourself

1A logistic model has score $z = 0$ for example A, $z = 2$ for example B, and $z = -2$ for example C. Using $\sigma(0)=0.5$, $\sigma(2)\approx 0.88$, $\sigma(-2)\approx 0.12$, give each predicted probability and the predicted class at threshold 0.5.
Show solution & reasoning

A: $\sigma(0)=0.50$ — right on the boundary, predicted class 1 (ties usually go to 1, but it's a coin-flip case).

B: $\sigma(2)=0.88$ → above 0.5 → class 1, and confidently so.

C: $\sigma(-2)=0.12$ → below 0.5 → class 0, confidently. Notice the symmetry: a score of $+2$ and $-2$ give probabilities equidistant from 0.5 (0.88 and 0.12), because the sigmoid is symmetric about $z=0$.

2For a fraud detector you'd rather catch more fraud even at the cost of more false alarms. The model outputs probabilities. Without retraining, how do you make it flag more transactions as fraud?
Show solution & reasoning

Lower the threshold below 0.5 — say to 0.2. Now any transaction the model rates above 20% likely fraud gets flagged, so more get caught (higher recall) at the cost of more false alarms (lower precision). The model and its probabilities don't change at all; you're only moving the cutoff that turns a probability into a yes/no.

This is exactly why a model that outputs probabilities is so valuable: one trained model gives you a whole dial of precision/recall trade-offs, which is the subject of the next lesson. (A model that only emits hard labels throws this flexibility away.)

3Cross-entropy loss for a single example with true label $y=1$ is $-\log(\hat{y})$. Compute it for $\hat{y} = 0.9$ and for $\hat{y} = 0.1$ (use $\log$ base $e$: $\log 0.9 \approx -0.105$, $\log 0.1 \approx -2.303$). What does the comparison show?
Show solution & reasoning

For $\hat{y}=0.9$: loss $= -(-0.105) = \mathbf{0.105}$ — small, because the model was confident and right.

For $\hat{y}=0.1$: loss $= -(-2.303) = \mathbf{2.303}$ — over 20× larger, because the model confidently bet on the wrong class.

This is the defining behavior of cross-entropy: being confidently wrong is enormously more expensive than being confidently right is cheap. As $\hat{y}\to 0$ while the truth is 1, the loss $\to \infty$ — so the optimizer is pushed hard away from confident mistakes, which is exactly the incentive you want in a probabilistic classifier.

★ Deep dive: log-odds, why it's "linear," and calibration▸

There's an elegant reason the sigmoid is the right squashing function. If you ask "what does the linear score $z$ actually equal?", the answer is the log-odds: $z = \log\frac{p}{1-p}$, where $p$ is the probability of class 1. So logistic regression is literally a linear model — of the log-odds. Each weight $w_j$ has a crisp meaning: a one-unit increase in feature $j$ adds $w_j$ to the log-odds, i.e. multiplies the odds by $e^{w_j}$. That's why epidemiologists and credit-risk analysts love it: "this risk factor multiplies the odds by 1.4" is a sentence a human can act on.

Cross-entropy isn't an arbitrary choice either — it's the maximum-likelihood estimator (Lesson 1.10) for this model. Minimizing cross-entropy is identical to choosing the weights that make the observed labels most probable. The same principle that gave us MSE for regression (under Gaussian noise) gives cross-entropy for classification (under the Bernoulli/categorical model). One idea, two losses.

A subtle but important property is calibration: do the probabilities mean what they say? If you gather all the examples a model rated "0.7," about 70% of them should truly be class 1 for the model to be well-calibrated. Logistic regression tends to be reasonably calibrated out of the box (a side benefit of the maximum-likelihood fit), whereas many other classifiers are not, and need a post-hoc calibration step. We'll return to trusting model outputs when we reach evaluation (Track 12), but it's worth planting the flag now: an accurate model and a well-calibrated model are not the same thing.

You can now predict categories and get probabilities. But "98% accuracy" can hide a useless model — imagine 98% of emails aren't spam.
